# 结构化输出提示：让AI模型输出你想要的格式

## 一、什么是结构化输出提示

在日常使用大语言模型时，我们通常得到的是自由格式的自然语言回复。但在实际开发中，我们往往需要模型以特定的格式返回数据，比如JSON格式的API响应、Markdown格式的文档、XML格式的配置文件等。**结构化输出提示（Structured Output Prompting）** 就是通过精心设计提示词，引导模型按照预定义的结构化格式输出内容的技术。

为什么结构化输出如此重要？想象一下，如果你正在开发一个智能客服系统，你需要从用户的对话中提取关键信息（如姓名、问题类型、紧急程度），然后传递给下游系统处理。如果模型的输出是一段自由文本，你需要额外编写解析代码，而且解析过程容易出错。但如果让模型直接输出JSON格式，下游系统就可以直接使用，大大简化了系统架构。

## 二、常见的结构化格式

### JSON格式

JSON（JavaScript Object Notation）是目前最常用的结构化输出格式，几乎所有编程语言都能方便地解析。在提示词中指定JSON输出时，你需要：

1. 明确说明输出格式为JSON
2. 定义字段名称和含义
3. 指定数据类型（字符串、数字、布尔值、数组等）
4. 说明是否为必填字段

```
**基本示例：**

提示词：
"请分析以下用户反馈，并以JSON格式输出。包含字段：sentiment（正面/负面/中性）、keywords（关键词数组）、summary（一句话总结）。用户反馈：「这个产品太好用了，界面简洁，功能强大，但是价格稍微有点贵。」"

期望输出：
{
  "sentiment": "正面",
  "keywords": ["好用", "界面简洁", "功能强大", "价格贵"],
  "summary": "用户对产品的功能和界面非常满意，但对价格有顾虑。"
}
```

### Markdown格式

Markdown是用于文档生成的优秀格式。当你需要生成报告、教程或任何带格式的文档时，可以让模型输出Markdown。

```
**示例：**

提示词：
"请用Markdown格式写一份Python入门教程大纲，包含多级标题、列表、代码块和表格。"
```

### XML格式

XML在企业级应用中仍然广泛使用，特别是需要严格的标签嵌套结构时。

```
**示例：**

提示词：
"请将以下对话记录转换为XML格式，根节点为<conversation>，每条消息用<message>包裹，包含role属性和文本内容。"
```

## 三、使用Schema约束输出

为了让模型输出更加可控，你可以使用**JSON Schema**来精确描述输出格式。JSON Schema是一种用于描述JSON数据结构的规范。

```markdown
**使用JSON Schema定义输出结构：**

提示词：
"请根据以下Schema输出用户信息JSON。Schema定义如下：
- name: string, 必填, 用户姓名
- age: integer, 必填, 年龄（1-120）
- email: string, 必填, 邮箱格式
- tags: array of strings, 可选, 用户标签
- address: object, 可选
  - city: string
  - district: string

用户描述：「张三，28岁，住在北京市朝阳区，邮箱是zhangsan@example.com，喜欢编程和摄影。」"
```

使用Schema的好处是：
- **类型安全**：字段类型被明确定义，减少解析错误
- **结构验证**：可以在后端使用Schema验证器检查输出
- **减少歧义**：模型清楚知道每个字段的要求

## 四、实用场景与示例

### 场景一：从文本中提取结构化数据

这在处理非结构化文档时特别有用，比如从简历中提取信息、从合同中标引出关键条款。

```markdown
提示词：
"你是一个简历解析助手。请从以下简历文本中提取信息，输出为JSON：
- personal_info: {name, phone, email}
- education: [{school, degree, major, year}]
- experience: [{company, title, duration, responsibilities}]
- skills: [string]

简历文本：{粘贴简历内容}"
```

### 场景二：生成Markdown表格

当你需要对比分析或生成报表时：

```markdown
提示词：
"请比较Python、JavaScript和Go三种语言在Web开发中的优劣，用Markdown表格输出，包含：语言名称、学习难度、性能、生态系统、适用场景、推荐指数五列。"
```

## 五、处理格式错误

即使精心设计了提示词，模型有时仍会输出格式不正确的内容。以下是一些常见问题和解决方案：

### 问题1：JSON中混入了额外文本

模型有时会在JSON前后添加解释性文字，如"根据分析，输出如下："。

**解决方案**：在提示词中明确要求"只输出JSON，不要添加任何解释或说明"。你还可以在后端使用正则表达式提取花括号之间的内容。

### 问题2：字段缺失或类型错误

**解决方案**：提供更详细的Schema说明，并使用代码进行后验证：

```python
import json

def validate_and_fix_llm_output(raw_output, schema):
    # 尝试提取JSON
    try:
        data = json.loads(raw_output)
    except json.JSONDecodeError:
        # 尝试从文本中提取JSON块
        import re
        match = re.search(r'\{.*\}', raw_output, re.DOTALL)
        if match:
            data = json.loads(match.group())
        else:
            raise ValueError("无法从输出中提取有效的JSON")
    
    # 验证并填充缺失字段
    for field, info in schema.items():
        if field not in data:
            data[field] = info.get("default", None)
    
    return data
```

### 问题3：多层嵌套结构处理不好

**解决方案**：提供逐级的格式示例，让模型看到完整的嵌套结构。如果嵌套太深，可以考虑分步提取，每步处理一层。

## 六、结合Function Calling

**Function Calling（函数调用）** 是结构化输出的一种高级形式。它的核心思想是：让模型不直接生成答案文本，而是生成一个函数调用的请求，由外部系统执行该函数并返回结果。

与普通的JSON输出相比，Function Calling的优势在于：
- 格式更加标准化，各大模型厂商都支持相似的格式
- 模型被专门训练识别何时应该调用函数
- 可以直接集成到应用的业务逻辑中

```markdown
**Function Calling示例：**

定义工具：
{
  "name": "get_weather",
  "description": "获取指定城市的天气信息",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "城市名称，如'北京'"
      },
      "unit": {
        "type": "string",
        "enum": ["celsius", "fahrenheit"],
        "description": "温度单位"
      }
    },
    "required": ["city"]
  }
}

当用户问"北京今天多少度？"时，模型不会直接编造答案，而是返回一个函数调用：
{"name": "get_weather", "arguments": {"city": "北京", "unit": "celsius"}}

系统执行这个函数后，将真实的天气数据返回给模型，模型再生成最终的自然语言回答。
```

## 七、最佳实践总结

1. **提供示例是王道**：在提示词中给出1-2个完整的输入输出示例（Few-shot），模型的格式遵循率会大幅提升。

2. **使用占位符**：对于动态内容，使用占位符（如`{{field_name}}`）让模型填充。

3. **分步处理复杂结构**：如果目标结构非常复杂，可以拆分为多个步骤，每步生成一部分，逐步拼接。

4. **始终进行后验证**：不要完全信任模型的输出格式。在代码中添加格式验证和异常处理逻辑。

5. **结合系统提示**：在系统提示（System Prompt）中设定输出格式规则，可以持续生效。

6. **关注模型版本**：不同版本的模型对格式遵循能力差异很大，较新的模型通常表现更好。

掌握结构化输出提示技术，你就能将大语言模型从简单的对话工具转变为可编程的、可集成到软件系统中的强大组件。这是AI应用开发中最核心的技能之一。
