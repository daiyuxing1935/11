## 二、最小流式代码

```python
def stream_answer(model, messages):
    full_text = []
    for chunk in model.stream(messages):
        text = chunk.content
        if isinstance(text, str) and text:
            print(text, end="", flush=True)
            full_text.append(text)
    print()
    return "".join(full_text)
```

调用后仍要把完整结果写回历史：

```python
messages.append({"role": "user", "content": user_text})
answer = stream_answer(model, messages)
messages.append({"role": "assistant", "content": answer})
```

### `stream()` 与 `invoke()` 的框架差异

| 调用 | 返回方式 | 适合场景 |
|---|---|---|
| `model.invoke(messages)` | 等模型生成完后返回一个完整 `AIMessage` | 后台任务、短回答、只关心最终结果 |
| `model.stream(messages)` | 返回可迭代的 `AIMessageChunk` 序列 | 对话界面、长回答、需要尽快展示首字 |

`chunk.content` 是“本次新增片段”，不是截至当前的完整答案，所以程序需要按顺序累积。`flush=True` 只是让终端立即显示字符，不属于 LangChain。流结束后仍应得到一份完整文本并写回消息历史，否则下一轮上下文会缺少模型回答。

某些供应商会把 `content` 表示成内容块列表，因此工程代码要在模型适配边界统一提取文本，而不是让页面、日志和业务层分别猜测返回结构。

## 三、为什么片段不总是字符串

不同模型供应商可能返回字符串、空片段，或结构化内容块。不要写 `str(chunk.content)` 作为兜底，否则 `None` 会变成回答中的文字。

在 `solution.py` 中把边界处理实现为 `normalize_stream_chunks`。真实的 `model.stream()` 返回一次性迭代器，元素通常是带 `content` 属性的 `AIMessageChunk`，测试时也要覆盖这种结构：

```python
def normalize_stream_chunks(chunks):
    parts = []
    for chunk in chunks:
        if chunk is None:
            continue
        if isinstance(chunk, str):
            content = chunk
        elif isinstance(chunk, dict):
            content = chunk.get("content")
        elif hasattr(chunk, "content"):
            content = chunk.content
        else:
            raise ValueError("不支持的模型片段结构")

        if content is None or content == "":
            continue
        if isinstance(content, str):
            parts.append(content)
        elif isinstance(content, list):
            for block in content:
                if isinstance(block, dict) and block.get("type") == "text":
                    text = block.get("text")
                    if isinstance(text, str) and text:
                        parts.append(text)
        else:
            raise ValueError("不支持的模型内容结构")
    return "".join(parts)
```

这个函数只负责统一片段；是否真正“边生成边显示”，还要由 `app.py` 的流式循环保证。

## 四、加入可执行的错误提示

```python
try:
    answer = stream_answer(model, messages)
except TimeoutError:
    print("请求超时：请检查网络，或把问题缩短后重试")
except Exception as exc:
    print(f"模型调用失败：{type(exc).__name__}。请检查 Key、Base URL 与模型名")
```

不要在错误日志中输出 API Key 或完整请求头。

在 `app.py` 中导入 `normalize_stream_chunks` 并接入 `model.stream()`，再用正常片段、空片段和内容块各运行一次。

集成检查会确认三个可观察行为：代码逐个遍历 `model.stream()`，在循环内用 `flush=True` 立即输出，并同时累积完整回答。若先把流全部收集完再统一打印，即使最终文字正确也不能通过“逐片段即时输出”测试点。
