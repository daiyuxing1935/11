# 订阅/发布模式（事件驱动协作）

## 引言

在多智能体系统中，并非所有的通信都需要知道"谁应该接收我的消息"。很多时候，Agent只需要宣布"发生了一件事"，而哪些Agent关心这件事、如何响应，则完全由接收方自己决定。这种"我只管发布，谁订阅谁接收"的模式，就是**订阅/发布模式（Pub/Sub Pattern）**。

订阅/发布模式的核心思想很简单：**让信息的发送者（Publisher）和接收者（Subscriber）彻底解耦**。发布者不需要知道有哪些订阅者、订阅者在哪里、甚至不需要知道是否有人订阅。这就像Twitter上的推文——你只管发，关注你的人自然会看到。在多智能体系统中，这种松耦合带来了极大的灵活性。

---

## 核心概念

### 三大角色

```
┌─────────────┐      事件(Event)      ┌─────────────┐
│  Publisher  │ ──────────────────>  │   Broker     │
│  (发布者)   │                       │  (消息代理)  │
└─────────────┘                       └──────┬──────┘
                                              │
                              ┌───────────────┼───────────────┐
                              │               │               │
                              ▼               ▼               ▼
                        ┌──────────┐  ┌──────────┐  ┌──────────┐
                        │Subscriber│  │Subscriber│  │Subscriber│
                        │   A      │  │   B      │  │   C      │
                        └──────────┘  └──────────┘  └──────────┘
```

| 角色 | 职责 | 在Agent系统中的体现 |
|------|------|-------------------|
| **Publisher（发布者）** | 产生事件并发送给Broker | 完成任务的Agent广播结果 |
| **Broker（消息代理）** | 接收事件、管理订阅、路由分发 | 消息中间件/事件总线 |
| **Subscriber（订阅者）** | 声明兴趣并接收匹配的事件 | 关注特定类型事件的Agent |

## Topic-based发布订阅

Topic-based是最常用的Pub/Sub实现方式。事件按Topic分类，订阅者声明自己对哪些Topic感兴趣。

### Topic设计

```python
# ============ Topic层级设计 ============

class TopicHierarchy:
    """
    Topic推荐采用层级命名结构，使用 '.' 作为分隔符。

    例如：
    - task.research.completed        # 研究任务完成
    - task.research.failed           # 研究任务失败
    - system.alert.critical          # 系统严重告警
    - data.updated.knowledge_base    # 知识库数据更新
    - agent.status.online            # Agent上线通知
    - agent.status.offline           # Agent离线通知
    - human.feedback.required        # 需要人工介入
    """

    # 推荐的Topic设计规范
    TOPIC_PATTERN = "{domain}.{entity}.{action}"

    # 常用Topic示例
    TASK_COMPLETED = "task.*.completed"
    TASK_FAILED = "task.*.failed"
```

### Wildcard订阅

Pub/Sub系统通常支持两种通配符：

| 通配符 | 含义 | 示例 |
|--------|------|------|
| `*` (star) | 匹配单个Topic层级 | `task.*.completed` 匹配 `task.research.completed`，但不匹配 `task.research.sub.completed` |
| `#` (hash) | 匹配零个或多个Topic层级 | `task.#` 匹配 `task.research.completed`，`task.translate.failed`，甚至 `task` |

```python
import re
from typing import Pattern


class TopicMatcher:
    """Topic通配符匹配器"""

    @staticmethod
    def matches(pattern: str, topic: str) -> bool:
        """
        检查一个topic是否匹配给定的pattern。

        pattern支持:
        - * : 匹配单个层级
        - # : 匹配零个或多个层级
        """
        # 将pattern转换为正则表达式
        parts = pattern.split(".")
        regex_parts = []

        for part in parts:
            if part == "#":
                regex_parts.append(r"(?:\.[^.]+)*")
            elif part == "*":
                regex_parts.append(r"\.[^.]+")
            else:
                regex_parts.append(r"\." + re.escape(part))

        # 去除开头的空匹配
        regex_str = "^" + "".join(regex_parts)
        if regex_str.startswith("^\."):
            regex_str = "^" + regex_str[2:]

        regex_str += "$"
        return bool(re.match(regex_str, topic))


# 测试
assert TopicMatcher.matches("task.*.completed",
                            "task.research.completed") == True
assert TopicMatcher.matches("task.*.completed",
                            "task.research.failed") == False
assert TopicMatcher.matches("task.#",
                            "task.research.sub.completed") == True
assert TopicMatcher.matches("system.alert.*",
                            "system.alert.critical") == True
```

## Agent如何订阅感兴趣的事件

### 订阅管理

```python
from typing import Callable, Any
from collections import defaultdict
import asyncio
import json


class Event:
    """标准事件格式"""
    def __init__(self, topic: str, payload: Any,
                 source: str = None,
                 event_id: str = None,
                 timestamp: float = None,
                 correlation_id: str = None):
        self.topic = topic
        self.payload = payload
        self.source = source  # 发布者ID
        self.event_id = event_id or str(uuid.uuid4())
        self.timestamp = timestamp or time.time()
        self.correlation_id = correlation_id  # 关联ID（用于追踪）

    def to_dict(self) -> dict:
        return {
            "topic": self.topic,
            "payload": self.payload,
            "source": self.source,
            "event_id": self.event_id,
            "timestamp": self.timestamp,
            "correlation_id": self.correlation_id
        }


class Subscription:
    """订阅信息"""
    def __init__(self, subscriber_id: str,
                 topic_pattern: str,
                 callback: Callable,
                 filter_func: Callable = None):
        self.subscriber_id = subscriber_id
        self.topic_pattern = topic_pattern
        self.callback = callback
        self.filter_func = filter_func  # 额外的内容过滤

    def matches(self, event: Event) -> bool:
        """检查事件是否匹配此订阅"""
        if not TopicMatcher.matches(self.topic_pattern, event.topic):
            return False
        if self.filter_func and not self.filter_func(event):
            return False
        return True
```

### Agent端订阅实现

```python
class PubSubAgent:
    """具有发布/订阅能力的Agent"""

    def __init__(self, agent_id: str, broker: 'EventBroker'):
        self.agent_id = agent_id
        self.broker = broker
        self._subscriptions: dict[str, Subscription] = {}
        self._event_queue = asyncio.Queue()

    def subscribe(self, topic_pattern: str,
                  callback: Callable = None,
                  filter_func: Callable = None) -> str:
        """
        订阅一个Topic模式。

        返回subscription_id，用于后续退订。
        """
        sub = Subscription(
            subscriber_id=self.agent_id,
            topic_pattern=topic_pattern,
            callback=callback or self._default_handler,
            filter_func=filter_func
        )
        sub_id = self.broker.subscribe(sub)
        self._subscriptions[sub_id] = sub
        print(f"[{self.agent_id}] 订阅了: {topic_pattern}")
        return sub_id

    def unsubscribe(self, sub_id: str):
        """退订"""
        if sub_id in self._subscriptions:
            self.broker.unsubscribe(sub_id)
            pattern = self._subscriptions[sub_id].topic_pattern
            del self._subscriptions[sub_id]
            print(f"[{self.agent_id}] 退订: {pattern}")

    def publish(self, topic: str, payload: Any,
                correlation_id: str = None):
        """发布事件"""
        event = Event(
            topic=topic,
            payload=payload,
            source=self.agent_id,
            correlation_id=correlation_id
        )
        self.broker.publish(event)
        print(f"[{self.agent_id}] 发布 → {topic}: "
              f"{str(payload)[:50]}...")

    async def _default_handler(self, event: Event):
        """默认事件处理器：放入队列"""
        await self._event_queue.put(event)

    async def process_events(self):
        """处理收到的事件"""
        while True:
            event = await self._event_queue.get()
            await self.handle_event(event)

    async def handle_event(self, event: Event):
        """子类重写此方法以处理事件"""
        print(f"[{self.agent_id}] 收到事件: {event.topic}")
```

## 事件路由和过滤

### EventBroker实现

```python
import uuid
import time
import threading
from collections import defaultdict


class EventBroker:
    """
    事件代理：管理订阅关系并路由事件。

    职责：
    1. 维护订阅表（topic_pattern → subscriptions）
    2. 接收事件并匹配订阅者
    3. 异步分发事件
    """

    def __init__(self):
        self._subscriptions: dict[str, Subscription] = {}
        self._topic_index: dict[str, set[str]] = defaultdict(set)
        self._lock = threading.RLock()

        # 事件历史（用于持久化和重放）
        self._event_history: list[Event] = []
        self._max_history = 10000

        # 统计信息
        self._stats = {
            "events_published": 0,
            "events_delivered": 0,
            "events_dropped": 0,
            "active_subscriptions": 0
        }

    def subscribe(self, subscription: Subscription) -> str:
        """注册订阅"""
        sub_id = f"sub_{uuid.uuid4().hex[:8]}"
        with self._lock:
            self._subscriptions[sub_id] = subscription
            # 构建topic索引（用于高效匹配）
            # 对不包含通配符的topic做精确索引
            if "*" not in subscription.topic_pattern and \
               "#" not in subscription.topic_pattern:
                self._topic_index[subscription.topic_pattern].add(sub_id)
            self._stats["active_subscriptions"] = len(self._subscriptions)
        return sub_id

    def unsubscribe(self, sub_id: str):
        """取消订阅"""
        with self._lock:
            if sub_id in self._subscriptions:
                sub = self._subscriptions[sub_id]
                # 清理索引
                if sub.topic_pattern in self._topic_index:
                    self._topic_index[sub.topic_pattern].discard(sub_id)
                del self._subscriptions[sub_id]
                self._stats["active_subscriptions"] = len(self._subscriptions)

    def publish(self, event: Event, sync: bool = False):
        """
        发布事件。

        sync=True: 同步发送（等待所有订阅者处理完）
        sync=False: 异步发送（放入队列即返回，默认）
        """
        self._stats["events_published"] += 1

        # 保存历史
        if len(self._event_history) >= self._max_history:
            self._event_history.pop(0)
        self._event_history.append(event)

        # 查找匹配的订阅者
        matched_subs = self._find_matching_subscriptions(event)

        if not matched_subs:
            self._stats["events_dropped"] += 1
            return

        self._stats["events_delivered"] += len(matched_subs)

        # 分发事件
        for sub in matched_subs:
            if sync:
                sub.callback(event)
            else:
                # 异步分发（使用线程池或事件循环）
                threading.Thread(
                    target=sub.callback,
                    args=(event,),
                    daemon=True
                ).start()

    def _find_matching_subscriptions(self,
                                      event: Event) -> list[Subscription]:
        """查找匹配的订阅者"""
        matched = []

        with self._lock:
            # 1. 先查精确索引（O(1)查找）
            exact_matches = self._topic_index.get(event.topic, set())
            for sub_id in exact_matches:
                sub = self._subscriptions.get(sub_id)
                if sub and sub.matches(event):
                    matched.append(sub)

            # 2. 再遍历带有通配符的订阅
            for sub_id, sub in self._subscriptions.items():
                if sub_id in exact_matches:
                    continue  # 已通过精确索引匹配过
                if sub.matches(event):
                    matched.append(sub)

        return matched

    def get_stats(self) -> dict:
        """获取Broker统计信息"""
        return dict(self._stats)

    def replay_history(self, topic_pattern: str = None,
                       from_timestamp: float = None) -> list[Event]:
        """重放历史事件（用于新的订阅者追赶状态）"""
        events = self._event_history
        if from_timestamp:
            events = [e for e in events
                     if e.timestamp >= from_timestamp]
        if topic_pattern:
            events = [e for e in events
                     if TopicMatcher.matches(topic_pattern, e.topic)]
        return events
```

### 内容过滤（Content-based Filtering）

除了Topic匹配，订阅者还可以通过filter_func进行更细粒度的内容过滤。

```python
def create_content_filter(field: str, operator: str,
                          value: Any) -> Callable:
    """
    创建内容过滤函数。

    支持操作符: eq, ne, gt, lt, gte, lte, in, contains
    """
    def filter_func(event: Event) -> bool:
        payload = event.payload
        if isinstance(payload, dict):
            actual = payload.get(field)
        else:
            actual = getattr(payload, field, None)

        if operator == "eq": return actual == value
        if operator == "ne": return actual != value
        if operator == "gt": return actual > value
        if operator == "lt": return actual < value
        if operator == "gte": return actual >= value
        if operator == "lte": return actual <= value
        if operator == "in": return actual in value
        if operator == "contains": return value in str(actual)
        return False

    return filter_func


# ============ 使用示例 ============
agent = PubSubAgent("alert_handler", broker)

# 只关注严重程度 >= 8 的告警
agent.subscribe(
    topic_pattern="system.alert.*",
    filter_func=create_content_filter("severity", "gte", 8)
)

# 只关注特定来源的事件
agent.subscribe(
    topic_pattern="task.completed",
    filter_func=lambda e: e.source in ["worker_01", "worker_02"]
)
```

## 松耦合的优势

### 1. 即插即用

新增Agent不需要修改任何现有代码——只需要订阅它感兴趣的Topic。

```python
# 新增一个"日志记录Agent"——零侵入
logger_agent = PubSubAgent("logger", broker)
logger_agent.subscribe("#")  # 订阅所有事件，记录日志

# 新增一个"监控告警Agent"——同样零侵入
monitor_agent = PubSubAgent("monitor", broker)
monitor_agent.subscribe("system.alert.*")
monitor_agent.subscribe("task.*.failed")
```

### 2. 动态重配置

可以在运行时改变订阅关系，无需重启系统。

```python
# 在流量高峰时，让分析Agent暂时减少订阅
analysis_agent.unsubscribe("task.*.low_priority")

# 发现问题后，让调试Agent临时订阅更多Topic
debug_agent.subscribe("agent.status.*")
debug_agent.subscribe("error.#")
```

### 3. 天然的广播和多播

一条消息自动到达所有感兴趣的订阅者，不需要发布者做任何额外工作。

```python
# Manager只需发一条消息
manager.publish("task.research.completed", {
    "task_id": "task_001",
    "result_summary": "市场调研完成"
})

# 同时触达多个订阅者，各做各的事：
# - Analytics Agent → 更新仪表盘
# - Logging Agent → 记录日志
# - Writing Agent → 开始撰写报告
# - Notification Agent → 发送提醒
# 所有这些都不需要Manager知道！
```

### 4. 弹性容错

订阅者宕机不影响发布者；订阅者恢复后可以重放历史事件。

```python
# 新启动的Agent通过重放历史事件来追上当前状态
class LateJoiningAgent(PubSubAgent):
    async def catch_up(self):
        """新Agent通过重放历史事件来同步状态"""
        history = self.broker.replay_history(
            topic_pattern="data.updated.#",
            from_timestamp=time.time() - 3600  # 最近1小时
        )
        for event in history:
            await self.handle_event(event)
        print(f"[{self.agent_id}] 状态同步完成，处理了 {len(history)} 个历史事件")
```

## 在分布式Agent系统中的应用

### 基于Redis的实现

```python
import redis
import json
import threading


class RedisEventBroker:
    """
    基于Redis Pub/Sub的分布式事件代理。

    适合跨进程、跨服务器的多Agent系统。

    Redis Pub/Sub的优势：
    - 高性能（数十万消息/秒）
    - 天然分布式
    - 成熟的生态
    """

    def __init__(self, redis_url: str = "redis://localhost:6379",
                 channel_prefix: str = "agent:events"):
        self.redis = redis.from_url(redis_url)
        self.channel_prefix = channel_prefix
        self._local_subscribers: dict[str, list[Callable]] = \
            defaultdict(list)
        self._pubsub = None

    def publish(self, event: Event):
        """通过Redis发布事件"""
        channel = f"{self.channel_prefix}:{event.topic}"
        message = json.dumps(event.to_dict())
        self.redis.publish(channel, message)

        # 也可以同时写入Redis Stream用于持久化
        self.redis.xadd(
            f"{self.channel_prefix}:stream",
            {"event": message},
            maxlen=10000
        )

    def subscribe(self, topic_pattern: str,
                  callback: Callable) -> str:
        """
        订阅Topic（在Redis中注册监听）。

        注意：Redis Pub/Sub不直接支持通配符匹配，
        但Redis支持PSUBSCRIBE命令使用glob模式。
        """
        import re

        # 将topic pattern转换为Redis glob pattern
        redis_pattern = topic_pattern.replace(".", "/")
        redis_pattern = redis_pattern.replace("*", "[^/]*")\
                                     .replace("#", ".*")

        # 启动监听线程
        def listener():
            pubsub = self.redis.pubsub()
            pubsub.psubscribe(f"{self.channel_prefix}:{redis_pattern}")
            for message in pubsub.listen():
                if message["type"] == "pmessage":
                    event_data = json.loads(message["data"])
                    event = Event(
                        topic=event_data["topic"],
                        payload=event_data["payload"],
                        source=event_data.get("source"),
                        event_id=event_data.get("event_id"),
                        timestamp=event_data.get("timestamp"),
                        correlation_id=event_data.get("correlation_id")
                    )
                    callback(event)

        thread = threading.Thread(target=listener, daemon=True)
        thread.start()

        return f"redis_sub_{uuid.uuid4().hex[:8]}"
```

### 基于RabbitMQ的实现

```python
# RabbitMQ天然支持Topic Exchange，是Pub/Sub的理想后端

class RabbitMQEventBroker:
    """
    基于RabbitMQ的Topic Exchange实现。

    RabbitMQ的Topic Exchange原生支持 '.' 分隔的routing key
    和 '*' / '#' 通配符，完美匹配我们的Topic设计。
    """

    def __init__(self, amqp_url: str = "amqp://localhost",
                 exchange_name: str = "agent_events"):
        import pika
        self.connection = pika.BlockingConnection(
            pika.URLParameters(amqp_url)
        )
        self.channel = self.connection.channel()
        self.exchange = exchange_name

        # 声明Topic Exchange
        self.channel.exchange_declare(
            exchange=self.exchange,
            exchange_type="topic",
            durable=True
        )

    def publish(self, event: Event):
        """通过RabbitMQ发布事件"""
        message = json.dumps(event.to_dict())
        self.channel.basic_publish(
            exchange=self.exchange,
            routing_key=event.topic,  # topic作为routing key
            body=message,
            properties=pika.BasicProperties(
                delivery_mode=2,  # 持久化
                content_type="application/json"
            )
        )

    def subscribe(self, topic_pattern: str, callback: Callable):
        """订阅Topic"""
        # 声明临时队列
        result = self.channel.queue_declare(queue="", exclusive=True)
        queue_name = result.method.queue

        # 绑定队列到Exchange
        self.channel.queue_bind(
            exchange=self.exchange,
            queue=queue_name,
            routing_key=topic_pattern
        )

        # 开始消费
        def on_message(ch, method, properties, body):
            event_data = json.loads(body)
            event = Event(**event_data)
            callback(event)
            ch.basic_ack(delivery_tag=method.delivery_tag)

        self.channel.basic_consume(
            queue=queue_name,
            on_message_callback=on_message
        )

        # 在单独的线程中运行
        thread = threading.Thread(
            target=self.channel.start_consuming,
            daemon=True
        )
        thread.start()
```

## 完整实战示例：事件驱动的多Agent系统

```python
import asyncio
import time

class EventDrivenSystem:
    """
    事件驱动的多Agent系统示例。

    场景：智能内容生产流水线。
    当"原材料就绪"事件发生时，多个Agent自动响应，最终产出成品。
    """

    def __init__(self):
        self.broker = EventBroker()

        # 创建Agents
        self.collector = CollectorAgent("collector", self.broker)
        self.analyzer = AnalyzerAgent("analyzer", self.broker)
        self.writer = WriterAgent("writer", self.broker)
        self.reviewer = ReviewerAgent("reviewer", self.broker)
        self.notifier = NotifierAgent("notifier", self.broker)
        self.orchestrator = OrchestratorAgent("orch", self.broker)

    def setup(self):
        """
        配置订阅关系。

        注意：这里展示了Pub/Sub强大的解耦能力——
        每个Agent只订阅自己关心的Topic，不需要知道消息来自谁。
        """

        # Collector: 关注数据请求
        self.collector.subscribe("request.data.*")

        # Analyzer: 数据收集完成后开始分析
        self.analyzer.subscribe("data.collected")

        # Writer: 分析完成后开始写作
        self.writer.subscribe("analysis.completed")

        # Reviewer: 草稿完成后开始审校
        self.reviewer.subscribe("draft.completed")

        # Notifier: 关注所有完成和异常事件
        self.notifier.subscribe("pipeline.completed")
        self.notifier.subscribe("*.failed")

        # Orchestrator: 总管，关注所有阶段性事件
        self.orchestrator.subscribe("pipeline.#")

        print("[System] 所有订阅关系已配置")

    def start(self, task_description: str):
        """启动流水线"""
        print(f"\n{'='*60}")
        print(f"启动流水线: {task_description}")

        # Orchestrator启动流水线
        self.orchestrator.start_pipeline(task_description)


class OrchestratorAgent(PubSubAgent):
    """流水线编排Agent"""

    def __init__(self, agent_id, broker):
        super().__init__(agent_id, broker)
        self.pipeline_state = {}

    def start_pipeline(self, description: str):
        """启动流水线——只需发布一个事件"""
        pipeline_id = f"pipeline_{int(time.time())}"
        self.pipeline_state[pipeline_id] = {
            "description": description,
            "status": "started",
            "stages": {}
        }
        # 发布流水线启动事件
        self.publish("pipeline.started", {
            "pipeline_id": pipeline_id,
            "description": description
        })

    async def handle_event(self, event: Event):
        """响应各阶段事件，推进流水线"""
        topic = event.topic

        if topic == "pipeline.started":
            # 流水线启动 → 请求数据收集
            payload = event.payload
            pipeline_id = payload["pipeline_id"]

            # 按需发布数据请求（面向角色的数据收集）
            self.publish("request.data.market", {
                "pipeline_id": pipeline_id,
                "query": payload["description"],
                "source": "market_db"
            })
            self.publish("request.data.technical", {
                "pipeline_id": pipeline_id,
                "query": payload["description"],
                "source": "tech_papers"
            })

        elif topic == "data.collected":
            # 数据收集完成，记录状态
            self.pipeline_state[event.payload["pipeline_id"]]["stages"]\
                ["data_collection"] = "completed"

        elif topic == "analysis.completed":
            self.pipeline_state[event.payload["pipeline_id"]]["stages"]\
                ["analysis"] = "completed"

        elif topic == "draft.completed":
            self.pipeline_state[event.payload["pipeline_id"]]["stages"]\
                ["draft"] = "completed"

        elif topic == "review.completed":
            # 审校完成 → 流水线完成
            pipeline_id = event.payload["pipeline_id"]
            self.publish("pipeline.completed", {
                "pipeline_id": pipeline_id,
                "final_output": event.payload["final_draft"],
                "completed_at": time.time()
            })

        elif topic.endswith(".failed"):
            # 任何阶段失败 → 记录并通知
            pipeline_id = event.payload.get("pipeline_id")
            error_stage = topic.split(".")[0]
            print(f"[Orchestrator] {error_stage}阶段失败: "
                  f"{event.payload.get('error')}")


class CollectorAgent(PubSubAgent):
    """数据收集Agent"""

    async def handle_event(self, event: Event):
        if event.topic.startswith("request.data."):
            data_type = event.topic.split(".")[-1]
            print(f"[Collector] 收集{data_type}数据...")
            await asyncio.sleep(1)  # 模拟数据收集

            # 数据收集完成 → 发布事件
            collected_data = {
                "market": "市场数据：2024年新能源汽车销量增长30%...",
                "technical": "技术数据：固态电池能量密度突破500Wh/kg..."
            }
            self.publish("data.collected", {
                "pipeline_id": event.payload["pipeline_id"],
                "data": collected_data.get(data_type, {}),
                "data_type": data_type
            })


class AnalyzerAgent(PubSubAgent):
    """数据分析Agent"""

    async def handle_event(self, event: Event):
        if event.topic == "data.collected":
            print(f"[Analyzer] 分析数据...")
            await asyncio.sleep(1.5)

            self.publish("analysis.completed", {
                "pipeline_id": event.payload["pipeline_id"],
                "findings": "关键发现：1. 市场集中度提升... 2. 技术路线分化..."
            })


class WriterAgent(PubSubAgent):
    """内容写作Agent"""

    async def handle_event(self, event: Event):
        if event.topic == "analysis.completed":
            print(f"[Writer] 撰写初稿...")
            await asyncio.sleep(2)

            self.publish("draft.completed", {
                "pipeline_id": event.payload["pipeline_id"],
                "draft": "# 行业分析报告\n\n## 摘要\n..."
            })


class ReviewerAgent(PubSubAgent):
    """内容审校Agent"""

    async def handle_event(self, event: Event):
        if event.topic == "draft.completed":
            print(f"[Reviewer] 审校初稿...")
            await asyncio.sleep(1)

            # 审校通过
            self.publish("review.completed", {
                "pipeline_id": event.payload["pipeline_id"],
                "final_draft": event.payload["draft"] + "\n\n*已审校*",
                "quality_score": 0.92
            })


class NotifierAgent(PubSubAgent):
    """通知Agent"""

    async def handle_event(self, event: Event):
        if event.topic == "pipeline.completed":
            print(f"[Notifier] ✅ 流水线{pipeline_id}已完成！")
        elif event.topic.endswith(".failed"):
            print(f"[Notifier] ❌ 告警：{event.topic}")


# ============ 运行 ============
async def main():
    system = EventDrivenSystem()
    system.setup()

    # 启动流水线
    system.start("分析2024年新能源汽车行业趋势")

    # 给事件处理一些时间
    await asyncio.sleep(5)

    # 查看Broker统计
    print(f"\n[Stats] {system.broker.get_stats()}")


if __name__ == "__main__":
    asyncio.run(main())
```

## 优缺点分析

### 优势

| 优势 | 说明 |
|------|------|
| **极致解耦** | 发布者完全不知道订阅者的存在 |
| **动态灵活** | 可运行时增删Agent、改变订阅关系 |
| **天然广播** | 一条消息自动触达所有订阅者 |
| **可扩展性好** | 新增订阅者不影响现有系统 |
| **事件历史** | 可以存储和重放事件，便于调试和恢复 |
| **跨进程/跨服务器** | 结合Redis/RabbitMQ天然支持分布式 |

### 劣势

| 劣势 | 说明 | 缓解策略 |
|------|------|---------|
| **消息顺序难保证** | 分布式环境下事件到达顺序可能不一致 | 使用correlation_id + 序列号 |
| **调试困难** | 事件流转路径不直观 | 全链路追踪（OpenTelemetry） |
| **可能消息风暴** | 一个事件触发连锁反应，导致事件爆炸 | 限流 + 熔断 |
| **无响应确认** | 发布者不知道事件是否被正确处理 | 引入请求-响应模式补充 |
| **语义不明确** | Topic命名不规范会导致混乱 | 建立Topic命名规范 |

## 与其他模式的组合

Pub/Sub通常不单独使用，而是与其他模式组合：

```
Manager-Worker + Pub/Sub:
  Manager通过Broker发布任务 → Worker订阅特定任务Topic
  这样Manager不需要知道有哪些Worker

专家小组 + Pub/Sub:
  协调器发布评审请求 → 各专家订阅并响应
  评审结果通过Broker汇总

公告板 + Pub/Sub:
  公告板作为持久存储，Pub/Sub作为实时通知
  Agent通过Pub/Sub获得通知，通过公告板读取详情
```

## 小结

订阅/发布模式是多智能体系统中实现**事件驱动协作**的核心机制。它的精髓在于：让Agent像Twitter用户一样——你只管发布你做的事，别人管别人关心什么。这种彻底的解耦让系统获得了极大的灵活性和可扩展性。

**使用场景**：
- 事件驱动的业务流程（如本文的流水线示例）
- 一对多的广播/通知
- 需要运行时动态调整Agent行为的系统
- 跨服务、跨进程的分布式Agent系统

**核心原则**：
- 设计清晰的Topic命名规范，避免语义混乱
- 为关键事件添加correlation_id，便于端到端追踪
- 存储事件历史，支持故障恢复和状态同步
- 在性能敏感场景中使用Redis/RabbitMQ等成熟中间件

**松耦合不等于无秩序**——好的Pub/Sub系统需要清晰的Topic规范和完善的监控，否则就会变成"事件沼泽"。
