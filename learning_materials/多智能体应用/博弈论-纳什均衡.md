# 博弈论入门：纳什均衡与多智能体系统

## 引言

如果你玩过石头剪刀布，你就已经参与过博弈。博弈论（Game Theory）研究的正是多个决策者（玩家）在互动情境下的策略选择——当你的最优决策依赖于别人的决策时，你该怎么做？

在多智能体系统中，博弈论提供了一套强大的数学框架来分析Agent之间的互动行为。特别是当Agent的目标不完全一致时——比如在拍卖中竞争资源、在辩论中争取胜利、或者在协商中追求自身利益最大化——博弈论能帮助我们理解和设计更好的互动策略。本文将从最基础的纳什均衡出发，探索博弈论如何赋能多智能体系统。

## 博弈论基础概念

### 博弈的三要素

任何一个博弈都包含三个基本要素：

```
┌────────────────────────────────────────┐
│                 博弈                   │
│                                       │
│  ┌──────────┐  ┌──────────┐          │
│  │  玩家1   │  │  玩家2   │   ← 玩家 │
│  │ (Agent A)│  │ (Agent B)│          │
│  └────┬─────┘  └────┬─────┘          │
│       │              │               │
│       │  策略空间    │               │
│       │  {合作,背叛} │               │
│       │              │               │
│       ▼              ▼               │
│  ┌────────────────────────┐          │
│  │      收益矩阵          │ ← 收益   │
│  │  (谁得到什么)          │          │
│  └────────────────────────┘          │
└────────────────────────────────────────┘
```

| 要素 | 定义 | 在多Agent系统中的体现 |
|------|------|---------------------|
| **玩家（Players）** | 参与博弈的决策主体 | 每个Agent是一个玩家 |
| **策略（Strategies）** | 每个玩家可选择的行动方案 | Agent的行为策略（合作/竞争/出价） |
| **收益（Payoffs）** | 每个玩家在每种策略组合下的回报 | Agent的奖励函数（任务完成/资源获得） |

### 收益矩阵（Payoff Matrix）

收益矩阵是描述博弈最标准的方式。以经典的"囚徒困境"为例：

```
        玩家B
        合作    背叛
       ┌───────┬───────┐
  合作 │ -1,-1 │ -3,0  │
玩家A  ├───────┼───────┤
  背叛 │ 0,-3  │ -2,-2 │
       └───────┴───────┘
```

解读：每个格子中，第一个数字是玩家A的收益，第二个数字是玩家B的收益。例如，(背叛, 合作) → 玩家A得0（自由），玩家B得-3（重判）。

## 纳什均衡（Nash Equilibrium）

### 概念阐释

**纳什均衡**是博弈论中最核心的概念，由约翰-纳什（John Nash）于1950年提出。它的定义简洁而深刻：

> 在纳什均衡状态下，**没有任何玩家可以通过单方面改变自己的策略来获得更高的收益**。

换句话说，给定其他人的策略，每个玩家都已经选择了对自己最优的策略。纳什均衡是一种"稳定的僵局"——没有人有动机偏离。

### 直观理解

想象你开车行驶在一条公路上。如果所有车都靠右行驶，那你也最好靠右——否则会撞车。同样，如果所有车都靠左，你也应该靠左。这两种状态（全右或全左）都是纳什均衡，因为没有哪个司机会想单方面改变行驶侧。

### 纯策略 vs 混合策略纳什均衡

**纯策略纳什均衡**：每个玩家确定性地选择一个策略。

```
例子：协调博弈
        玩家B
        左      右
       ┌───────┬───────┐
  左   │ 2,2   │ 0,0   │
玩家A  ├───────┼───────┤
  右   │ 0,0   │ 2,2   │
       └───────┴───────┘

两个纯策略纳什均衡：(左,左) 和 (右,右)
```

**混合策略纳什均衡**：玩家以一定概率随机选择策略。

```
例子：石头剪刀布
        玩家B
        石头  剪刀  布
       ┌──────┬──────┬──────┐
  石头 │ 0,0  │ 1,-1 │ -1,1 │
玩家A  ├──────┼──────┼──────┤
  剪刀 │ -1,1 │ 0,0  │ 1,-1 │
       ├──────┼──────┼──────┤
  布   │ 1,-1 │ -1,1 │ 0,0  │
       └──────┴──────┴──────┘

唯一的纳什均衡：每个玩家以1/3概率选择每种策略
```

### 计算纳什均衡的Python实现

```python
import numpy as np
from scipy.optimize import linprog
from itertools import product


class NashEquilibriumSolver:
    """
    求解两人博弈的纳什均衡。

    对于两人零和博弈：用线性规划求解
    对于两人一般博弈：用Lemke-Howson算法或支持枚举
    """

    @staticmethod
    def solve_zero_sum(payoff_matrix: np.ndarray) -> tuple:
        """
        求解两人零和博弈的纳什均衡（最小最大定理）。

        零和博弈: A的收益 + B的收益 = 0（一个赢多少，另一个就输多少）

        对玩家A（行玩家）：
        maximize v
        subject to:
            sum_i x_i * payoff[i][j] >= v  for all j
            sum_i x_i = 1
            x_i >= 0
        """
        m, n = payoff_matrix.shape

        # 用线性规划求解
        # 转换为标准形式：minimize -v
        c = np.zeros(m + 1)
        c[-1] = -1  # 最大化v等价于最小化-v

        # 约束：payoff_matrix^T * x >= v
        A_ub = np.hstack([-payoff_matrix.T, np.ones((n, 1))])
        b_ub = np.zeros(n)

        # 等式约束：sum(x) = 1
        A_eq = np.ones((1, m + 1))
        A_eq[0, -1] = 0
        b_eq = np.array([1.0])

        # 变量边界
        bounds = [(0, None)] * m + [(None, None)]

        result = linprog(c, A_ub=A_ub, b_ub=b_ub,
                        A_eq=A_eq, b_eq=b_eq, bounds=bounds,
                        method='highs')

        if result.success:
            strategy_a = result.x[:m]
            value = result.x[-1]
            return strategy_a, value
        else:
            raise ValueError(f"线性规划求解失败: {result.message}")

    @staticmethod
    def find_pure_nash(payoff_a: np.ndarray,
                       payoff_b: np.ndarray) -> list:
        """
        通过枚举查找纯策略纳什均衡。

        对每个策略组合(i,j)，检查是否没有任何玩家可以通过
        单方面改变策略来获得更高收益。
        """
        m, n = payoff_a.shape
        equilibria = []

        for i in range(m):
            for j in range(n):
                is_nash = True

                # 检查玩家A是否可以通过改变行来提高收益
                current_payoff_a = payoff_a[i][j]
                for i2 in range(m):
                    if payoff_a[i2][j] > current_payoff_a:
                        is_nash = False
                        break

                if not is_nash:
                    continue

                # 检查玩家B是否可以通过改变列来提高收益
                current_payoff_b = payoff_b[i][j]
                for j2 in range(n):
                    if payoff_b[i][j2] > current_payoff_b:
                        is_nash = False
                        break

                if is_nash:
                    equilibria.append((i, j, payoff_a[i][j],
                                      payoff_b[i][j]))

        return equilibria


# ============ 示例：囚徒困境 ============
# 收益矩阵（支付年数，负数表示惩罚）
# 玩家A的收益（行玩家）
payoff_a = np.array([
    [-1, -3],   # A: 合作时
    [0,  -2]    # A: 背叛时
])
# 玩家B的收益（列玩家）
payoff_b = np.array([
    [-1,  0],   # B: 合作时
    [-3, -2]    # B: 背叛时
])

print("=== 囚徒困境 ===")
equilibria = NashEquilibriumSolver.find_pure_nash(payoff_a, payoff_b)
print(f"纯策略纳什均衡: {equilibria}")
# 输出: [(1, 1, -2, -2)] — (背叛, 背叛) 是唯一的纳什均衡
# 注意：虽然(合作,合作)对双方整体更好，但它不是纳什均衡
# 因为每一方都有动机单方面背叛
```

### 纳什均衡的意义与局限

**意义**：
1. **预测稳定行为**：在理性假设下，纳什均衡预测了博弈的稳定结果
2. **理解困境**：揭示了个体理性与集体理性的冲突（如囚徒困境）
3. **机制设计**：通过设计收益结构来引导Agent达到理想均衡

**局限**：
1. **存在性**：纯策略纳什均衡不一定存在
2. **唯一性**：可能存在多个纳什均衡，选择哪一个不确定
3. **理性假设**：需要玩家完全理性且了解博弈结构，现实中不一定成立
4. **静态性**：均衡分析是静态的，不考虑学习和动态调整过程

## 囚徒困境在多Agent系统中的体现

### 经典囚徒困境回顾

两个罪犯被分开审讯。如果都保持沉默（合作），各判1年。如果一人坦白（背叛）而另一人沉默，坦白者释放（0年），沉默者重判（3年）。如果都坦白，各判2年。

```
        囚犯B
        沉默    坦白
       ┌───────┬───────┐
  沉默 │ -1,-1 │ -3,0  │
囚犯A  ├───────┼───────┤
  坦白 │ 0,-3  │ -2,-2 │
       └───────┴───────┘
```

### 在多Agent系统中的表现

囚徒困境并非仅存在于审讯室，它在多Agent系统中广泛存在：

#### 1. 资源竞争

多个Agent竞争有限的API调用配额（如每分钟速率限制）：

```python
class APIResourceDilemma:
    """
    API速率限制下的囚徒困境。

    两个Agent共享每分钟100次API调用限额。
    如果都节制使用（合作）→ 各50次，系统稳定
    如果一个疯狂调用（背叛）→ 自己80次，对方20次（还被限流）
    如果都疯狂调用（背叛）→ 都被限流，实际效率都低
    """
    def __init__(self, rate_limit: int = 100):
        self.rate_limit = rate_limit
        self.usage = {"agent_a": 0, "agent_b": 0}

    def get_payoff(self, strategy_a: str,
                   strategy_b: str) -> tuple:
        """
        moderate = 节制（合作）
        aggressive = 激进（背叛）
        """
        usage_a = 50 if strategy_a == "moderate" else 100
        usage_b = 50 if strategy_b == "moderate" else 100

        # 超过限额后，超出的请求被拒绝
        if usage_a + usage_b > self.rate_limit:
            excess = usage_a + usage_b - self.rate_limit
            # 按比例削减
            reduction_a = excess * usage_a / (usage_a + usage_b)
            reduction_b = excess * usage_b / (usage_a + usage_b)
            effective_a = usage_a - reduction_a
            effective_b = usage_b - reduction_b
        else:
            effective_a = usage_a
            effective_b = usage_b

        # 收益 = 成功调用数 - 被拒绝调用带来的惩罚
        payoff_a = effective_a - (usage_a - effective_a) * 0.5
        payoff_b = effective_b - (usage_b - effective_b) * 0.5

        return (payoff_a, payoff_b)


# 模拟
dilemma = APIResourceDilemma(rate_limit=100)
print("(节制, 节制):", dilemma.get_payoff("moderate", "moderate"))
# → (50, 50)  双方都节制，各得50次调用
print("(节制, 激进):", dilemma.get_payoff("moderate", "aggressive"))
# → (16.67, 83.33)  节制方吃亏，激进方占便宜
print("(激进, 激进):", dilemma.get_payoff("aggressive", "aggressive"))
# → (25, 25)  双方都激进，实际效率只有25次
```

#### 2. 信息共享困境

Agent之间是否应该共享有价值的信息？

```python
class InformationSharingDilemma:
    """
    信息共享困境。

    如果都共享（合作）→ 所有Agent的知识都增长，集体进步
    如果一方隐藏（背叛）→ 隐藏方获得对方信息外加自身优势
    如果都隐藏（背叛）→ 各自为政，重复劳动
    """

    def __init__(self):
        self.shared_knowledge = {}  # 共享知识库
        self.private_knowledge = {"agent_a": 10, "agent_b": 10}

    def simulate_round(self, share_a: bool,
                       share_b: bool) -> tuple:
        knowledge_a = self.private_knowledge["agent_a"]
        knowledge_b = self.private_knowledge["agent_b"]

        # 共享的知识进入公共池
        if share_a:
            self.shared_knowledge["a"] = knowledge_a
        if share_b:
            self.shared_knowledge["b"] = knowledge_b

        # 各Agent可以访问公共池
        effective_a = knowledge_a
        effective_b = knowledge_b

        if share_a and share_b:
            effective_a += knowledge_b  # 互相学习，知识翻倍
            effective_b += knowledge_a
        elif share_a and not share_b:
            effective_b += knowledge_a  # B搭了便车
        elif not share_a and share_b:
            effective_a += knowledge_b  # A搭了便车
        # 都不共享 → 各自为政

        # 但共享者自身知识也在增长（通过整理/输出而学习）
        if share_a:
            effective_a += 2  # 教是最好的学
        if share_b:
            effective_b += 2

        return (effective_a, effective_b)
```

#### 3. 逃避协作责任

在Manager-Worker模式中，Worker可能"偷懒"——让其他Worker多干活：

```python
class FreeRiderProblem:
    """
    搭便车问题：Worker可能倾向于让其他Worker承担更多工作。

    如果都积极（合作）→ 工作量均分，任务快速完成
    如果一人积极一人偷懒 → 积极者承担更多（不公平）
    如果都偷懒 → 任务无法完成（最差结果）
    """

    def __init__(self, total_workload: int = 100):
        self.total = total_workload

    def get_outcome(self, effort_a: str,
                    effort_b: str) -> dict:
        effort_levels = {
            "high": 0.6,    # 承担60%工作量
            "medium": 0.5,  # 承担50%
            "low": 0.2      # 承担20%（剩余别人补或失败）
        }

        capacity_a = effort_levels[effort_a]
        capacity_b = effort_levels[effort_b]

        actual_a = self.total * capacity_a
        actual_b = self.total * capacity_b

        completed = actual_a + actual_b
        completion_rate = min(1.0, completed / self.total)

        # 个人收益 = 任务完成度 * 50 - 个人付出 * 30
        payoff_a = completion_rate * 50 - capacity_a * 30
        payoff_b = completion_rate * 50 - capacity_b * 30

        return {
            "completion_rate": completion_rate,
            "payoff_a": payoff_a,
            "payoff_b": payoff_b,
            "is_nash_equilibrium": (
                effort_a == "low" and effort_b == "low"
            )  # 都偷懒是唯一的纳什均衡！
        }
```

## 合作博弈 vs 非合作博弈

### 核心区别

| 维度 | 合作博弈（Cooperative Game） | 非合作博弈（Non-Cooperative Game） |
|------|---------------------------|----------------------------------|
| **分析单元** | 联盟（Coalition） | 个体玩家 |
| **核心问题** | 如何分配合作产生的剩余价值？ | 在互动中如何选择最优策略？ |
| **协议约束** | 有约束力的协议 | 无约束力的协议（或协议不可执行） |
| **典型概念** | Shapley值、核（Core）、联盟结构 | 纳什均衡、占优策略、子博弈完美均衡 |
| **多Agent应用** | 任务分配、利润分成、联邦学习 | 竞争定价、资源竞标、对抗博弈 |

### 合作博弈：Shapley值

**Shapley值**回答了合作博弈中的公平分配问题：如果在联盟中每个成员对总收益都有贡献，那么如何公平地分配总收益？

```python
from itertools import combinations
from math import factorial


def shapley_value(players: list[str],
                  coalition_value: callable) -> dict:
    """
    计算每个玩家的Shapley值。

    Shapley值的直观含义：
    玩家i的Shapley值 = 玩家i对所有可能联盟的"边际贡献"的加权平均

    公式：
    φ_i(v) = Σ_{S⊆N\{i}} [|S|! * (|N|-|S|-1)! / |N|!] * [v(S∪{i}) - v(S)]
    """
    n = len(players)
    shapley = {p: 0.0 for p in players}

    for player in players:
        others = [p for p in players if p != player]

        # 遍历所有不包含该玩家的子集
        for r in range(len(others) + 1):
            for subset in combinations(others, r):
                S = list(subset)
                S_with = S + [player]

                # 边际贡献
                marginal = coalition_value(S_with) - coalition_value(S)

                # 权重 = |S|! * (n-|S|-1)! / n!
                weight = (factorial(len(S)) *
                         factorial(n - len(S) - 1) /
                         factorial(n))

                shapley[player] += weight * marginal

    return shapley


# ============ 示例：多Agent任务分配 ============
# 三个Agent组合能完成的任务数
def task_coalition_value(agents: list[str]) -> float:
    """定义任意Agent组合能完成的任务数量"""
    values = {
        frozenset([]): 0,
        frozenset(["researcher"]): 5,     # 研究员单独：5个任务
        frozenset(["analyst"]): 3,         # 分析师单独：3个任务
        frozenset(["writer"]): 4,          # 写手单独：4个任务
        frozenset(["researcher", "analyst"]): 15,   # 研究+分析：15
        frozenset(["researcher", "writer"]): 12,    # 研究+写作：12
        frozenset(["analyst", "writer"]): 10,       # 分析+写作：10
        frozenset(["researcher", "analyst", "writer"]): 25,  # 全员：25
    }
    return values.get(frozenset(sorted(agents)), 0)


players = ["researcher", "analyst", "writer"]
shapley = shapley_value(players, task_coalition_value)

print("=== Shapley值分配 ===")
for player, value in shapley.items():
    print(f"  {player}: {value:.1f} 个任务份额")
# 这给出了一个"公平"的任务分配方案
```

### 在多Agent系统中的应用场景

| 场景 | 博弈类型 | 核心问题 |
|------|---------|---------|
| 联邦学习 | 合作博弈 | 如何分配模型改进带来的收益？ |
| 竞价广告 | 非合作博弈 | 如何出价才能获得最优展示位置？ |

### 重复博弈与声誉机制

如果Agent反复互动（而非一次性博弈），策略会如何改变？

```python
class RepeatedPrisonersDilemma:
    """
    重复囚徒困境：Agent反复互动，声誉和长期关系变得重要。

    著名策略分析：
    """

    @staticmethod
    def tit_for_tat(my_history: list, opponent_history: list) -> str:
        """
        以牙还牙 (Tit-for-Tat)：

        第一轮合作，之后模仿对手上一轮的行为。
        这是Axelrod锦标赛的冠军策略！

        特点：
        - 善良（Nice）：从不先背叛
        - 报复（Retaliating）：对背叛立即报复
        - 宽容（Forgiving）：对方恢复合作就立即恢复
        - 清晰（Clear）：策略简单透明
        """
        if not opponent_history:
            return "cooperate"
        return opponent_history[-1]

    @staticmethod
    def grim_trigger(my_history: list,
                     opponent_history: list) -> str:
        """
        冷酷触发 (Grim Trigger)：

        一旦对方背叛一次，永久转为背叛。
        这是最严厉的惩罚策略，不给第二次机会。
        优势：威慑力极强
        劣势：一次误会就导致永久破裂
        """
        if "defect" in opponent_history:
            return "defect"
        return "cooperate"

    @staticmethod
    def forgiving_tit_for_tat(my_history: list,
                               opponent_history: list,
                               forgiveness_rate: float = 0.1
                               ) -> str:
        """
        宽容版以牙还牙 (Forgiving Tit-for-Tat)：

        和Tit-for-Tat一样，但有概率原谅对方的背叛。
        这在有噪声（可能误解意图）的环境中更有效。
        """
        if not opponent_history:
            return "cooperate"
        if opponent_history[-1] == "defect":
            if np.random.random() < forgiveness_rate:
                return "cooperate"  # 原谅
        return opponent_history[-1]

    @staticmethod
    def pavlov(my_history: list, opponent_history: list) -> str:
        """
        巴甫洛夫策略 (Win-Stay, Lose-Shift)：

        如果上轮收益好 → 保持策略
        如果上轮收益差 → 切换策略

        这模仿了"条件反射"学习。
        """
        if not my_history:
            return "cooperate"

        # 简化：如果最近一轮双方合作就继续合作
        if (opponent_history[-1] == "cooperate" and
                my_history[-1] == "cooperate"):
            return "cooperate"
        else:
            return "defect"


# ============ 策略锦标赛模拟 ============
def run_tournament(strategies: dict,
                   payoff_matrix: dict,
                   rounds: int = 100) -> dict:
    """让不同策略在重复囚徒困境中对战"""
    scores = {name: 0 for name in strategies}

    for name_a, strat_a in strategies.items():
        for name_b, strat_b in strategies.items():
            if name_a >= name_b:  # 避免重复对战
                continue

            history_a = []
            history_b = []

            for _ in range(rounds):
                move_a = strat_a(history_a, history_b)
                move_b = strat_b(history_b, history_a)

                payoff_a, payoff_b = payoff_matrix[(move_a, move_b)]

                scores[name_a] += payoff_a
                scores[name_b] += payoff_b

                history_a.append(move_a)
                history_b.append(move_b)

    return scores


# 收益矩阵
payoff = {
    ("cooperate", "cooperate"): (3, 3),
    ("cooperate", "defect"):    (0, 5),
    ("defect", "cooperate"):    (5, 0),
    ("defect", "defect"):       (1, 1),
}

strategies = {
    "Always Cooperate": lambda h1, h2: "cooperate",
    "Always Defect": lambda h1, h2: "defect",
    "Tit-for-Tat": RepeatedPrisonersDilemma.tit_for_tat,
    "Random": lambda h1, h2: np.random.choice(
        ["cooperate", "defect"]
    ),
}

results = run_tournament(strategies, payoff, rounds=200)
print("=== 策略锦标赛结果 ===")
for name, score in sorted(results.items(),
                          key=lambda x: x[1], reverse=True):
    print(f"  {name}: {score}")
```

### 贝叶斯博弈与不完全信息

在多Agent系统中，Agent通常不知道其他Agent的"类型"（偏好、能力、目标）。贝叶斯博弈处理这种不确定性。

```python
class BayesianAuction:
    """
    贝叶斯拍卖博弈。

    多个Agent竞拍一个资源（如GPU时间），每个Agent对资源有
    私有估值（private value），但不知道其他Agent的估值。
    """

    def __init__(self, num_bidders: int,
                 value_distribution: callable):
        """
        value_distribution: 估值分布函数
        每个Agent的估值独立从该分布中抽取
        """
        self.num_bidders = num_bidders
        self.value_dist = value_distribution

    def second_price_auction(self) -> dict:
        """
        第二价格拍卖（Vickrey拍卖）。

        理论结果：诚实出价（bid = true value）是一个占优策略！
        无论别人怎么出价，按真实估值出价都是最优的。

        这是博弈论在机制设计中最漂亮的结果之一。
        """
        # 生成每个Agent的真实估值
        true_values = {
            f"agent_{i}": self.value_dist()
            for i in range(self.num_bidders)
        }

        # 每个Agent出价（假设都诚实出价）
        bids = dict(true_values)

        # 确定赢家（出价最高者）
        sorted_bids = sorted(bids.items(),
                            key=lambda x: x[1], reverse=True)

        winner = sorted_bids[0][0]
        second_highest = sorted_bids[1][1]  # 支付第二高出价

        # 赢家收益 = 真实价值 - 实际支付
        winner_surplus = true_values[winner] - second_highest

        return {
            "winner": winner,
            "winning_bid": sorted_bids[0][1],
            "price_paid": second_highest,
            "true_value": true_values[winner],
            "surplus": winner_surplus,
            "all_values": true_values
        }

    def first_price_auction_strategy(self,
                                      true_value: float,
                                      num_bidders: int) -> float:
        """
        第一价格拍卖的最优出价策略（贝叶斯纳什均衡）。

        假设估值服从[0,1]均匀分布：
        最优出价 = true_value * (n-1) / n

        注意：在第一价格拍卖中，诚实出价不再是占优策略！
        Agent需要策略性地"压低"出价。
        """
        return true_value * (num_bidders - 1) / num_bidders


# ============ 拍卖模拟 ============
np.random.seed(42)

auction = BayesianAuction(
    num_bidders=5,
    value_distribution=lambda: np.random.uniform(50, 200)
)

result = auction.second_price_auction()
print("=== 第二价格拍卖结果 ===")
print(f"赢家: {result['winner']}")
print(f"真实估值: {result['true_value']:.1f}")
print(f"实际支付: {result['price_paid']:.1f}")
print(f"赢家剩余: {result['surplus']:.1f}")

# 验证诚实出价的占优性
value = 100
print(f"\n第一价格拍卖最优出价: "
      f"{auction.first_price_auction_strategy(value, 5):.1f}")
# 最优出价 = 100 * 4/5 = 80
```

```python
class CombinatorialAuctionScheduler:
    """
    组合拍卖调度器：将多种资源分配给多个Agent。

    场景：多个Agent需要组合资源（如GPU+内存+存储）来完成任务，
    每个Agent对不同资源组合有不同估值。
    """

    def __init__(self, resources: dict):
        """
        resources: {"gpu": 4, "memory_gb": 64, "storage_gb": 500}
        """
        self.resources = resources
        self.allocations = {}  # agent_id -> allocated bundle

    def allocate_VCG(self,
                      agents: dict[str, dict]) -> dict:
        """
        VCG（Vickrey-Clarke-Groves）机制分配资源。

        VCG是第二价格拍卖的泛化：
        1. 选择能最大化社会总福利的分配方案
        2. 每个Agent支付它对其他Agent造成的"外部性"

        VCG的优雅之处：
        - 诚实报告估值是占优策略
        - 结果达到帕累托最优
        """
        # 简化实现：贪心分配（最优分配是NP-hard）
        remaining = dict(self.resources)
        allocation = {}
        payments = {}

        # 按单位资源估值排序
        ranked_agents = sorted(
            agents.items(),
            key=lambda x: x[1]["value"] / sum(x[1]["demand"].values()),
            reverse=True
        )

        for agent_id, info in ranked_agents:
            demand = info["demand"]
            # 检查资源是否足够
            if all(remaining.get(r, 0) >= demand.get(r, 0)
                   for r in demand):
                # 分配资源
                allocation[agent_id] = dict(demand)
                for r, amount in demand.items():
                    remaining[r] -= amount

                # 计算VCG支付 = 该Agent对系统中其他Agent造成的损失
                externality = self._compute_externality(
                    agent_id, agents, allocation, demand
                )
                payments[agent_id] = externality
            else:
                allocation[agent_id] = None

        return {
            "allocation": allocation,
            "payments": payments,
            "remaining_resources": remaining
        }

    def _compute_externality(self, excluded_agent: str,
                              agents: dict,
                              current_alloc: dict,
                              excluded_demand: dict) -> float:
        """
        计算外部性：排除该Agent后，其他Agent能获得的最大价值变化。

        这是VCG支付的核心。
        """
        # 简化计算
        other_values = sum(
            info["value"]
            for aid, info in agents.items()
            if aid != excluded_agent and aid in current_alloc
        )
        return 0  # 简化为0，实际需要求解两个优化问题
```

### 协商与讨价还价

```python
class BargainingGame:
    """
    讨价还价博弈：两个Agent分割一笔"蛋糕"。

    纳什讨价还价解（Nash Bargaining Solution）：
    最大化 (u1 - d1)(u2 - d2)
    其中di是"破裂点"——谈判失败时各方能获得的保留效用。
    """

    def __init__(self, total_pie: float = 100,
                 reservation_a: float = 10,
                 reservation_b: float = 20):
        self.total = total_pie
        self.reservation_a = reservation_a
        self.reservation_b = reservation_b

    def nash_bargaining_solution(self) -> tuple:
        """
        计算纳什讨价还价解。

        最大化 (x_a - r_a)(x_b - r_b)
        s.t. x_a + x_b = total_pie
        """
        surplus = self.total - self.reservation_a - self.reservation_b
        x_a = self.reservation_a + surplus / 2
        x_b = self.reservation_b + surplus / 2
        return (x_a, x_b)

    def rubinstein_bargaining(self,
                               discount_a: float = 0.9,
                               discount_b: float = 0.8,
                               horizon: int = 100) -> tuple:
        """
        Rubinstein交替出价博弈。

        两个Agent轮流提出分配方案，对方可以接受或拒绝。
        拒绝后由拒绝方提出新方案，但蛋糕随时间缩小(discount)。

        无限期界的子博弈完美均衡：
        x_a = (1 - δ_b) / (1 - δ_a * δ_b)
        """
        if horizon == 0:
            return (0, 0)

        # 逆推求解
        # 简化：直接使用无限期界公式
        x_a_share = (1 - discount_b) / (1 - discount_a * discount_b)
        return (self.total * x_a_share,
                self.total * (1 - x_a_share))


bg = BargainingGame(total_pie=100, reservation_a=10, reservation_b=20)

nbs = bg.nash_bargaining_solution()
print(f"纳什讨价还价解: A={nbs[0]:.1f}, B={nbs[1]:.1f}")

rbs = bg.rubinstein_bargaining()
print(f"Rubinstein解: A={rbs[0]:.1f}, B={rbs[1]:.1f}")
# A更有耐心(discount高)，获得更大份额
```

## 博弈论对多Agent系统设计的启示

### 设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **机制设计** | 设计规则让Agent的自利行为产生理想结果 | VCG拍卖、声誉系统 |
| **激励兼容** | 确保诚实报告是Agent的最优策略 | 第二价格拍卖 |
| **防策略操纵** | 机制应抵抗策略性操纵 | 避免先到先得等易操纵规则 |
| **稳健性** | 考虑Agent可能不理性、信息不完全 | 重复博弈、声誉机制 |
| **公平性** | 各Agent感觉被公平对待 | Shapley值分配 |

## 小结

博弈论为多智能体系统提供了一套强大的分析和设计工具。从最基础的纳什均衡到复杂的机制设计，博弈论帮助我们理解和预测Agent在互动中的行为——这是设计稳健、高效的多Agent系统不可或缺的能力。

**核心要点**：
1. **纳什均衡**是关于"稳定状态"的预测——没有人有动机单方面改变
2. **囚徒困境**揭示了个人理性与集体理性的冲突，在多Agent资源竞争中普遍存在
3. **合作博弈**（如Shapley值）解决公平分配问题，**非合作博弈**解决策略选择问题
4. **重复博弈**中，声誉和长期关系改变了短期博弈的均衡
5. **机制设计**（如VCG拍卖）通过巧妙的规则设计引导Agent的自利行为产生理想结果

**实践建议**：
- 设计多Agent奖惩机制时，确保合作比背叛更有利可图
- 对于重复互动的Agent，引入声誉系统和历史记忆
- 对于资源分配，考虑VCG或Shapley值等机制确保公平和效率
- 始终为Agent的不理性行为和噪声留有余地——理论上的均衡在现实中不一定实现

博弈论的美妙之处在于，它让我们能够在Agent开始"玩"之前，就理解和预测游戏的走向。掌握了博弈论，你就掌握了多Agent系统的"游戏规则设计"能力。
