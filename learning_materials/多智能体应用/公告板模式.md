# 公告板模式（共享信息空间）

## 引言

在现实世界中，公告板（Bulletin Board）是一种历史悠久的协作工具——人们把信息贴在公告板上，其他人需要时就会去查看。大学里的选课公告、公司的招聘信息、社区的买卖信息，都是通过公告板这一"共享信息空间"来组织的。

在AI多智能体系统中，**公告板模式（Blackboard Pattern）** 借鉴了同样的思想：所有Agent共享一个中心化的信息空间，Agent可以在这个空间中发布信息、读取他人的信息，以及响应那些与自己能力匹配的"求助帖"。这种模式实现了Agent之间的松散耦合——Agent不需要知道彼此的存在，它们只需要知道公告板在哪里。

## 公告板架构

### 核心组成

```
┌──────────────────────────────────────────────────────────┐
│                    公告板 (Blackboard)                    │
│                                                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐      │
│  │  知识条目   │  │  待办任务   │  │  中间结果   │      │
│  │ (Knowledge) │  │  (Tasks)    │  │ (Partial)   │      │
│  └─────────────┘  └─────────────┘  └─────────────┘      │
│                                                          │
│  ┌─────────────────────────────────────────────┐        │
│  │              调度器 (Scheduler)              │        │
│  │   • 匹配任务与Agent能力                      │        │
│  │   • 管理任务优先级                           │        │
│  │   • 触发条件检测                             │        │
│  └─────────────────────────────────────────────┘        │
└──────────────────────────────────────────────────────────┘
         ▲              ▲              ▲
         │              │              │
    ┌────┴────┐    ┌────┴────┐    ┌────┴────┐
    │ Agent A │    │ Agent B │    │ Agent C │
    │ 搜索能力│    │ 分析能力│    │ 写作能力│
    └─────────┘    └─────────┘    └─────────┘
```

公告板模式由三部分组成：

1. **公告板（Blackboard）**：共享的、结构化的信息存储空间
2. **知识源（Knowledge Sources）**：即各个Agent，它们读/写公告板
3. **调度器（Scheduler）**：决定哪个Agent在什么时候被激活

### 与传统"消息传递"的区别

| 维度 | 消息传递模式 | 公告板模式 |
|------|------------|-----------|
| 通信方式 | 点对点（Agent → Agent） | 发布-发现（Agent → Board → Agent） |
| Agent感知 | Agent需要知道接收方 | Agent不需要知道其他Agent |
| 信息持久性 | 消息消费后消失 | 信息持续存在，可多次读取 |
| 耦合度 | 中-高 | 极低 |
| 发现机制 | 预先配置或注册 | 通过能力匹配自动发现 |
| 历史追溯 | 困难 | 容易（全部记录在板上） |

## Agent如何发布和读取信息

### 信息发布

Agent向公告板贡献信息，类似于在论坛发帖。发布的信息需要携带结构化的元数据，方便其他Agent检索和理解。

```python
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Any, Optional
import uuid


class EntryType(Enum):
    TASK_REQUEST = "task_request"       # 任务请求："谁能帮我做X？"
    TASK_RESULT = "task_result"         # 任务结果："X已完成，结果如下"
    KNOWLEDGE = "knowledge"             # 知识贡献："关于Y的最新发现"
    HYPOTHESIS = "hypothesis"           # 假设/推测："我推测Z可能..."
    ANNOTATION = "annotation"           # 批注/评论："对条目W的补充说明"
    ALERT = "alert"                     # 告警："发现问题，需要关注"


@dataclass
class BlackboardEntry:
    """公告板条目的标准格式"""
    entry_id: str = field(default_factory=lambda: str(uuid.uuid4()))
    entry_type: EntryType
    title: str
    content: Any
    author: str                        # 发布者Agent的ID
    tags: list[str] = field(default_factory=list)    # 标签，便于搜索
    required_skills: list[str] = field(default_factory=list)  # 需要的技能
    priority: int = 5                  # 1-10，10最高
    status: str = "open"               # open / claimed / resolved
    claimed_by: Optional[str] = None   # 谁认领了这个任务
    parent_id: Optional[str] = None    # 关联的父条目
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)
    metadata: dict = field(default_factory=dict)

    def to_dict(self) -> dict:
        return {
            "entry_id": self.entry_id,
            "entry_type": self.entry_type.value,
            "title": self.title,
            "content": self.content,
            "author": self.author,
            "tags": self.tags,
            "required_skills": self.required_skills,
            "priority": self.priority,
            "status": self.status,
            "claimed_by": self.claimed_by,
            "parent_id": self.parent_id,
            "created_at": self.created_at.isoformat(),
            "updated_at": self.updated_at.isoformat(),
            "metadata": self.metadata
        }
```

### Agent发布信息

```python
class BlackboardAgent:
    """接入公告板的Agent基类"""
    def __init__(self, agent_id: str, skills: list[str],
                 blackboard: 'Blackboard'):
        self.agent_id = agent_id
        self.skills = skills
        self.blackboard = blackboard

        # 向调度器注册自己的能力
        blackboard.register_agent(self)

    def post_task_request(self, title: str, description: str,
                          required_skills: list[str],
                          parent_id: str = None,
                          priority: int = 5) -> str:
        """
        发布一个任务请求："我需要帮助"

        这相当于在公告板上贴了一张"求助帖"。
        """
        entry = BlackboardEntry(
            entry_type=EntryType.TASK_REQUEST,
            title=title,
            content=description,
            author=self.agent_id,
            required_skills=required_skills,
            priority=priority,
            parent_id=parent_id
        )
        entry_id = self.blackboard.post(entry)
        print(f"[{self.agent_id}] 发布任务请求: {title} (ID: {entry_id})")
        return entry_id

    def post_result(self, title: str, result: Any,
                    task_entry_id: str) -> str:
        """
        发布任务结果："这个任务我完成了"

        将结果贴回公告板，关联原始任务。
        """
        entry = BlackboardEntry(
            entry_type=EntryType.TASK_RESULT,
            title=title,
            content=result,
            author=self.agent_id,
            parent_id=task_entry_id
        )
        entry_id = self.blackboard.post(entry)

        # 标记原始任务为已完成
        self.blackboard.update_status(task_entry_id, "resolved")
        print(f"[{self.agent_id}] 发布结果: {title}")
        return entry_id

    def post_knowledge(self, title: str, knowledge: str,
                       tags: list[str]) -> str:
        """
        贡献知识："我发现了这个，分享给大家"

        将知识写入公告板的共享知识区。
        """
        entry = BlackboardEntry(
            entry_type=EntryType.KNOWLEDGE,
            title=title,
            content=knowledge,
            author=self.agent_id,
            tags=tags
        )
        return self.blackboard.post(entry)
```

### Agent读取信息

```python
class BlackboardAgent:
    # ... 前面的代码 ...

    def find_available_tasks(self, limit: int = 10) -> list[BlackboardEntry]:
        """
        查找我可以处理的待办任务

        Agent定期查看公告板，寻找匹配自己技能的任务。
        """
        all_tasks = self.blackboard.query(
            entry_type=EntryType.TASK_REQUEST,
            status="open",
            limit=limit
        )
        # 过滤出自己技能匹配的任务
        matching = [
            t for t in all_tasks
            if any(skill in t.required_skills for skill in self.skills)
            or not t.required_skills  # 没有指定技能要求的，任何人都可以
        ]
        matching.sort(key=lambda t: t.priority, reverse=True)
        return matching

    def claim_task(self, entry_id: str) -> bool:
        """
        认领一个任务："这个让我来做"

        使用CAS（Compare-And-Swap）确保任务只被一个Agent认领。
        """
        entry = self.blackboard.get_entry(entry_id)
        if entry and entry.status == "open":
            # 原子操作：先到先得
            claimed = self.blackboard.claim(entry_id, self.agent_id)
            if claimed:
                print(f"[{self.agent_id}] 认领任务: {entry.title}")
            return claimed
        return False

    def read_knowledge(self, tags: list[str] = None,
                       keyword: str = None) -> list[BlackboardEntry]:
        """
        搜索公告板上的知识："有没有关于X的信息？"
        """
        return self.blackboard.search(
            entry_type=EntryType.KNOWLEDGE,
            tags=tags,
            keyword=keyword
        )

    def read_results(self, parent_task_id: str) -> list[BlackboardEntry]:
        """
        查看某个任务的所有相关结果
        """
        return self.blackboard.get_children(parent_task_id)
```

## 公告板的内部实现

### 核心数据结构

```python
from collections import defaultdict
import threading
import json


class Blackboard:
    """公告板的内部实现"""

    def __init__(self):
        self._entries: dict[str, BlackboardEntry] = {}
        self._agents: dict[str, BlackboardAgent] = {}
        self._tags_index: dict[str, set[str]] = defaultdict(set)
        self._skill_index: dict[str, set[str]] = defaultdict(set)
        self._status_index: dict[str, set[str]] = defaultdict(set)

        self._lock = threading.RLock()  # 线程安全
        self._subscribers: dict[str, list[callable]] = defaultdict(list)

    def register_agent(self, agent: 'BlackboardAgent'):
        """注册Agent及其技能，构建技能索引"""
        with self._lock:
            self._agents[agent.agent_id] = agent
            for skill in agent.skills:
                self._skill_index[skill].add(agent.agent_id)
        print(f"[Blackboard] Agent {agent.agent_id} 已注册, "
              f"技能: {agent.skills}")

    def post(self, entry: BlackboardEntry) -> str:
        """发布条目到公告板，并通知相关订阅者"""
        with self._lock:
            self._entries[entry.entry_id] = entry

            # 更新索引
            for tag in entry.tags:
                self._tags_index[tag].add(entry.entry_id)
            for skill in entry.required_skills:
                self._skill_index[skill].add(entry.entry_id)
            self._status_index[entry.status].add(entry.entry_id)

        # 通知能力匹配的Agent（异步）
        self._notify_interested_agents(entry)
        return entry.entry_id

    def get_entry(self, entry_id: str) -> Optional[BlackboardEntry]:
        """读取指定条目"""
        return self._entries.get(entry_id)

    def query(self, entry_type: EntryType = None,
              status: str = None,
              tags: list[str] = None,
              skills: list[str] = None,
              limit: int = 50) -> list[BlackboardEntry]:
        """查询公告板条目"""
        with self._lock:
            results = list(self._entries.values())

            if entry_type:
                results = [e for e in results
                          if e.entry_type == entry_type]
            if status:
                results = [e for e in results
                          if e.status == status]
            if tags:
                results = [e for e in results
                          if any(t in e.tags for t in tags)]
            if skills:
                results = [e for e in results
                          if any(s in e.required_skills
                                for s in skills)]

            results.sort(key=lambda e: (e.priority, e.created_at),
                        reverse=True)
            return results[:limit]

    def search(self, entry_type: EntryType = None,
               tags: list[str] = None,
               keyword: str = None,
               limit: int = 50) -> list[BlackboardEntry]:
        """全文搜索公告板"""
        with self._lock:
            results = list(self._entries.values())

            if entry_type:
                results = [e for e in results
                          if e.entry_type == entry_type]
            if tags:
                results = [e for e in results
                          if any(t in e.tags for t in tags)]
            if keyword:
                keyword_lower = keyword.lower()
                results = [e for e in results
                          if keyword_lower in e.title.lower()
                          or keyword_lower in str(e.content).lower()]

            return results[:limit]

    def claim(self, entry_id: str, agent_id: str) -> bool:
        """
        原子认领任务（CAS操作）
        防止多个Agent同时认领同一任务。
        """
        with self._lock:
            entry = self._entries.get(entry_id)
            if entry and entry.status == "open":
                entry.status = "claimed"
                entry.claimed_by = agent_id
                entry.updated_at = datetime.now()

                # 更新状态索引
                self._status_index["open"].discard(entry_id)
                self._status_index["claimed"].add(entry_id)
                return True
            return False

    def update_status(self, entry_id: str, new_status: str):
        """更新条目状态"""
        with self._lock:
            entry = self._entries.get(entry_id)
            if entry:
                old_status = entry.status
                entry.status = new_status
                entry.updated_at = datetime.now()
                self._status_index[old_status].discard(entry_id)
                self._status_index[new_status].add(entry_id)

    def get_children(self, parent_id: str) -> list[BlackboardEntry]:
        """获取某条目的所有子条目（如任务的所有评论/结果）"""
        return [e for e in self._entries.values()
                if e.parent_id == parent_id]

    def subscribe(self, agent_id: str, callback: callable):
        """
        订阅通知：当有匹配Agent技能的新任务发布时，通知Agent。
        """
        self._subscribers[agent_id].append(callback)

    def _notify_interested_agents(self, entry: BlackboardEntry):
        """通知可能对新条目感兴趣的Agent"""
        interested_agents = set()

        # 找出技能匹配的Agent
        for skill in entry.required_skills:
            interested_agents.update(self._skill_index.get(skill, set()))

        # 如果没有指定技能要求，通知所有Agent
        if not entry.required_skills:
            interested_agents = set(self._agents.keys())

        # 不通知作者自己
        interested_agents.discard(entry.author)

        # 异步通知
        for agent_id in interested_agents:
            for callback in self._subscribers.get(agent_id, []):
                try:
                    callback(entry)
                except Exception as e:
                    print(f"通知Agent {agent_id} 失败: {e}")
```

### 调度器

```python
class Scheduler:
    """
    调度器：管理公告板上的任务生命周期。

    核心职责：
    1. 周期性检查公告板，驱动任务流转
    2. 匹配任务和能力
    3. 处理超时任务
    """

    def __init__(self, blackboard: Blackboard,
                 check_interval: float = 1.0):
        self.blackboard = blackboard
        self.check_interval = check_interval
        self.running = False

    async def start(self):
        """启动调度循环"""
        self.running = True
        print("[Scheduler] 调度器启动")

        while self.running:
            await self._tick()
            await asyncio.sleep(self.check_interval)

    async def _tick(self):
        """每一轮调度需要做的事情"""
        # 1. 检查超时的已认领任务
        await self._handle_timeouts()

        # 2. 检查是否有孤儿知识（无归属条目的知识）
        await self._classify_orphans()

        # 3. 检查是否可以触发依赖任务
        await self._check_dependencies()

    async def _handle_timeouts(self, timeout_seconds: int = 300):
        """
        处理超时任务：
        被认领但长时间未完成的任务，重新标记为 open。
        """
        now = datetime.now()
        claimed_tasks = self.blackboard.query(status="claimed")

        for task in claimed_tasks:
            elapsed = (now - task.updated_at).total_seconds()
            if elapsed > timeout_seconds:
                print(f"[Scheduler] 任务 {task.entry_id} 已超时 "
                      f"({elapsed:.0f}s)，重新开放")
                self.blackboard.update_status(task.entry_id, "open")
                task.claimed_by = None

    async def _classify_orphans(self):
        """处理无父条目的知识，尝试关联到现有任务"""
        orphan_knowledge = self.blackboard.query(
            entry_type=EntryType.KNOWLEDGE
        )
        orphan_knowledge = [k for k in orphan_knowledge
                           if k.parent_id is None]

        for knowledge in orphan_knowledge:
            # 尝试匹配相关任务
            related = self.blackboard.search(
                entry_type=EntryType.TASK_REQUEST,
                keyword=knowledge.title
            )
            if related:
                knowledge.parent_id = related[0].entry_id
                knowledge.updated_at = datetime.now()

    def stop(self):
        self.running = False
```

## JADE框架中的公告板实现

JADE（Java Agent DEvelopment Framework）是最经典的Agent开发框架之一，其中包含了公告板模式的参考实现。

### JADE的DF（Directory Facilitator）

DF是JADE中的"黄页服务"——Agent向DF注册自己的服务能力，其他Agent通过DF查找需要的服务。这本质上就是一种公告板模式。

```python
# 基于JADE思想的公告板服务注册（Python实现）

class DirectoryFacilitator:
    """
    受JADE DF启发的Agent目录服务。

    在JADE中，DF是Agent发现彼此能力的关键组件。
    每个Agent启动后向DF注册，其他Agent通过DF查找服务。
    """

    def __init__(self):
        self.services = {}  # service_type -> [ServiceDescription]

    def register(self, agent_id: str,
                 service_type: str,
                 service_name: str,
                 properties: dict = None) -> str:
        """
        Agent注册服务（类似API注册到服务注册中心）

        service_type: 服务类型，如 "data-analysis", "web-search"
        service_name: 服务实例名称
        properties: 额外属性（如所有权、成本等）
        """
        registration = ServiceDescription(
            agent_id=agent_id,
            service_type=service_type,
            service_name=service_name,
            properties=properties or {}
        )
        if service_type not in self.services:
            self.services[service_type] = []
        self.services[service_type].append(registration)
        print(f"[DF] {agent_id} 注册服务: {service_type}/{service_name}")
        return registration.service_name

    def search(self, service_type: str,
               constraints: dict = None) -> list[dict]:
        """
        搜索提供特定服务的Agent

        constraints: 过滤条件，如 {"max_cost": 0.01, "language": "zh"}
        """
        results = self.services.get(service_type, [])

        if constraints:
            results = [
                r for r in results
                if all(r.properties.get(k) == v
                      for k, v in constraints.items())
            ]

        return [r.to_dict() for r in results]

    def deregister(self, agent_id: str, service_name: str = None):
        """Agent注销服务"""
        for stype, services in self.services.items():
            self.services[stype] = [
                s for s in services
                if not (s.agent_id == agent_id
                       and (service_name is None
                           or s.service_name == service_name))
            ]
```

### JADE风格的Agent行为模型

```python
class JADEStyleAgent:
    """
    模仿JADE Agent行为模型的Agent实现。

    JADE Agent通过"行为（Behaviour）"来组织逻辑，
    每个行为是一个可调度的执行单元。
    """

    def __init__(self, agent_id: str, df: DirectoryFacilitator):
        self.agent_id = agent_id
        self.df = df
        self.behaviours = []
        self.blackboard = None

    def setup(self):
        """Agent初始化——在JADE中这是生命周期入口"""
        # 注册自己的服务
        self.df.register(
            agent_id=self.agent_id,
            service_type="document-analysis",
            service_name=f"{self.agent_id}-analysis-service"
        )

        # 添加查找任务的行为
        self.add_behaviour(self.TaskSeekingBehaviour(self))

        # 添加处理任务的行为
        self.add_behaviour(self.TaskProcessingBehaviour(self))

    def add_behaviour(self, behaviour):
        """添加行为到调度队列"""
        self.behaviours.append(behaviour)

    class TaskSeekingBehaviour:
        """定期查找公告板上可处理任务的行为"""
        def __init__(self, agent):
            self.agent = agent
            self.interval = 2.0  # 每2秒检查一次

        async def action(self):
            # 查找匹配的任务
            tasks = self.agent.blackboard.query(
                entry_type=EntryType.TASK_REQUEST,
                status="open"
            )
            for task in tasks:
                if self._can_handle(task):
                    self.agent.blackboard.claim(
                        task.entry_id, self.agent.agent_id
                    )

    class TaskProcessingBehaviour:
        """处理已认领任务的行为"""
        def __init__(self, agent):
            self.agent = agent

        async def action(self):
            claimed = self.agent.blackboard.query(
                status="claimed",
            )
            my_tasks = [t for t in claimed
                       if t.claimed_by == self.agent.agent_id]
            for task in my_tasks:
                result = await self._process(task)
                self.agent.post_result(
                    title=f"结果: {task.title}",
                    result=result,
                    task_entry_id=task.entry_id
                )
```

## 与消息队列的对比

公告板模式和消息队列是两种常见的异步协作机制，但适用场景有显著不同。

| 维度 | 公告板模式 | 消息队列 |
|------|-----------|---------|
| **信息可见性** | 所有Agent可见，公开透明 | 点对点或广播，有明确接收方 |
| **信息持久性** | 信息持续保存在板上 | 消息消费后通常删除 |
| **任务发现** | Agent主动查找（Pull） | 消息推送（Push） |
| **任务认领** | 先到先得（Requires CAS） | 由队列路由决定 |
| **知识积累** | 自然形成知识库 | 需要额外存储 |
| **通信粒度** | 更适合粗粒度任务 | 适合细粒度事件和指令 |
| **解耦程度** | 极高（Agent不知彼此） | 高（知道Queue不知道消费者） |
| **典型框架** | JADE DF, Tuplespace | RabbitMQ, Kafka, Redis Pub/Sub |

### 何时使用公告板 vs 消息队列

```
任务是否具有"知识价值"（结果值得保存供后续参考）？
├── 是 → 公告板模式（保留所有中间结果）
└── 否 → 继续判断
    ├── 是否需要精确的负载均衡？
    │   ├── 是 → 消息队列
    │   └── 否 → 公告板模式
    └── 是否需要保证消息顺序？
        ├── 是 → 消息队列
        └── 否 → 公告板模式
```

## 完整示例：多Agent文档处理系统

```python
import asyncio

class DocumentProcessingSystem:
    """
    基于公告板模式的多Agent文档处理系统。

    工作流程：
    1. 用户上传文档 → 文档解析Agent
    2. 解析Agent发布分析任务到公告板
    3. 分析Agent认领并处理
    4. 翻译Agent认领翻译任务
    5. 汇总Agent收集结果并输出
    """

    def __init__(self):
        self.blackboard = Blackboard()
        self.scheduler = Scheduler(self.blackboard)

        # 创建各Agent
        self.parser_agent = ParserAgent("parser-01", self.blackboard)
        self.analysis_agent = AnalysisAgent("analysis-01", self.blackboard)
        self.translate_agent = TranslateAgent("translate-01", self.blackboard)
        self.summary_agent = SummaryAgent("summary-01", self.blackboard)

    async def process_document(self, document_path: str):
        """处理一个文档的入口方法"""
        print(f"\n{'='*50}")
        print(f"开始处理文档: {document_path}")

        # 启动调度器
        scheduler_task = asyncio.create_task(self.scheduler.start())

        # 第1步：文档解析
        parsed = await self.parser_agent.parse_document(document_path)

        # 发布到公告板
        doc_entry_id = self.parser_agent.post_knowledge(
            title=f"文档内容: {document_path}",
            knowledge=parsed,
            tags=["document", "raw_content"]
        )

        # 第2步：发布分析任务
        analysis_task_id = self.parser_agent.post_task_request(
            title="分析文档内容和结构",
            description=f"请分析公告板上 {doc_entry_id} 的文档内容",
            required_skills=["document-analysis"],
            parent_id=doc_entry_id,
            priority=8
        )

        # 第3步：等待分析完成（带超时）
        analysis_result = await self._wait_for_result(
            analysis_task_id, timeout=30
        )

        if analysis_result:
            # 第4步：发布翻译任务
            translate_task_id = self.analysis_agent.post_task_request(
                title="翻译分析结果",
                description=f"将分析结果翻译为英文",
                required_skills=["translation"],
                parent_id=analysis_result.entry_id,
                priority=5
            )

            translate_result = await self._wait_for_result(
                translate_task_id, timeout=20
            )

            # 第5步：汇总输出
            final_result = await self.summary_agent.generate_summary(
                doc_entry_id, analysis_result, translate_result
            )
        else:
            final_result = {"error": "分析任务超时"}

        self.scheduler.stop()
        return final_result

    async def _wait_for_result(self, task_id: str,
                                timeout: int) -> Optional[BlackboardEntry]:
        """等待某个任务的结果出现在公告板上"""
        start = asyncio.get_event_loop().time()
        while asyncio.get_event_loop().time() - start < timeout:
            results = self.blackboard.get_children(task_id)
            for r in results:
                if r.entry_type == EntryType.TASK_RESULT:
                    return r
            await asyncio.sleep(0.5)
        return None


class ParserAgent(BlackboardAgent):
    """文档解析Agent：解析文档内容并发布到公告板"""
    def __init__(self, agent_id, blackboard):
        super().__init__(agent_id, ["document-parsing"], blackboard)

    async def parse_document(self, path: str) -> str:
        # 模拟文档解析
        await asyncio.sleep(1)
        content = f"[解析自 {path}] 文档内容摘要..."
        print(f"[{self.agent_id}] 文档解析完成")
        return content


class AnalysisAgent(BlackboardAgent):
    """文档分析Agent：从公告板认领分析任务"""
    def __init__(self, agent_id, blackboard):
        super().__init__(agent_id, ["document-analysis"], blackboard)

    async def run(self):
        """分析Agent的主循环"""
        while True:
            tasks = self.find_available_tasks()
            for task in tasks:
                if self.claim_task(task.entry_id):
                    # 读取需要的上下文
                    context = self.read_knowledge()
                    parent = self.blackboard.get_entry(task.parent_id)
                    if parent:
                        context.append(parent)

                    # 执行分析
                    result = f"分析结果: 文档包含..."
                    await asyncio.sleep(1.5)

                    # 发布结果
                    self.post_result(
                        title=f"分析完成: {task.title}",
                        result=result,
                        task_entry_id=task.entry_id
                    )
            await asyncio.sleep(2)


class TranslateAgent(BlackboardAgent):
    """翻译Agent"""
    def __init__(self, agent_id, blackboard):
        super().__init__(agent_id, ["translation"], blackboard)


class SummaryAgent(BlackboardAgent):
    """汇总Agent：收集公告板上的所有结果并生成最终输出"""
    def __init__(self, agent_id, blackboard):
        super().__init__(agent_id, ["summarization"], blackboard)

    async def generate_summary(self, doc_entry_id,
                                analysis_result, translate_result):
        print(f"[{self.agent_id}] 生成最终汇总...")
        await asyncio.sleep(0.5)
        return {
            "status": "success",
            "document_id": doc_entry_id,
            "analysis": analysis_result.content if analysis_result else None,
            "translation": translate_result.content if translate_result else None,
            "summary": "这是文档处理的完整汇总..."
        }
```

## 优缺点分析

### 优势

| 优势 | 说明 |
|------|------|
| **极端解耦** | Agent互不相知，仅通过公告板交互。新增Agent零侵入 |
| **自然的知识管理** | 所有中间产物自然沉淀为可复用的知识资产 |
| **灵活性** | Agent可以随时加入、离开，不影响系统运行 |
| **可追溯性** | 完整记录所有协作者的贡献，便于审计 |

### 劣势

| 劣势 | 说明 | 缓解策略 |
|------|------|---------|
| **公告板成为瓶颈** | 所有读写集中在公告板 | 分区公告板 + 缓存 |
| **信息过载** | 条目越来越多，查找困难 | 自动过期 + 标签体系 |
| **被动等待** | Agent需轮询是否有新任务 | 引入订阅通知机制 |
| **无强制流程** | 缺乏任务间的顺序保证 | 引入依赖关系和状态机 |

## 小结

公告板模式是多智能体系统中实现松散耦合协作的优雅方案。它的核心价值在于让Agent"不需要认识彼此"——Agent只需知道公告板的存在和自己的能力，就能自然地参与到协作中。

**适用场景**：
- 需要大量Agent并行协作的系统
- Agent频繁增减的动态环境
- 需要保留协作历史的场景
- 知识密集型系统（中间产物有复用价值）

**不适用场景**：
- 强时序要求的流水线任务
- 低延迟实时系统
- 简单的请求-响应交互

记住：公告板模式追求的不是速度和效率，而是**灵活性和可扩展性**。在正确场景下，它让多智能体系统变得优雅而健壮。
