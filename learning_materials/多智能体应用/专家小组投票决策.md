# 专家小组模式与投票决策

## 引言

如果Manager-Worker模式像一支军队（有明确的指挥官和士兵），那么**专家小组模式（Peer-to-Peer / Panel of Experts）**就像一个学术评审委员会——所有成员地位平等，通过讨论、辩论和投票来达成共识。

在AI多智能体系统中，专家小组模式让多个Agent以对等身份协作，各自从专业角度提供意见，最终通过投票机制汇总为集体决策。这种模式特别适合需要多角度审视、高度审慎、或者没有明显"正确答案"的开放性问题。

### 与Manager-Worker模式的关键区别

| 维度 | Manager-Worker | 专家小组 (Peer-to-Peer) |
|------|---------------|------------------------|
| 层级关系 | 有明确上下级 | 所有Agent平等 |
| 决策方式 | Manager做最终决定 | 集体投票/协商决定 |
| 通信模式 | 星型（Manager居中） | 全连接（任意两者可通信） |
| 任务分配 | Manager分配 | 自组织或协调器建议 |
| 适用场景 | 有明确分解的任务 | 需要多视角的审议任务 |
| 优势 | 效率高、结构清晰 | 视角广、决策稳健 |
| 劣势 | 单点决策风险 | 协调成本高、效率低 |

## 多专家评审机制

### 评审流程

```
①问题发布 → ②独立评审 → ③意见汇总 → ④交叉讨论 → ⑤投票决策 → ⑥结果输出
```

一步骤详解：

**① 问题发布**：协调器（Facilitator）将问题同时发送给所有专家。

**② 独立评审**：每个专家基于自己的专业视角独立分析。这一步的关键是确保专家之间互不影响，避免"锚定效应"（第一个发言的影响后续判断）。

```python
async def independent_review(question: str, experts: list[Agent]) -> list[Opinion]:
    """各专家独立评审，互不影响"""
    async def single_expert_review(expert: Agent) -> Opinion:
        # 每个专家独立进行评估
        result = await expert.analyze(question)
        return Opinion(
            expert_name=expert.name,
            expertise=expert.expertise,
            recommendation=result.recommendation,
            reasoning=result.reasoning,
            confidence=result.confidence,
            risks=result.identified_risks,
            assumptions=result.assumptions
        )

    # 并行执行所有专家的独立评审
    opinions = await asyncio.gather(
        *[single_expert_review(e) for e in experts]
    )
    return opinions
```

**③ 意见汇总**：协调器收集所有意见，识别共识和分歧点。

**④ 交叉讨论**：专家们阅读彼此的意见，进行讨论和辩论。这一步允许专家在了解到其他视角后修正自己的观点。

**⑤ 投票决策**：根据预定义的投票规则做出最终决定。

**⑥ 结果输出**：输出最终决策及完整的推理链。

## 投票策略

投票机制是专家小组模式的核心。不同的投票策略适用于不同的决策场景。

### 1. 多数投票（Majority Voting）

最简单直接的投票方式——少数服从多数。

```python
def majority_vote(opinions: list[Opinion]) -> VotingResult:
    """
    多数投票：选择得票最多的选项。
    平票时触发平票处理机制。
    """
    from collections import Counter

    recommendations = [o.recommendation for o in opinions]
    vote_count = Counter(recommendations)

    max_count = max(vote_count.values())
    winners = [opt for opt, cnt in vote_count.items() if cnt == max_count]

    if len(winners) == 1:
        return VotingResult(
            decision=winners[0],
            method="majority",
            confidence=vote_count[winners[0]] / len(opinions),
            vote_distribution=dict(vote_count),
            is_tie=False
        )
    else:
        return handle_tie(winners, vote_count, opinions)
```

**适用场景**：选项离散、非此即彼的决策（如"是否实施某方案"）。

**优势**：简单、透明、易理解。

**劣势**：忽略专家的专业领域差异和置信度差异。一个领域专家和门外汉的投票权重相同。

### 2. 加权投票（Weighted Voting）

根据专家的专业领域和问题的匹配度，给予不同的投票权重。

```python
def weighted_vote(opinions: list[Opinion],
                  question_domain: str,
                  expert_weights: dict[str, float] = None) -> VotingResult:
    """
    加权投票：根据专家在问题领域的专业度分配权重。

    权重因素：
    1. 专家领域与问题领域的匹配度
    2. 专家陈述的置信度
    3. 历史表现（如果跟踪的话）
    """
    if expert_weights is None:
        expert_weights = {}

    weighted_tally = {}
    total_weight = 0.0

    for opinion in opinions:
        # 计算该专家的综合权重
        domain_match = compute_domain_similarity(
            opinion.expertise, question_domain
        )
        base_weight = expert_weights.get(opinion.expert_name, 1.0)
        confidence_weight = opinion.confidence

        # 综合权重 = 领域匹配度 × 基础权重 × 置信度
        composite_weight = domain_match * base_weight * confidence_weight

        weighted_tally[opinion.recommendation] = \
            weighted_tally.get(opinion.recommendation, 0) + composite_weight
        total_weight += composite_weight

    # 标准化
    winner = max(weighted_tally, key=weighted_tally.get)
    return VotingResult(
        decision=winner,
        method="weighted",
        confidence=weighted_tally[winner] / total_weight,
        vote_distribution={
            k: v / total_weight for k, v in weighted_tally.items()
        },
        is_tie=False
    )


def compute_domain_similarity(expertise: str, question_domain: str) -> float:
    """计算专家领域与问题领域的相似度"""
    # 简单实现：关键词匹配
    # 生产环境可用embedding相似度
    expertise_keywords = set(expertise.lower().split())
    domain_keywords = set(question_domain.lower().split())
    overlap = expertise_keywords & domain_keywords
    if not overlap:
        return 0.3  # 基础权重
    return min(1.0, 0.3 + 0.7 * len(overlap) / len(domain_keywords))
```

**适用场景**：问题需要特定领域的深度知识，且各专家领域差异大。

**优势**：更精准——领域专家在相关问题上"话语权"更大。

**劣势**：权重设计主观，可能引入偏见。

### 3. 一致同意（Consensus / Unanimity）

要求所有专家达成一致。如果没有达成一致，进入讨论和协商阶段，直到达成一致或判定无法达成一致。

```python
async def consensus_decision(question: str, experts: list[Agent],
                              max_rounds: int = 3) -> VotingResult:
    """
    一致同意决策：要求所有专家达成共识。

    流程：
    1. 首轮独立评审
    2. 如果不一致，进入辩论轮
    3. 每轮辩论后重新投票
    4. 达到max_rounds仍不一致则降级为多数投票
    """
    opinions = await independent_review(question, experts)
    round_num = 1

    while round_num <= max_rounds:
        # 检查是否达成一致
        recommendations = {o.recommendation for o in opinions}
        if len(recommendations) == 1:
            return VotingResult(
                decision=recommendations.pop(),
                method="consensus",
                confidence=sum(o.confidence for o in opinions) / len(opinions),
                rounds=round_num,
                is_unanimous=True
            )

        if round_num == max_rounds:
            break

        # 进入辩论轮
        opinions = await debate_round(question, opinions, experts)
        round_num += 1

    # 降级为多数投票
    return majority_vote(opinions)


async def debate_round(question: str, current_opinions: list[Opinion],
                        experts: list[Agent]) -> list[Opinion]:
    """一轮辩论：专家阅读彼此意见后重新评估"""
    opinions_summary = "\n\n".join([
        f"专家 {o.expert_name}（{o.expertise}）的意见：\n"
        f"  建议：{o.recommendation}\n"
        f"  理据：{o.reasoning}\n"
        f"  风险：{', '.join(o.risks)}"
        for o in current_opinions
    ])

    async def re_evaluate(expert: Agent) -> Opinion:
        prompt = f"""
        你参与了对以下问题的专家小组评审。以下是所有专家的意见摘要。

        原始问题：{question}

        所有专家意见：
        {opinions_summary}

        请仔细阅读其他专家的意见。他们的视角中是否有你遗漏的点？
        是否有论据改变了你的看法？请给出你更新后的评估。

        如果你改变看法，请说明是什么论据说服了你。
        如果你坚持原观点，请说明为什么不同意其他专家的论据。
        """
        result = await expert.analyze(prompt)
        return Opinion(
            expert_name=expert.name,
            expertise=expert.expertise,
            recommendation=result.recommendation,
            reasoning=result.reasoning,
            confidence=result.confidence,
            risks=result.identified_risks,
            assumptions=result.assumptions
        )

    return await asyncio.gather(
        *[re_evaluate(e) for e in experts]
    )
```

**适用场景**：高风险决策（安全、法律、医疗），错误代价极高的场景。

**优势**：决策最稳健，充分考虑了各方意见。

**劣势**：耗时最长，可能陷入僵局（deadlock）。

### 4. Borda计数法

当需要在多个选项中排序时，Borda计数法是一种更公平的投票方式。

```python
def borda_count(opinions: list[Opinion]) -> VotingResult:
    """
    Borda计数：每位专家对选项进行排名，排名越高得分越多。
    最后总得分最高的选项获胜。

    适用于：需要在多个候选方案中选出最优的排序决策。
    """
    # 提取所有选项
    all_options = set()
    for opinion in opinions:
        all_options.update(opinion.rankings.keys())  # 假设有rankings字段

    n_options = len(all_options)
    scores = {opt: 0 for opt in all_options}

    for opinion in opinions:
        for rank, option in enumerate(opinion.rankings.keys()):
            # 排名第一得n-1分，排名最后得0分
            scores[option] += (n_options - 1 - rank)

    winner = max(scores, key=scores.get)
    return VotingResult(
        decision=winner,
        method="borda_count",
        confidence=scores[winner] / sum(scores.values()),
        vote_distribution=scores,
        is_tie=False
    )
```

### 投票策略对比

| 策略 | 复杂度 | 效率 | 公平性 | 抗操纵性 | 最佳场景 |
|------|--------|------|--------|---------|---------|
| 多数投票 | 低 | 高 | 中 | 低 | 快速的二元决策 |
| 加权投票 | 中 | 高 | 高 | 中 | 专业领域明确的复杂问题 |
| 一致同意 | 高 | 低 | 最高 | 高 | 高风险/不可逆决策 |
| Borda计数 | 中 | 中 | 高 | 中 | 多选项排序/优先级决策 |

## 辩论与说服机制

投票之前，专家之间的辩论是提升决策质量的关键环节。

### 结构化辩论框架

```python
class StructuredDebate:
    """
    结构化辩论框架：模仿学术会议的论文评审过程。
    """

    def __init__(self, topic: str, agents: list[Agent],
                 judge: Agent = None):
        self.topic = topic
        self.agents = agents
        self.judge = judge  # 可选的裁判Agent
        self.rounds = []

    async def run(self, num_rounds: int = 2) -> DebateResult:
        # 第一轮：各抒己见
        initial_statements = await self._opening_statements()

        # 中间轮：交叉质询
        for r in range(num_rounds):
            critiques = await self._peer_critique(initial_statements)
            rebuttals = await self._rebuttals(critiques)
            initial_statements = rebuttals  # 更新为反驳后的立场

        # 最终轮：总结陈词
        final_statements = await self._closing_statements(initial_statements)

        # 裁判判决（或投票）
        if self.judge:
            decision = await self._judge_decision(final_statements)
        else:
            decision = await self._vote()

        return DebateResult(
            topic=self.topic,
            decision=decision,
            transcript=self.rounds
        )

    async def _peer_critique(self, statements):
        """同行评审：每个专家对其他专家的观点提出批评"""
        critiques = {}
        for agent in self.agents:
            for other_name, other_statement in statements.items():
                if other_name == agent.name:
                    continue
                critique = await agent.critique(
                    target=other_name,
                    target_statement=other_statement,
                    topic=self.topic
                )
                critiques[f"{agent.name}->{other_name}"] = critique
        return critiques

    async def _rebuttals(self, critiques):
        """反驳：各专家回应收到的批评"""
        rebuttals = {}
        for agent in self.agents:
            my_critiques = {
                k: v for k, v in critiques.items()
                if k.endswith(f"->{agent.name}")
            }
            rebuttal = await agent.rebut(
                critiques=my_critiques,
                topic=self.topic
            )
            rebuttals[agent.name] = rebuttal
        return rebuttals
```

### 辩论对决策质量的提升

研究表明，引入辩论机制可以显著提升LLM的推理质量：

| 效果 | 机制 |
|------|------|
| 减少幻觉 | 被其他Agent质疑的信息需要提供依据 |
| 发现盲点 | 不同视角能发现单一视角忽略的问题 |
| 深化推理 | 被迫回应批评时，推理更加严谨 |
| 消除偏见 | 多视角碰撞减少单一视角的固有偏见 |

## 代码示例

### 专家小组评审系统完整实现

```python
import asyncio
from dataclasses import dataclass, field
from typing import Optional
from enum import Enum


class VoteMethod(Enum):
    MAJORITY = "majority"
    WEIGHTED = "weighted"
    CONSENSUS = "consensus"
    BORDA = "borda_count"


@dataclass
class ExpertConfig:
    name: str
    expertise: str
    system_prompt: str
    tools: list = field(default_factory=list)
    weight: float = 1.0


@dataclass
class Opinion:
    expert_name: str
    expertise: str
    recommendation: str
    reasoning: str
    confidence: float
    risks: list
    assumptions: list


class ExpertPanelSystem:
    def __init__(self, experts: list[ExpertConfig],
                 vote_method: VoteMethod):
        self.experts = experts
        self.vote_method = vote_method
        self.expert_agents = self._create_agents()

    def _create_agents(self) -> dict:
        """基于配置创建Agent实例"""
        # 这里简化实现，实际使用时接入LLM
        agents = {}
        for config in self.experts:
            agents[config.name] = ExpertAgent(
                name=config.name,
                expertise=config.expertise,
                system_prompt=config.system_prompt,
                tools=config.tools
            )
        return agents

    async def deliberate(self, question: str) -> dict:
        """执行完整的专家评审流程"""
        print(f"=== 专家小组评审开始 ===")
        print(f"问题: {question}")
        print(f"投票方式: {self.vote_method.value}")
        print(f"参与专家: {[e.name for e in self.experts]}")
        print()

        # 第1步：独立评审
        print("[1/4] 各专家独立评审...")
        opinions = await self._gather_independent_opinions(question)

        # 第2步：交叉评审
        print("[2/4] 专家交叉评审...")
        opinions = await self._cross_review(opinions, question)

        # 第3步：辩论（可选）
        if self.vote_method == VoteMethod.CONSENSUS:
            print("[3/4] 辩论达成共识...")
            opinions = await self._debate_for_consensus(opinions, question)
        else:
            print("[3/4] 跳过辩论（非共识模式）")

        # 第4步：投票决策
        print("[4/4] 投票决策...")
        result = self._vote(opinions)

        return {
            "question": question,
            "opinions": [o.to_dict() for o in opinions],
            "result": result,
            "vote_method": self.vote_method.value
        }

    async def _gather_independent_opinions(self, question: str):
        """各专家独立形成意见"""
        async def get_opinion(expert_config):
            agent = self.expert_agents[expert_config.name]
            result = await agent.analyze(question)
            return Opinion(
                expert_name=expert_config.name,
                expertise=expert_config.expertise,
                recommendation=result.get("recommendation", ""),
                reasoning=result.get("reasoning", ""),
                confidence=float(result.get("confidence", 0.8)),
                risks=result.get("risks", []),
                assumptions=result.get("assumptions", [])
            )

        return await asyncio.gather(
            *[get_opinion(c) for c in self.experts]
        )

    async def _cross_review(self, opinions, question):
        """让专家互相审查意见"""
        opinions_text = "\n---\n".join([
            f"[专家{o.expert_name} ({o.expertise})]\n"
            f"建议: {o.recommendation}\n"
            f"推理: {o.reasoning}"
            for o in opinions
        ])

        async def review_own(expert_config, my_opinion):
            agent = self.expert_agents[expert_config.name]
            prompt = f"""
            以下是关于问题的所有专家意见，请重新审视你的立场。

            原始问题: {question}

            所有意见:
            {opinions_text}

            你之前的意见: {my_opinion.recommendation}

            请回答：
            1. 其他专家的意见是否改变了你的看法？为什么？
            2. 你更新的建议是什么？
            3. 你对新建议的置信度（0-1）？
            """
            result = await agent.analyze(prompt)
            return Opinion(
                expert_name=expert_config.name,
                expertise=expert_config.expertise,
                recommendation=result.get("recommendation",
                                         my_opinion.recommendation),
                reasoning=result.get("reasoning", my_opinion.reasoning),
                confidence=float(result.get("confidence",
                                           my_opinion.confidence)),
                risks=result.get("risks", my_opinion.risks),
                assumptions=result.get("assumptions",
                                      my_opinion.assumptions)
            )

        return await asyncio.gather(
            *[review_own(self.experts[i], opinions[i])
              for i in range(len(self.experts))]
        )

    async def _debate_for_consensus(self, opinions, question):
        """辩论直到达成一致（最多3轮）"""
        for round_num in range(1, 4):
            recommendations = {o.recommendation for o in opinions}
            if len(recommendations) == 1:
                print(f"  第{round_num}轮达成共识！")
                return opinions

            print(f"  第{round_num}轮辩论开始（当前{len(recommendations)}种意见）...")
            opinions = await self._cross_review(opinions, question)

        print(f"  辩论结束，未能达成完全一致，将使用投票决定")
        return opinions

    def _vote(self, opinions) -> dict:
        """根据配置的投票方式执行投票"""
        if self.vote_method == VoteMethod.MAJORITY:
            return self._majority_vote(opinions)
        elif self.vote_method == VoteMethod.WEIGHTED:
            weights = {e.name: e.weight for e in self.experts}
            return self._weighted_vote(opinions, weights)
        elif self.vote_method == VoteMethod.CONSENSUS:
            return self._consensus_vote(opinions)
        else:
            return self._majority_vote(opinions)

    def _majority_vote(self, opinions):
        from collections import Counter
        recs = [o.recommendation for o in opinions]
        counter = Counter(recs)
        winner = counter.most_common(1)[0][0]
        return {
            "decision": winner,
            "votes": dict(counter),
            "confidence": counter[winner] / len(opinions)
        }

    def _weighted_vote(self, opinions, weights):
        tally = {}
        total = 0.0
        for o in opinions:
            w = weights.get(o.expert_name, 1.0) * o.confidence
            tally[o.recommendation] = tally.get(o.recommendation, 0) + w
            total += w
        winner = max(tally, key=tally.get)
        return {
            "decision": winner,
            "weighted_votes": {k: v/total for k, v in tally.items()},
            "confidence": tally[winner] / total
        }
```

## 与Manager-Worker模式的对比

### 场景选择指南

```
是否需要单一Agent做最终决策？
├── 是 → 需要明确的Owner → Manager-Worker
└── 否 → 继续判断
    ├── 决策是否高度可逆？
    │   ├── 是 → Manager-Worker（效率优先）
    │   └── 否 → 专家小组（稳健优先）
    ├── 是否需要多学科/多视角？
    │   ├── 是 → 专家小组
    │   └── 否 → Manager-Worker
    └── 时间是否紧迫？
        ├── 是 → Manager-Worker
        └── 否 → 专家小组
```

### 混合模式

在实践中，两种模式可以混合使用：

```
                    Manager
                       │
         ┌─────────────┼─────────────┐
         │             │             │
    关键决策点A    关键决策点B    关键决策点C
         │             │             │
    ┌────▼────┐   ┌────▼────┐   ┌────▼────┐
    │专家小组 │   │专家小组 │   │单专家   │
    │(投票)   │   │(投票)   │   │(快速)   │
    └─────────┘   └─────────┘   └─────────┘
```

Manager负责整体流程管理，但在关键决策点引入专家小组投票，兼顾了效率与稳健。

## 最佳实践

1. **独立先行**：首轮评审必须独立进行，避免锚定效应
2. **匿名评审**：评审意见不要过早透露来源，减少权威偏见
3. **明确分歧**：鼓励专家明确指出与其他意见的分歧点及其原因
4. **记录异议**：即使最终按多数意见决策，少数意见也应被记录
5. **设定超时**：辩论不能无限进行，设定轮次上限和超时机制
6. **人数为奇**：投票人数最好为奇数，减少平票概率
7. **专业加权**：在技术决策中，让领域专家的意见有更高权重

## 小结

专家小组投票模式借鉴了人类学术评审、司法裁决和委员会决策的智慧，让AI Agent能够以"集体智慧"的方式来处理复杂、开放性的问题。

**核心价值**：
- **多视角**：不同专业的Agent带来不同的分析框架
- **制衡**：Agent之间天然形成制衡，减少单一Agent的偏见和错误
- **稳健**：通过辩论和投票，最终决策经过了多轮审视

**使用建议**：
- 高风险、不可逆的决策：使用一致同意或加权投票
- 中等风险、需要专业判断：使用加权投票
- 低风险、需要快速决策：使用多数投票（甚至直接用Manager-Worker）

记住：**好决策来自多元视角的碰撞，而不是一个声音的独白**。
