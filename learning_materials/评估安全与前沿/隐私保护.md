# 隐私保护：数据脱敏与合规

## 引言

AI Agent处理用户数据的能力越强，隐私风险就越大。一个Agent可能在处理用户请求时接触到姓名、电话号码、身份证号、银行卡号、医疗记录、位置信息等敏感数据。这些数据一旦被不当使用、泄露或在模型训练中被"记住"，可能造成严重的隐私侵犯和法律后果。

隐私保护不是一个可选功能，而是Agent系统的**法律红线**——GDPR（欧盟通用数据保护条例）的最高罚款可达全球年营收的4%或2000万欧元（取高者），中国的《个人信息保护法》（PIPL）同样设有严厉的处罚条款。本文将从数据脱敏技术、隐私法规合规、本地化处理策略和前沿隐私技术四个维度，系统介绍Agent系统中的隐私保护体系。

## 数据脱敏技术

### 敏感信息识别

在脱敏之前，首先要能识别哪些数据是敏感的。

```python
import re
from typing import List, Tuple, Set


class SensitiveDataDetector:
    """
    敏感信息检测器。

    在数据进入Agent处理流程之前，先识别其中的敏感信息。
    """

    # 中国敏感信息正则模式
    CN_PATTERNS = {
        "身份证号": r'\b[1-9]\d{5}(?:19|20)\d{2}(?:0[1-9]|1[0-2])'
                    r'(?:0[1-9]|[12]\d|3[01])\d{3}[\dXx]\b',
        "手机号": r'\b1[3-9]\d{9}\b',
        "银行卡号": r'\b\d{16,19}\b',
        "邮箱": r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b',
        "车牌号": r'\b[京津沪渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽'
                  r'贵粤川青藏琼宁][A-HJ-NP-Z][A-HJ-NP-Z0-9]{4,5}'
                  r'[A-HJ-NP-Z0-9挂学警港澳]\b',
        "IP地址": r'\b(?:\d{1,3}\.){3}\d{1,3}\b',
        "家庭住址": r'(?:省|市|区|县|镇|街道|路|巷|号|栋|单元|室)',
    }

    # 通用敏感信息模式
    GENERAL_PATTERNS = {
        "密码": r'(?:password|passwd|pwd|密码)\s*[:=]\s*\S+',
        "API密钥": r'\b(?:sk-|api_key=|apikey=)[A-Za-z0-9_-]{20,}\b',
        "Token": r'\b[A-Za-z0-9_-]{32,}\b',
        "信用卡号": r'\b(?:4[0-9]{12}(?:[0-9]{3})?|'
                    r'5[1-5][0-9]{14}|'
                    r'3[47][0-9]{13})\b',
        "SSN": r'\b\d{3}-\d{2}-\d{4}\b',
    }

    def detect(self, text: str) -> List[dict]:
        """
        检测文本中的所有敏感信息。

        返回匹配项的列表，每项包含类型、位置、原始值。
        """
        detections = []

        # 检查中国模式
        for label, pattern in self.CN_PATTERNS.items():
            for match in re.finditer(pattern, text):
                detections.append({
                    "type": label,
                    "value": match.group(),
                    "start": match.start(),
                    "end": match.end(),
                    "jurisdiction": "CN"
                })

        # 检查通用模式（排除已匹配的区域）
        matched_ranges = [(d["start"], d["end"]) for d in detections]
        for label, pattern in self.GENERAL_PATTERNS.items():
            for match in re.finditer(pattern, text, re.IGNORECASE):
                # 检查是否与已有匹配重叠
                if not self._overlaps(match.start(), match.end(),
                                       matched_ranges):
                    detections.append({
                        "type": label,
                        "value": match.group(),
                        "start": match.start(),
                        "end": match.end(),
                        "jurisdiction": "general"
                    })

        # 按位置排序（从左到右）
        detections.sort(key=lambda d: d["start"])
        return detections

    def _overlaps(self, start: int, end: int,
                   existing: List[Tuple[int, int]]) -> bool:
        """检查范围是否与已有范围重叠"""
        for es, ee in existing:
            if not (end <= es or start >= ee):
                return True
        return False
```

### 脱敏策略

```python
class DataMasker:
    """
    数据脱敏器。

    根据脱敏级别和场景选择合适的策略。
    """

    # 脱敏策略
    STRATEGIES = {
        "redaction": "完全移除",
        "replacement": "替换为固定占位符",
        "pseudonymization": "替换为一致的假名",
        "generalization": "降低精度（如具体年龄 → 年龄段）",
        "tokenization": "替换为不可逆的Token",
        "encryption": "加密存储（可逆，需要密钥）",
    }

    def mask_text(self, text: str,
                   detections: List[dict],
                   strategy: str = "replacement") -> str:
        """
        对文本进行脱敏处理。

        从后往前替换，避免位置偏移问题。
        """
        # 按位置从后往前排序
        detections_sorted = sorted(detections,
                                    key=lambda d: d["start"],
                                    reverse=True)

        masked_text = text
        for i, detection in enumerate(detections_sorted):
            mask_value = self._generate_mask(
                detection, i, strategy
            )
            masked_text = (
                masked_text[:detection["start"]] +
                mask_value +
                masked_text[detection["end"]:]
            )

        return masked_text

    def _generate_mask(self, detection: dict,
                        index: int,
                        strategy: str) -> str:
        """生成脱敏替换值"""
        dtype = detection["type"]

        if strategy == "redaction":
            return ""  # 完全移除

        elif strategy == "replacement":
            # 固定占位符
            placeholders = {
                "身份证号": "[身份证号]",
                "手机号": "[手机号]",
                "银行卡号": "[银行卡号]",
                "邮箱": "[邮箱]",
                "密码": "[密码]",
                "API密钥": "[API_KEY]",
                "家庭住址": "[地址]",
            }
            return placeholders.get(dtype, f"[{dtype}]")

        elif strategy == "pseudonymization":
            # 保持一致性的假名
            import hashlib
            original = detection["value"]
            hash_val = hashlib.sha256(original.encode()).hexdigest()[:8]

            if dtype == "手机号":
                return f"USER_{hash_val}"
            elif dtype == "邮箱":
                return f"user_{hash_val}@anonymous.com"
            elif dtype == "身份证号":
                return f"ID_{hash_val}"
            else:
                return f"ANON_{hash_val}"

        elif strategy == "generalization":
            # 降低精度
            if dtype == "家庭住址":
                # 保留城市，隐藏具体地址
                return self._generalize_address(detection["value"])
            elif dtype == "银行卡号":
                # 只保留银行信息
                return f"银行账户-{detection['value'][:4]}****"

        return f"[MASKED_{dtype}]"

    def _generalize_address(self, address: str) -> str:
        """地址泛化：只保留省市信息"""
        import re
        # 找到"省"或"市"的位置，之后的内容隐藏
        match = re.search(r'(.+?[省市])', address)
        if match:
            return match.group(1) + "[详细地址已隐藏]"
        return "[地址已隐藏]"


class PrivacyPipeline:
    """
    隐私处理流水线：检测 → 脱敏 → 处理 → 还原（如需要）

    集成到Agent的输入/输出处理流程中。
    """

    def __init__(self, strategy: str = "replacement"):
        self.detector = SensitiveDataDetector()
        self.masker = DataMasker()
        self.strategy = strategy

        # 存储脱敏映射，用于后续还原
        self._pseudonym_map: dict[str, str] = {}

    def sanitize_input(self, user_input: str) -> dict:
        """
        在Agent处理之前脱敏用户输入。
        """
        detections = self.detector.detect(user_input)
        masked = self.masker.mask_text(
            user_input, detections, self.strategy
        )

        return {
            "original": user_input,
            "sanitized": masked,
            "detections": detections,
            "has_sensitive_data": len(detections) > 0
        }

    def sanitize_output(self, agent_output: str,
                         input_detections: List[dict]) -> str:
        """
        脱敏Agent输出（防止Agent"回忆"起被脱敏的信息）。
        """
        # 重新检测Agent输出
        output_detections = self.detector.detect(agent_output)

        # 合并输入和输出的检测结果
        all_detections = input_detections + output_detections

        # 对输出也进行脱敏
        masked = self.masker.mask_text(
            agent_output, output_detections, self.strategy
        )

        return masked
```

## GDPR与PIPL合规要点

### 两大法规对比

| 维度 | GDPR（欧盟） | PIPL（中国） |
|------|------------|------------|
| **生效日期** | 2018年5月25日 | 2021年11月1日 |
| **管辖范围** | 处理欧盟居民数据的所有组织 | 在境内处理自然人个人信息 |
| **合法基础** | 同意/合同/法定义务/合法利益等6项 | 同意/合同/法定职责等7项 |
| **最高罚款** | 2000万欧元或全球营收4% | 5000万元或上年营收5% |
| **数据主体权利** | 访问/更正/删除/限制/可携/反对 | 知情/决定/查阅/复制/更正/删除 |
| **跨境传输** | 充分性认定/SCC/BCR | 安全评估/标准合同/认证 |
| **DPO要求** | 特定情况下强制的数据保护官 | 特定情况下指定个人信息保护负责人 |
| **数据泄露通知** | 72小时内 | 立即（未明确具体时限） |

### Agent系统合规检查清单

```python
class ComplianceChecker:
    """
    Agent系统隐私合规检查器。

    基于GDPR和PIPL的要求，检查Agent系统是否合规。
    """

    def __init__(self, jurisdiction: str = "CN"):
        """
        jurisdiction: "CN" (PIPL), "EU" (GDPR), "BOTH"
        """
        self.jurisdiction = jurisdiction

    def run_compliance_check(self, agent_system_config: dict) -> dict:
        """
        运行全面的合规检查。

        返回每个检查点的通过/失败状态和风险等级。
        """
        results = []

        # 1. 数据收集
        results.append(self._check_data_collection(
            agent_system_config
        ))

        # 2. 数据存储
        results.append(self._check_data_storage(
            agent_system_config
        ))

        # 3. 数据处理
        results.append(self._check_data_processing(
            agent_system_config
        ))

        # 4. 用户权利
        results.append(self._check_user_rights(
            agent_system_config
        ))

        # 5. 安全措施
        results.append(self._check_security_measures(
            agent_system_config
        ))

        # 6. 跨境数据
        results.append(self._check_cross_border_transfer(
            agent_system_config
        ))

        # 汇总
        passed = sum(1 for r in results if r["status"] == "pass")
        failed = sum(1 for r in results if r["status"] == "fail")
        warnings = sum(1 for r in results if r["status"] == "warning")

        return {
            "total_checks": len(results),
            "passed": passed,
            "failed": failed,
            "warnings": warnings,
            "compliance_score": passed / len(results) * 100,
            "details": results,
            "critical_issues": [
                r for r in results
                if r["status"] == "fail" and r["severity"] == "critical"
            ]
        }

    def _check_data_collection(self, config: dict) -> dict:
        """检查数据收集是否合规"""
        issues = []

        # (1) 是否有用户同意机制
        if not config.get("has_consent_mechanism"):
            issues.append("缺少用户同意机制")
            severity = "critical"
        else:
            severity = "low"

        # (2) 是否收集了必要范围之外的数据
        collected_fields = config.get("collected_data_fields", [])
        necessary_fields = config.get("necessary_data_fields", [])
        extra_fields = set(collected_fields) - set(necessary_fields)
        if extra_fields and not config.get("justification_for_extra"):
            issues.append(f"收集了非必要数据字段: {extra_fields}")

        # (3) 是否有隐私政策
        if not config.get("has_privacy_policy"):
            issues.append("缺少隐私政策")
            severity = max(severity, "critical")

        return {
            "category": "data_collection",
            "status": "fail" if severity == "critical"
                     else "warning" if issues else "pass",
            "severity": severity,
            "issues": issues
        }

    def _check_data_storage(self, config: dict) -> dict:
        """检查数据存储是否合规"""
        issues = []

        # (1) 数据是否加密存储
        if not config.get("encryption_at_rest"):
            issues.append("存储数据未加密")

        # (2) 是否有数据保留期限
        if not config.get("data_retention_period"):
            issues.append("未定义数据保留期限")

        # (3) 数据是否存储在正确的司法管辖区
        storage_location = config.get("storage_location", "")
        if self.jurisdiction == "CN" and "china" not in storage_location.lower():
            issues.append("中国用户数据应存储在中国境内(PIPL第39条)")

        severity = "critical" if len(issues) >= 2 else "high"
        return {
            "category": "data_storage",
            "status": "fail" if issues else "pass",
            "severity": severity,
            "issues": issues
        }

    def _check_data_processing(self, config: dict) -> dict:
        """检查数据处理是否合规"""
        issues = []

        # (1) 是否将用户数据用于LLM训练
        if config.get("use_for_training"):
            issues.append("严禁将用户数据用于模型训练(GDPR第22条/PIPL第24条)")

        # (2) 是否有数据最小化处理
        if not config.get("data_minimization"):
            issues.append("未实施数据最小化原则")

        return {
            "category": "data_processing",
            "status": "fail" if issues else "pass",
            "severity": "critical" if issues else "low",
            "issues": issues
        }

    # ... 更多检查方法
```

## 本地化处理 vs 云端处理

### 决策框架

```
是否需要将用户数据发送到云端LLM API？

├── 数据是否包含个人身份信息（PII）？
│   ├── 是 → 优先考虑本地化处理
│   └── 否 → 可以云端处理（需脱敏）
│
├── 数据是否涉及商业机密？
│   ├── 是 → 优先本地化
│   └── 否 → 可选云端
│
├── 是否有监管要求（金融/医疗/政府）？
│   ├── 是 → 必须本地化 + 审计
│   └── 否 → 可选云端
│
└── 本地是否有足够的计算资源？
    ├── 是 → 本地化（最安全）
    └── 否 → 云端 + 强加密 + 合同保障
```

### 分层处理架构

```python
class TieredPrivacyArchitecture:
    """
    分层隐私架构：根据数据敏感度选择不同的处理层级。

    第1层：本地预处理（数据清理、脱敏）→ 始终本地
    第2层：边缘处理（简单任务用小模型）→ 可选本地
    第3层：云端处理（复杂任务用大模型）→ 仅脱敏数据
    """

    def __init__(self):
        self.local_model = None     # 本地小模型（如Llama-3-8B）
        self.cloud_model = "claude-sonnet-4-20250514"

    def process(self, user_input: str,
                sensitivity_level: str) -> dict:
        """
        根据数据敏感度选择处理层级。
        """
        # 第1层：始终在本地进行预处理和脱敏
        detector = SensitiveDataDetector()
        detections = detector.detect(user_input)

        if sensitivity_level == "high":
            # 包含个人身份信息 → 全部本地处理
            sanitized = self._local_sanitize(user_input, detections)
            result = self._process_locally(sanitized)
            result["processing_tier"] = "local_only"

        elif sensitivity_level == "medium":
            # 包含一般敏感信息 → 本地脱敏后云端处理
            sanitized = self._local_sanitize(user_input, detections)
            result = self._process_cloud(sanitized)
            result["processing_tier"] = "local_preprocess+cloud"

        else:
            # 公开信息 → 可直接云端
            sanitized = user_input
            result = self._process_cloud(sanitized)
            result["processing_tier"] = "cloud"

        # 第4层：输出过滤（确保结果中不包含敏感信息）
        result["output"] = self._filter_sensitive_output(
            result["output"]
        )

        return result

    def _local_sanitize(self, text, detections):
        """本地脱敏"""
        masker = DataMasker()
        return masker.mask_text(text, detections, "replacement")

    def _process_locally(self, sanitized_text):
        """使用本地模型处理"""
        pass

    def _process_cloud(self, sanitized_text):
        """发送到云端模型处理"""
        pass

    def _filter_sensitive_output(self, output):
        """过滤输出中的敏感信息"""
        detector = SensitiveDataDetector()
        detections = detector.detect(output)
        if detections:
            masker = DataMasker()
            output = masker.mask_text(output, detections, "replacement")
        return output
```

## 差分隐私基础

### 概念阐释

**差分隐私（Differential Privacy）** 是隐私保护的黄金标准。它提供了一种数学保证：**无论单个个体是否在数据集中，分析结果几乎相同**。

直观理解：如果有人能通过查询你的系统推断出"张三是否在数据库中"，那你的系统就不够私密。差分隐私通过添加精心校准的噪声，使得任何单个个体对输出结果的"贡献"不可区分。

```python
import numpy as np


class DifferentialPrivacy:
    """
    差分隐私基础实现。

    核心参数：
    ε (epsilon): 隐私预算，越小隐私保护越强（但数据可用性越低）
                 典型值：0.1（强） ~ 10（弱）
    δ (delta): 失败概率，通常设为极小值（10^-5 ~ 10^-7）
    """

    def __init__(self, epsilon: float = 1.0,
                 delta: float = 1e-5):
        self.epsilon = epsilon
        self.delta = delta

    def laplace_mechanism(self, query_result: float,
                           sensitivity: float) -> float:
        """
        拉普拉斯机制：最基础的差分隐私实现。

        向查询结果添加拉普拉斯噪声。

        sensitivity: 查询的敏感度（单条数据对结果的最大影响）
        例如：计数查询的敏感度 = 1（多一个人最多让计数+1）
        """
        scale = sensitivity / self.epsilon
        noise = np.random.laplace(0, scale)
        return query_result + noise

    def gaussian_mechanism(self, query_result: float,
                            sensitivity: float) -> float:
        """
        高斯机制：满足(ε, δ)-差分隐私。

        相比拉普拉斯机制，对异常值更不敏感。
        """
        sigma = (sensitivity / self.epsilon) * \
                np.sqrt(2 * np.log(1.25 / self.delta))
        noise = np.random.normal(0, sigma)
        return query_result + noise

    def exponential_mechanism(self, candidates: list,
                               scores: list[float],
                               sensitivity: float) -> int:
        """
        指数机制：用于非数值型查询（如选择最佳类别）。

        以与exp(ε * score / (2 * sensitivity))成正比的概率
        选择每个候选。
        """
        scaled_scores = [
            self.epsilon * s / (2 * sensitivity)
            for s in scores
        ]
        # Softmax采样
        exp_scores = np.exp(scaled_scores - np.max(scaled_scores))
        probs = exp_scores / exp_scores.sum()
        return np.random.choice(len(candidates), p=probs)

    def clip_sensitivity(self, values: list[float],
                          clip_bound: float) -> list[float]:
        """
        敏感度裁剪：限制单个数据对结果的贡献。

        在分析之前，将所有值裁剪到[-C, C]范围内。
        这限制了最大敏感度，从而减少需要添加的噪声。
        """
        return [max(-clip_bound, min(clip_bound, v))
                for v in values]


# ============ 在Agent系统中的应用 ============
class PrivacyPreservingAnalytics:
    """
    使用差分隐私的Agent分析系统。

    例如：Agent需要报告"用户平均满意度"，但不能暴露个人评分。
    """

    def __init__(self, dp: DifferentialPrivacy):
        self.dp = dp

    def private_average(self, values: list[float],
                         clip_bound: float = 5.0) -> float:
        """
        计算隐私保护的平均值。

        步骤：
        1. 裁剪值（限制敏感度）
        2. 计算总和
        3. 添加噪声
        4. 除以计数（加上噪声）得到平均值
        """
        n = len(values)

        # 裁剪
        clipped = self.dp.clip_sensitivity(values, clip_bound)

        # 带噪声的总和
        noisy_sum = self.dp.laplace_mechanism(
            sum(clipped), clip_bound
        )

        # 带噪声的计数
        noisy_count = max(1, int(self.dp.laplace_mechanism(n, 1)))

        return noisy_sum / noisy_count

    def private_top_k(self, items: list[str],
                       frequencies: dict[str, int],
                       k: int = 3) -> list[str]:
        """
        隐私保护的Top-K查询。

        不会泄露某个具体item的精确频率。
        """
        candidates = list(frequencies.keys())
        scores = [
            self.dp.laplace_mechanism(
                frequencies[c], sensitivity=1
            )
            for c in candidates
        ]

        # 按加噪后的分数排序
        ranked = sorted(zip(candidates, scores),
                       key=lambda x: x[1], reverse=True)
        return [item for item, _ in ranked[:k]]
```

## 实践建议

### Agent隐私保护checklist

```
□ 数据收集阶段
  □ 明确告知用户将收集哪些数据
  □ 获取用户的明确同意
  □ 只收集完成任务所必需的最少数据
  □ 提供隐私政策的链接

□ 数据存储阶段
  □ 加密存储所有个人数据（AES-256）
  □ 设置数据保留期限和自动删除机制
  □ 敏感数据不出境（遵守数据本地化要求）
  □ 分离存储标识符和业务数据

□ 数据处理阶段
  □ 输入数据进入Agent前先脱敏
  □ 不要将用户数据用于模型训练
  □ 实施数据最小化原则
  □ 记录所有数据处理活动（审计）

□ Agent输出阶段
  □ 检查输出是否包含敏感信息
  □ 对包含PII的输出进行过滤
  □ 不存储完整的对话历史（仅保留必要的上下文）

□ 用户权利保障
  □ 提供数据访问和导出功能
  □ 提供数据删除功能（被遗忘权）
  □ 提供处理限制和反对功能

□ 安全保障
  □ 传输中使用TLS 1.3加密
  □ API密钥定期轮换
  □ 实施访问控制和最小权限
  □ 定期安全审计和渗透测试
```

## 小结

隐私保护不是Agent系统的"附加功能"，而是必须内建在架构中的核心要求。一个没有隐私保护的Agent系统，无论功能多么强大，都是在法律和道德上的定时炸弹。

**核心原则**：

1. **默认隐私（Privacy by Default）**：系统默认以最高隐私标准运行
2. **设计隐私（Privacy by Design）**：隐私保护融入系统架构设计，而非事后修补
3. **数据最小化（Data Minimization）**：只收集和处理绝对必要的数据
4. **透明性（Transparency）**：让用户清楚知道数据如何被使用
5. **用户控制（User Control）**：让用户掌握自己数据的主导权

**技术栈总结**：

| 层级 | 技术手段 |
|------|---------|
| 识别 | 敏感信息检测（正则 + NER） |
| 防护 | 脱敏（匿名化/假名化/泛化） |
| 传输 | TLS 1.3加密 |
| 存储 | AES-256加密 + 访问控制 |
| 计算 | 差分隐私 + 联邦学习 |
| 审计 | 不可篡改日志 + 合规检查 |

记住：**隐私保护不是阻碍创新的枷锁，而是让用户信任你、放心使用你产品的基石**。在隐私和功能之间，永远不要牺牲隐私。
