# 量化评估：推理能力、工具使用准确率与任务完成率

## 引言

"这个Agent好用吗？" 这是每个Agent开发者都会被问到的问题。但"好用"是一个模糊的概念——是对指令的理解准确？是工具调用没有bug？还是能自主完成复杂任务？要真正回答这个问题，我们需要一套**可量化、可复现、可对比**的评估体系。

Agent的评估不同于传统模型评估。一个聊天模型只需评估"说的对不对"，但一个Agent需要评估"做的对不对"——它选择执行了哪个动作、使用了哪个工具、经过了多少步、最终是否达成了目标。这就要求我们在多个维度上建立精细化的评估框架。本文将系统介绍如何从推理能力、工具使用准确率和任务完成率三个核心维度来量化评估Agent的表现。

## 评估框架设计

### Agent能力金字塔

```
                        ┌─────────────┐
                        │   任务完成   │  ← 终极目标
                        │  (Task Done) │
                        └──────┬──────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
              ┌─────▼─────┐         ┌─────▼─────┐
              │ 工具使用   │         │  推理质量  │  ← 中间层能力
              │ (Tool Use) │         │(Reasoning) │
              └─────┬─────┘         └─────┬─────┘
                    │                     │
              ┌─────▼─────────────────────▼─────┐
              │         基础语言能力              │  ← 底层基础
              │     (理解、生成、遵循指令)        │
              └─────────────────────────────────┘
```

每个上层能力的评估都依赖于下层能力。一个Agent如果基础语言能力不达标，就不可能做好工具使用；工具使用不达标，就不可能完成复杂任务。

## 评估指标定义

### 1. 任务成功率（Task Success Rate, TSR）

任务成功率是最直观、最重要的Agent评估指标。它回答了一个简单的问题：**Agent完成了多少任务？**

```python
class TaskSuccessRate:
    """
    任务成功率评估。

    不同的任务类型有不同的"成功"定义。
    """

    @staticmethod
    def binary_success(completed: int,
                       total: int) -> float:
        """
        二元成功率：完成 / 总数

        适用场景：有明确完成条件的任务。
        例如：成功预订机票、成功生成报告、成功修复Bug。
        """
        return completed / total if total > 0 else 0.0

    @staticmethod
    def partial_success(scores: List[float]) -> float:
        """
        部分成功率：考虑部分完成的任务。

        适用场景：任务完成度有连续等级的场景。
        例如：完成了一个5步任务中的3步。

        评分：每个任务0-1之间的分数，取平均。
        """
        return sum(scores) / len(scores) if scores else 0.0

    @staticmethod
    def weighted_success(scores: List[float],
                         weights: List[float]) -> float:
        """
        加权成功率：不同任务有不同重要性。

        适用场景：任务重要性不均等。
        例如：核心任务权重10，辅助任务权重1。
        """
        total_weight = sum(weights)
        return (sum(s * w for s, w in zip(scores, weights)) /
                total_weight if total_weight > 0 else 0.0)

    @staticmethod
    def success_with_granularity(
            outcomes: List[dict]) -> dict:
        """
        细粒度成功率分析：按任务类型、难度等维度分解。

        返回每个子类的成功率，帮助定位Agent的强弱项。
        """
        by_category = {}
        for outcome in outcomes:
            cat = outcome.get("category", "unknown")
            if cat not in by_category:
                by_category[cat] = {"success": 0, "total": 0}
            by_category[cat]["total"] += 1
            if outcome["success"]:
                by_category[cat]["success"] += 1

        return {
            cat: data["success"] / data["total"]
            for cat, data in by_category.items()
        }
```

```python
class TSRPitfalls:

         预算不超过2000元，并将确认信息发送到用户邮箱。
         满足以上所有条件视为成功，任一条件不满足视为失败。"
        """
        pass

    @staticmethod
    def pitfall_2_insufficient_samples(n_success: int,
                                         n_total: int):
        """
        陷阱2：样本量不足导致统计不可靠。

        评估100个任务，成功率70%，置信区间是多少？
        """
        import math
        p = n_success / n_total
        se = math.sqrt(p * (1 - p) / n_total)
        ci_lower = p - 1.96 * se
        ci_upper = p + 1.96 * se
        print(f"成功率: {p:.1%}")
        print(f"95%置信区间: [{ci_lower:.1%}, {ci_upper:.1%}]")
        if ci_upper - ci_lower > 0.1:
            print("警告：置信区间过宽，建议增加样本量")
        return ci_lower, ci_upper

    @staticmethod
    def pitfall_3_benchmark_contamination():
        """
        陷阱3：评测数据污染。

        如果训练数据中包含了评测数据，评测结果会被高估。
        对策：
        - 使用闭源/私有的评测数据集
```

### 2. 工具使用准确率（Tool Accuracy, TA）

工具使用是Agent区别于普通聊天模型的核心能力。衡量工具使用质量需要从多个子维度入手。

```python
class ToolAccuracy:
    """
    工具使用准确率评估。

    工具使用的四个子维度：
    1. 工具选择 (Tool Selection)：选对了工具吗？
    2. 参数正确性 (Parameter Correctness)：参数填对了吗？
    3. 调用时机 (Call Timing)：在正确的时机调用了吗？
    4. 结果使用 (Result Utilization)：正确使用工具返回的结果了吗？
    """

    @staticmethod
    def tool_selection_accuracy(
            agent_tool_calls: List[str],
            oracle_tool_calls: List[str]) -> dict:
        """
        工具选择准确率。

        衡量Agent是否选择了正确的工具。

        指标：
        - Precision: Agent选择的工具中有多少是真正需要的
        - Recall: 真正需要的工具中Agent选择了多少
        """
        agent_set = set(agent_tool_calls)
        oracle_set = set(oracle_tool_calls)

        tp = agent_set & oracle_set
        fp = agent_set - oracle_set
        fn = oracle_set - agent_set

        precision = len(tp) / len(agent_set) if agent_set else 0.0
        recall = len(tp) / len(oracle_set) if oracle_set else 0.0
        f1 = (2 * precision * recall / (precision + recall)
              if (precision + recall) > 0 else 0.0)

        return {
            "precision": precision,
            "recall": recall,
            "f1": f1,
            "missed_tools": list(fn),      # 该用但没用的工具
            "extra_tools": list(fp),       # 不该用但用了的工具
        }

    @staticmethod
    def parameter_accuracy(tool_call: dict,
                            oracle_call: dict) -> float:
        """
        参数正确性。

        衡量工具调用的参数是否准确。

        参数准确性分级：
        - exact: 参数完全正确
        - type_correct: 参数类型正确但值不完全对
        - partially: 部分参数正确
        - wrong: 参数完全错误
        """
        agent_params = tool_call.get("parameters", {})
        oracle_params = oracle_call.get("parameters", {})

        if not oracle_params:
            return 1.0 if not agent_params else 0.0

        correct_count = 0
        for key, expected_value in oracle_params.items():
            actual_value = agent_params.get(key)
            if actual_value == expected_value:
                correct_count += 1
            elif (isinstance(actual_value, type(expected_value))
                  and actual_value is not None):
                correct_count += 0.5  # 类型正确但值不精确

        return correct_count / len(oracle_params)

    @staticmethod
    def tool_use_sequence_accuracy(
            agent_sequence: List[str],
            oracle_sequence: List[str]) -> float:
        """
        工具使用序列准确性。

        衡量工具调用的顺序是否合理。
        有些任务对工具调用顺序有严格要求。
        """
        # 计算最长公共子序列（LCS）长度
        m, n = len(agent_sequence), len(oracle_sequence)
        dp = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if agent_sequence[i-1] == oracle_sequence[j-1]:
                    dp[i][j] = dp[i-1][j-1] + 1
                else:
                    dp[i][j] = max(dp[i-1][j], dp[i][j-1])

        lcs_len = dp[m][n]
        # 用LCS长度与oracle序列长度的比值
        return lcs_len / len(oracle_sequence) if oracle_sequence else 1.0

    @staticmethod
    def result_utilization_rate(
            tool_output: str,
            agent_response: str) -> float:
        """
        工具结果使用率。

        衡量Agent是否真的用了工具返回的结果。

        如果Agent调用了工具但回答中完全没有体现工具结果，
        要么是工具调用没用（浪费资源），要么是Agent忽略了结果。
        """
        # 提取工具输出中的关键信息
        tool_keywords = set(tool_output.lower().split())

        # 检查Agent回答中包含多少工具输出的关键信息
        response_keywords = set(agent_response.lower().split())
        overlap = tool_keywords & response_keywords

        # 对罕见词给更高权重（IDF加权）
        utilization = (len(overlap) / len(tool_keywords)
                      if tool_keywords else 0.0)

        return utilization


class ComprehensiveToolEvaluator:
    """综合工具使用评估器"""

    def evaluate_episode(self,
                         agent_trajectory: List[dict],
                         oracle_trajectory: List[dict]) -> dict:
        """
        评估一个完整任务回合中的工具使用质量。
        """
        metrics = ToolAccuracy()

        # 1. 工具选择
        agent_tools = [t["tool_name"] for t in agent_trajectory]
        oracle_tools = [t["tool_name"] for t in oracle_trajectory]
        selection = metrics.tool_selection_accuracy(
            agent_tools, oracle_tools
        )

        # 2. 参数准确性
        param_scores = []
        for at, ot in zip(agent_trajectory, oracle_trajectory):
            score = metrics.parameter_accuracy(at, ot)
            param_scores.append(score)
        avg_param = (sum(param_scores) / len(param_scores)
                    if param_scores else 1.0)

        # 3. 序列准确性
        sequence_acc = metrics.tool_use_sequence_accuracy(
            agent_tools, oracle_tools
        )

        # 4. 效率（使用比标准更少的工具调用是加分项）
        efficiency = (len(oracle_trajectory) /
                     max(len(agent_trajectory), 1))

        return {
            "tool_selection": selection,
            "parameter_accuracy": avg_param,
            "sequence_accuracy": sequence_acc,
            "efficiency": min(1.0, efficiency),
            "total_tool_calls": len(agent_trajectory),
            "oracle_tool_calls": len(oracle_trajectory),
        }
```

### 3. 推理步骤评估（Reasoning Steps Evaluation）

Agent的推理质量是决定其行为质量的根本。推理评估是最难自动化的部分。

```python
class ReasoningEvaluation:
    """
    推理质量评估。

    评估维度：
    1. 推理链条完整性 (Chain Completeness)
    2. 推理逻辑正确性 (Logical Correctness)
    3. 推理步骤效率 (Step Efficiency)
    4. 中间结论准确性 (Intermediate Accuracy)
    """

    @staticmethod
    def chain_completeness(agent_reasoning: str,
                            oracle_reasoning: str) -> dict:
        """
        推理链条完整性。

        衡量Agent的推理是否覆盖了所有必要的推理步骤。
        """
        # 提取推理步骤
        agent_steps = ReasoningEvaluation._extract_steps(
            agent_reasoning
        )
        oracle_steps = ReasoningEvaluation._extract_steps(
            oracle_reasoning
        )

        # 计算每个标准步骤是否被Agent的推理覆盖
        step_match_scores = []
        for o_step in oracle_steps:
            best_match = max(
                (ReasoningEvaluation._step_similarity(
                    o_step, a_step
                ) for a_step in agent_steps),
                default=0.0
            )
            step_match_scores.append(best_match)

        completeness = (sum(step_match_scores) /
                       len(step_match_scores)
                       if step_match_scores else 0.0)

        # 识别遗漏的关键步骤
        missed_steps = [
            oracle_steps[i] for i, score in enumerate(step_match_scores)
            if score < 0.3
        ]

        return {
            "completeness": completeness,
            "total_oracle_steps": len(oracle_steps),
            "covered_steps": sum(1 for s in step_match_scores
                                if s > 0.5),
            "missed_critical_steps": missed_steps
        }

    @staticmethod
    def logical_consistency(
            reasoning_steps: List[str]) -> float:
        """
        逻辑一致性：推理步骤之间是否有矛盾。

        使用LLM-as-Judge方法：让另一个LLM检查推理链中是否存在矛盾。
        """
        if len(reasoning_steps) < 2:
            return 1.0

        # 检查每对相邻步骤是否存在矛盾
        consistent_pairs = 0
        total_pairs = len(reasoning_steps) - 1

        for i in range(total_pairs):
            is_consistent = ReasoningEvaluation._check_step_pair(
                reasoning_steps[i], reasoning_steps[i+1]
            )
            if is_consistent:
                consistent_pairs += 1

        return consistent_pairs / total_pairs if total_pairs > 0 else 1.0

    @staticmethod
    def step_efficiency(agent_steps: int,
                        min_required_steps: int) -> float:
        """
        步骤效率：Agent用多少步完成了任务。

        最优效率 = min_required / actual_steps
        1.0表示用最优步数完成
        >0.5表示可以接受
        <0.3表示存在大量冗余步骤
        """
        if agent_steps <= 0:
            return 0.0
        return min(1.0, min_required_steps / agent_steps)

    @staticmethod
    def intermediate_accuracy(
            agent_intermediate_results: List[str],
            oracle_intermediate_results: List[str]) -> float:
        """
        中间结论准确性：中间推理步骤的结果是否正确。

        很多时候Agent最终答案对了但中间推错了（碰巧蒙对）。
        评估中间步骤可以更准确地衡量推理质量。
        """
        if not oracle_intermediate_results:
            return 1.0

        correct = 0
        for i, (agent_res, oracle_res) in enumerate(
            zip(agent_intermediate_results, oracle_intermediate_results)
        ):
            # 使用精确匹配或语义匹配
            if ReasoningEvaluation._is_correct(
                agent_res, oracle_res
            ):
                correct += 1

        return correct / len(oracle_intermediate_results)

    @staticmethod
    def _extract_steps(reasoning: str) -> List[str]:
        """从推理文本中提取步骤"""
        # 按"首先/然后/其次/最后"等连接词切分
        # 或按编号（1. 2. 3.）切分
        import re
        steps = re.split(
            r'(?:^|\n)\s*(?:\d+[\.\)]|第[一二三四五六七八九十]+步[：:]|'
            r'首先|然后|其次|接着|最后|此外|另外)',
            reasoning
        )
        return [s.strip() for s in steps if s.strip()]

    @staticmethod
    def _step_similarity(step_a: str, step_b: str) -> float:
        """计算两个推理步骤的语义相似度"""
        # 实际使用embedding余弦相似度
        words_a = set(step_a.lower().split())
        words_b = set(step_b.lower().split())
        if not words_a or not words_b:
            return 0.0
        overlap = words_a & words_b
        return len(overlap) / max(len(words_a), len(words_b))

    @staticmethod
    def _check_step_pair(step_i: str, step_j: str) -> bool:
        """检查两个推理步骤是否一致"""
        # 使用NLI模型检查是否entailment
        return True  # 简化实现

    @staticmethod
    def _is_correct(agent_res: str, oracle_res: str) -> bool:
        """判断Agent的中间结果是否与标准答案一致"""
        return agent_res.strip().lower() == oracle_res.strip().lower()
```

## 自动化评估 vs 人工评估

### 对比分析

| 维度 | 自动化评估 | 人工评估 |
|------|-----------|---------|
| **速度** | 极快（秒级） | 慢（分钟到小时级） |
| **成本** | 低（API费用） | 高（人力成本） |
| **一致性** | 高（规则或模型确定） | 中（评估者间有差异） |
| **可扩展性** | 极好（可评估大量任务） | 差（人力有限） |
| **细粒度** | 受限（依赖评估设计） | 灵活（可评判任意维度） |
| **开放性任务** | 困难 | 擅长 |
| **主观维度** | 困难（如"回答是否有帮助"） | 擅长 |
| **可复现性** | 完美（相同输入相同输出） | 一般（因人而异） |

### 最佳实践：混合评估

## 评估数据集构建

### 构建原则

```python
class EvaluationDatasetBuilder:
    """
    评估数据集构建器。

    核心原则：
    1. 代表性：覆盖Agent实际使用场景
    2. 多样性：不同难度、类型、领域
    3. 无污染：严格分离训练和评估数据
    4. 可复现：任务定义清晰，输出可判断
    """

    def __init__(self, domain: str):
        self.domain = domain
        self.tasks = []

    def add_task(self, task: dict):
        """添加一个评估任务"""
        required_fields = [
            "task_id", "description",
            "expected_output", "success_criteria",
            "difficulty", "category"
        ]
        for field in required_fields:
            assert field in task, f"Missing required field: {field}"

        self.tasks.append(task)

    def get_statistics(self) -> dict:
        """查看数据集的统计信息"""
        difficulties = {}
        categories = {}
        total_estimated_steps = 0

        for task in self.tasks:
            diff = task["difficulty"]
            difficulties[diff] = difficulties.get(diff, 0) + 1

            cat = task["category"]
            categories[cat] = categories.get(cat, 0) + 1

            total_estimated_steps += task.get("estimated_steps", 1)

        return {
            "total_tasks": len(self.tasks),
            "difficulty_distribution": difficulties,
            "category_distribution": categories,
            "avg_estimated_steps": (total_estimated_steps /
                                   len(self.tasks)
                                   if self.tasks else 0)
        }

    def export_splits(self, train_ratio: float = 0.0,
                      val_ratio: float = 0.2,
                      test_ratio: float = 0.8,
                      seed: int = 42):
        """
        导出训练/验证/测试集划分。

        注意：对于大多数Agent评估，不需要"训练集"——
        因为评估数据不应该被用于训练。
        """
        import random
        random.seed(seed)

        shuffled = list(self.tasks)
        random.shuffle(shuffled)

        n = len(shuffled)
        train_end = int(n * train_ratio)
        val_end = train_end + int(n * val_ratio)

        return {
            "train": shuffled[:train_end],
            "val": shuffled[train_end:val_end],
            "test": shuffled[val_end:]
        }

    @staticmethod
    def difficulty_guidelines():
        """
        难度分级指南。

        Easy: 1-3步可完成，工具不超过2个，操作明确
        Medium: 4-8步，2-4个工具，需要简单条件判断
```

```python
class AgentEvaluationReport:
    """
    Agent综合评估报告生成器。

    输出结构化的评估报告，包含：
    - 总分和各维度得分
    - 按难度/类别的分解得分
    - 典型成功和失败案例
    - 改进建议

    def generate(self,
                 agent_name: str,
                 results: dict,
                 config: EvaluationConfig) -> dict:
        report = {
            "agent": agent_name,
            "evaluation_date": datetime.now().isoformat(),
            "config": config.__dict__,

            # 核心指标
            "overall_score": self._compute_overall(results),

            # 分解得分
            "by_dimension": {
                "task_success_rate": results.get("tsr", 0),
                "tool_accuracy": results.get("ta", 0),
                "reasoning_quality": results.get("rq", 0),
            },

            "by_difficulty": results.get("by_difficulty", {}),
            "by_category": results.get("by_category", {}),

            # 效率指标
            "efficiency": {
                "avg_steps_per_task": results.get("avg_steps", 0),
                "avg_time_per_task": results.get("avg_time", 0),
                "avg_token_per_task": results.get("avg_tokens", 0),
                "avg_cost_per_task": results.get("avg_cost", 0),
            },

            # 可靠性指标
            "reliability": {
                "success_rate_variance":
                    results.get("tsr_variance", 0),
                "timeout_rate": results.get("timeout_rate", 0),
                "error_rate": results.get("error_rate", 0),
            },

            # 典型案例
            "case_study": {
                "best_cases": results.get("best_examples", [])[:3],
                "worst_cases": results.get("worst_examples", [])[:3],
                "failure_analysis": results.get("failure_analysis", {}),
            },

            # 改进建议
            "recommendations": self._generate_recommendations(results),
        }

        return report

    def _compute_overall(self, results: dict) -> float:
        """计算综合得分"""
        weights = {
            "tsr": 0.50,     # 任务成功率权重最高
            "ta": 0.25,      # 工具准确率
            "rq": 0.15,      # 推理质量
            "efficiency": 0.10  # 效率
        }
        score = (
            results.get("tsr", 0) * weights["tsr"] +
            results.get("ta", 0) * weights["ta"] +
            results.get("rq", 0) * weights["rq"] +
            results.get("efficiency", 0) * weights["efficiency"]
        )
        return round(score, 3)

    def _generate_recommendations(self,
                                   results: dict) -> list:
        """基于评估结果生成改进建议"""
        recommendations = []

        if results.get("tsr", 1.0) < 0.7:
            recommendations.append({
                "priority": "high",
                "area": "task_success",
                "suggestion": "任务成功率偏低。建议检查Agent的规划能力"
                             "和工具链是否完整。"
            })

        if results.get("ta", 1.0) < 0.8:
            recommendations.append({
                "priority": "high",
                "area": "tool_accuracy",
                "suggestion": "工具使用准确率不足。建议优化工具描述"
                             "和System Prompt，增加工具使用示例。"
            })

        timeout_rate = results.get("timeout_rate", 0)
        if timeout_rate > 0.05:
            recommendations.append({
                "priority": "medium",
                "area": "timeout",
                "suggestion": f"超时率{timeout_rate:.1%}偏高。"
                             "建议优化Agent的步骤效率或增加超时时间。"
            })

        return recommendations
```

## 评估最佳实践

### Do's and Don'ts

| Do | Don't |
|-----|-------|
| 使用多种指标综合评估 | 只看单一指标（如只看成功率） |
| 按难度和类别分解评估结果 | 只看综合得分 |
| 多次运行取平均值 | 单次运行下结论 |
| 记录所有失败案例进行分析 | 只看数字不看案例 |
| 与baseline和竞品对比 | 孤立地看绝对数值 |
| 定期重新评估（模型在变化） | 评估一次就永远引用 |
| 结合自动化+人工评估 | 完全依赖自动化评估 |

### 评估基础设施

```python
class EvaluationPipeline:
    """
    标准化的Agent评估流水线。
    """

    def __init__(self):
        self.steps = []

    def add_metric(self, metric_fn: Callable, name: str):
        self.steps.append((name, metric_fn))

    def run(self, agent, test_set: List[dict]) -> dict:
        """运行完整评估流水线"""
        all_results = []

        for task in test_set:
            # 对每个任务运行多次
            task_results = []
            for run in range(3):  # 运行3次
                trajectory = agent.execute(task)
                eval_result = {}
                for name, metric_fn in self.steps:
                    eval_result[name] = metric_fn(
                        trajectory, task
                    )
                task_results.append(eval_result)

            # 取平均
            avg_result = {
                k: np.mean([r[k] for r in task_results])
                for k in task_results[0]
            }
            avg_result["variance"] = {
                k: np.var([r[k] for r in task_results])
                for k in task_results[0]
            }
            all_results.append(avg_result)

        return self._aggregate(all_results)
```

## 小结

量化评估Agent的能力是一项系统性工程。它需要的不是一个指标，而是一个**多维度的评估框架**：

1. **任务成功率（TSR）**：能不能完成任务，是最直观的上层指标
2. **工具使用准确率（TA）**：工具选得对不对、参数填得准不准，是中间层能力指标
3. **推理步骤质量（RQ）**：推理链条完不完整、逻辑一不一致，是底层能力指标
4. **效率指标**：用了多少步、多少时间、多少Token，是性价比指标
5. **可靠性指标**：成功率是否稳定、会不会超时，是稳定性指标

**核心原则**：
- 没有完美的指标，只有**合适组合的指标**
- 自动化评估是基础，人工评估是保障
- 分解比汇总更重要——知道"哪里不好"比知道"总分多少"更有价值
- 评估不是一次性的，而是**伴随整个Agent生命周期**的持续过程

记住：**评估不是为了证明Agent"好不好"，而是为了发现Agent"哪里需要改进"**。
