# 幻觉抑制：检索增强与约束解码

## 引言

"幻觉（Hallucination）"是AI Agent最令人头疼的问题之一——模型生成的内容看起来合理、流畅、自信，但实际上是虚构的。一个Agent可能告诉你"某公司2024年Q2的营收增长了23.5%"，数字精确到小数点，语气斩钉截铁，但你去查财报发现根本没有这个数据。

对于聊天机器人，幻觉可能是可容忍的（用户通常会二次核实）。但对于一个被赋予行动能力的Agent，幻觉的后果可能是灾难性的——它可能基于虚构的数据做出错误的商业决策、发送错误的医疗建议、或者执行破坏性的系统操作。

**幻觉抑制**不是要消除幻觉（目前技术上做不到完全消除），而是通过一系列技术手段将幻觉率降低到可接受的水平。本文将从幻觉的成因分析出发，系统介绍检索增强、约束解码、事实一致性验证等核心抑制技术。

## 幻觉的成因分析

### 幻觉分类

```
幻觉分类体系：

1. 内在幻觉（Intrinsic Hallucination）
   - 生成内容与提供的上下文矛盾
   - 例如：上下文说"A公司营收100亿"，但生成"营收150亿"

2. 外在幻觉（Extrinsic Hallucination）
   - 生成内容无法从上下文验证，也不在训练数据中
   - 例如：生成一个完全不存在的产品名称和其"市场数据"

3. 事实性幻觉（Factual Hallucination）
   - 生成的事实与真实世界不符
   - 例如："珠穆朗玛峰高10000米"

4. 逻辑性幻觉（Logical Hallucination）
   - 推理链条中的逻辑错误
   - 例如："因为A>B，且B>C，所以C>A"
```

### 幻觉与自信的悖论

```
幻觉的一个危险特征是：模型对自己生成的内容"看起来"非常自信。

研究显示：
- 模型在生成错误信息时的语言模式与生成正确信息时几乎不可区分
- 模型的内部置信度（log probability）与其实际正确率之间只有弱相关
- 模型更倾向于在"不确定"时生成更长的解释（用篇幅掩盖不确定性）

对策：永远不要依赖模型的"自我评估"来判断其输出是否正确。
```

## 检索增强降低幻觉的机制

RAG（Retrieval-Augmented Generation）是目前最有效的幻觉抑制技术之一。其核心思想简单而强大：**让模型基于真实的外部知识来生成回答，而不是仅依赖内部参数记忆**。

### RAG降低幻觉的机理

```
无RAG：
  问题 → LLM（仅靠参数记忆） → 可能包含幻觉的回答

有RAG：
  问题 → 检索 → 真实文档 → LLM（基于文档生成） → 有依据的回答
                                              ↓
                                   可事后验证（引用来源）
```

RAG从三个层面抑制幻觉：

1. **知识锚定（Knowledge Anchoring）**：为生成提供真实的依据
2. **注意力引导（Attention Guidance）**：通过上下文引导模型关注正确信息
3. **可验证性（Verifiability）**：答案可追溯到具体来源

### RAG实施的最佳实践

```python
class RAGHallucinationReducer:
    """
    基于RAG的幻觉降低系统。

    不只是"检索-生成"那么简单，需要多环节的质量保障。
    """

    def __init__(self, retriever, generator, validator):
        self.retriever = retriever
        self.generator = generator
        self.validator = validator

    def answer_with_grounding(self, question: str) -> dict:
        """
        基于真实文档生成回答，并附带引用。
        """
        # 第1步：检索（高质量检索是基础）
        docs = self.retriever.retrieve(question, top_k=10)
        docs = self._filter_relevant(docs, question)

        if not docs:
            # 没有找到相关文档 → 直接告知，不编造
            return {
                "answer": "我无法找到回答此问题所需的可靠信息。",
                "sources": [],
                "confidence": "low"
            }

        # 第2步：引证生成（Citation-based Generation）
        response = self.generator.generate_with_citations(
            question=question,
            context_docs=docs,
            instruction=(
                "请基于提供的文档回答问题。\n"
                "重要规则：\n"
                "1. 每个事实性陈述后标注来源编号 [1], [2] 等\n"
                "2. 不要添加文档中没有的信息\n"
                "3. 如果不确定，明确说'根据现有资料无法确定'\n"
                "4. 如果文档之间有矛盾，指出矛盾并引用各自的来源"
            )
        )

        # 第3步：幻觉检测（事后验证）
        validation = self.validator.check(
            answer=response["text"],
            citations=response["citations"],
            contexts=[d["content"] for d in docs]
        )

        if validation["hallucination_rate"] > 0.1:
            # 幻觉率过高 → 重新生成或降级
            response = self._regenerate_with_stricter_prompt(
                question, docs, validation["issues"]
            )

        return {
            "answer": response["text"],
            "sources": [d["source"] for d in docs],
            "confidence": validation["confidence"],
            "hallucination_checks": validation
        }

    def _filter_relevant(self, docs, question, threshold=0.6):
        """过滤不相关的检索结果，减少噪音"""
        filtered = []
        for doc in docs:
            relevance = self._compute_relevance(doc["content"], question)
            if relevance >= threshold:
                doc["relevance_score"] = relevance
                filtered.append(doc)
        return filtered

    def _regenerate_with_stricter_prompt(self, question, docs, issues):
        """
        当检测到幻觉时，用更严格的Prompt重新生成。

        明确告诉模型之前生成中的具体问题。
        """
        strict_prompt = (
            f"请重新回答以下问题。之前的回答存在以下问题：\n"
            f"{chr(10).join(f'- {i}' for i in issues)}\n\n"
            f"问题：{question}\n\n"
            f"参考文档：\n{self._format_docs(docs)}\n\n"
            f"要求：严格基于文档内容回答。对于每个陈述，请确保能在"
            f"文档中找到原文支持。不要添加任何文档中没有的信息。"
        )
        return self.generator.generate(strict_prompt)
```

### RAG的局限性

| 局限性 | 说明 | 缓解策略 |
|--------|------|---------|
| 检索质量依赖 | 如果检索不到相关文档，RAG无济于事 | 多源检索 + 查询重写 + HyDE |
| 文档本身可能有误 | 如果知识库中的文档是错误的，生成也会错 | 权威来源优先 + 多源交叉验证 |
| 模型仍可能忽略文档 | 即使检索到了，模型也可能"固执"地使用自己的知识 | 强约束Prompt + 事后验证 |
| 长上下文退化 | 过多的检索文档让模型注意力分散 | 精选top 3-5篇最相关的文档 |
| 延迟增加 | 检索 + 生成 > 纯生成 | 缓存常用检索结果 + 流式响应 |

## 约束解码技术

约束解码（Constrained Decoding）是一类在生成过程中施加约束的技术，确保输出符合预期的格式、内容或事实边界。

### 技术对比

| 技术 | 原理 | 时机 | 效果 |
|------|------|------|------|
| **Logit Manipulation** | 修改Token概率分布，压制不允许的Token | 生成时 | 强 |
| **Grammar-Guided Decoding** | 强迫输出符合指定语法（如JSON Schema） | 生成时 | 强 |
| **Factual Nucleus Sampling** | 降低可能引入幻觉的Token的概率 | 生成时 | 中 |
| **Contrastive Decoding** | 对比专家模型和业余模型的输出，选择两者差异大的Token | 生成时 | 中 |
| **Self-Consistency** | 多次采样取最一致的答案 | 生成后 | 中 |
| **Constitutional AI约束** | 在Prompt中嵌入行为约束规则 | 生成前 | 弱-中 |

### 实现示例

```python
import torch
import torch.nn.functional as F


class ConstrainedDecoder:
    """
    约束解码器：在生成过程中控制输出。
    """

    @staticmethod
    def factual_consistency_constraint(
            logits: torch.Tensor,
            context_tokens: list[int],
            tokenizer) -> torch.Tensor:
        """
        事实一致性约束：降低可能导致幻觉的Token概率。

        原理：如果上下文中有确定性的信息，就压制与之矛盾的Token。
        例如：上下文说"颜色是红色"，就降低"蓝色""绿色"等Token的概率。
        """
        # 从上下文中提取关键实体和属性
        context_entities = ConstrainedDecoder._extract_entities(
            tokenizer.decode(context_tokens)
        )

        # 对于每个关键实体，压制可能矛盾的Token
        modified_logits = logits.clone()

        for entity, value in context_entities.items():
            # 获取矛盾的Token IDs
            contradictory_tokens = ConstrainedDecoder._get_contradictory_tokens(
                entity, value, tokenizer
            )

            # 大幅降低矛盾Token的概率
            for token_id in contradictory_tokens:
                modified_logits[token_id] -= 100.0  # 强力压制

        return modified_logits

    @staticmethod
    def citation_aware_decoding(
            logits: torch.Tensor,
            beam_tokens: list,
            required_citations: list[str]) -> torch.Tensor:
        """
        引用感知解码：强制生成包含正确来源引用的内容。

        当模型需要生成事实性陈述时，强制其引用来源。
        """
        modified_logits = logits.clone()

        # 检查当前生成位置是否需要引用
        if ConstrainedDecoder._should_insert_citation(beam_tokens):
            # 大幅提升引用标记Token的概率
            citation_start_token = tokenizer.encode("[来源")[0]
            modified_logits[citation_start_token] += 50.0

            # 压制非引用Token（在需要引用的位置）
            for token_id in range(len(modified_logits)):
                if token_id != citation_start_token:
                    modified_logits[token_id] -= 10.0

        return modified_logits

    @staticmethod
    def uncertainty_aware_decoding(
            logits: torch.Tensor,
            entropy_threshold: float = 2.0) -> torch.Tensor:
        """
        不确定性感知解码：当模型不确定时，引导它表达不确定性。

        原理：计算Token分布的熵。如果熵很高（模型不确定），
        提升"可能""大约""据估计"等表达不确定性的Token概率。
        """
        probs = F.softmax(logits, dim=-1)
        entropy = -(probs * torch.log(probs + 1e-10)).sum()

        if entropy > entropy_threshold:
            # 模型处于不确定状态
            modified_logits = logits.clone()

            # 提升"不确定"相关的Token
            uncertainty_tokens = [
                "可能", "大约", "估计", "据", "或许", "大概",
                "可能", "approximately", "estimated", "roughly"
            ]
            for token_str in uncertainty_tokens:
                token_ids = tokenizer.encode(token_str)
                for tid in token_ids:
                    modified_logits[tid] += 3.0

            return modified_logits

        return logits

    @staticmethod
    def _extract_entities(text: str) -> dict:
        """从文本中提取关键实体-属性对"""
        # 实际使用NER模型提取
        # 示例：{"颜色": "红色", "价格": "100元"}
        entities = {}
        # ... NER提取逻辑
        return entities

    @staticmethod
    def _get_contradictory_tokens(entity: str, value: str,
                                   tokenizer) -> list[int]:
        """获取与给定值矛盾的其他可选值Token"""
        # 通过知识图谱或预先构建的反义词表
        contradiction_map = {
            "红色": ["蓝色", "绿色", "黄色", "黑色"],
            "是": ["不是", "否"],
            "正确": ["错误", "不正确"],
        }
        contradict_values = contradiction_map.get(value, [])
        token_ids = []
        for val in contradict_values:
            token_ids.extend(tokenizer.encode(val))
        return token_ids

    @staticmethod
    def _should_insert_citation(beam_tokens: list) -> bool:
```

## 事实一致性验证

### 验证框架

```python
class FactualConsistencyVerifier:
    """
    事实一致性验证器。

    验证Agent生成的回答是否与给定的上下文一致。
    这是RAG系统中最重要的质量保障环节。
    """

    def __init__(self, nli_model=None):
        """
        nli_model: 自然语言推理模型（用于判断声明是否被支撑）
        可以使用专门的NLI模型或LLM-as-Judge。
        """
        self.nli_model = nli_model

    def verify(self, answer: str,
               contexts: list[str]) -> dict:
        """
        验证回答的事实一致性。

        返回每个声明的验证结果和整体幻觉率。
        """
        # 1. 拆分回答为原子声明
        claims = self._decompose_into_claims(answer)
        print(f"拆分出 {len(claims)} 个原子声明")

        # 2. 对每个声明进行事实验证
        verification_results = []
        for claim in claims:
            result = self._verify_claim(claim, contexts)
            verification_results.append(result)

        # 3. 汇总并生成报告
        return self._generate_report(verification_results)

    def _decompose_into_claims(self, answer: str) -> list[dict]:
        """
        将回答拆分为独立的原子声明。

        例如：
        "Apple公司成立于1976年，总部位于加州Cupertino"
        → [
            {"text": "Apple公司成立于1976年", "type": "factual"},
            {"text": "Apple总部位于加州Cupertino", "type": "factual"}
          ]
        """
        # 实际使用LLM进行拆分
        prompt = f"""
        将以下文本拆分为独立的原子声明。每个声明应该是一个
        完整的、可以独立验证的事实陈述。

        文本：{answer}

        返回JSON格式：
        {{
            "claims": [
                {{"text": "声明内容", "type": "factual/opinion/definition"}}
            ]
        }}
        """
        # response = self.llm.generate(prompt)
        # return json.loads(response)
        pass

    def _verify_claim(self, claim: dict,
                       contexts: list[str]) -> dict:
        """
        验证单个声明是否被上下文支撑。

        使用NLI（自然语言推理）判断关系：
        - Entailment (蕴含)：上下文支持该声明 → 无幻觉
        - Contradiction (矛盾)：上下文反对该声明 → 幻觉
        - Neutral (中立)：上下文不涉及该声明 → 可能幻觉
        """
        context_text = "\n".join(contexts)

        # 使用NLI模型判断
        # nli_result = self.nli_model.predict(
        #     premise=context_text,
        #     hypothesis=claim["text"]
        # )

        # 简化示例
        nli_result = self._simulate_nli(context_text, claim["text"])

        is_supported = (nli_result == "entailment")

        return {
            "claim": claim["text"],
            "verdict": "supported" if is_supported else "unsupported",
            "nli_label": nli_result,
            "confidence": 0.85 if is_supported else 0.3,
            "evidence": self._find_evidence(context_text, claim["text"])
        }

    def _generate_report(self, results: list[dict]) -> dict:
        """生成验证报告"""
        total = len(results)
        supported = sum(1 for r in results
                       if r["verdict"] == "supported")
        unsupported = total - supported

        hallucination_rate = unsupported / total if total > 0 else 0

        report = {
            "total_claims": total,
            "supported_claims": supported,
            "unsupported_claims": unsupported,
            "hallucination_rate": hallucination_rate,
            "confidence": "high" if hallucination_rate < 0.05
                     else "medium" if hallucination_rate < 0.15
                     else "low",
            "details": results,
            "flagged_claims": [
                r for r in results
                if r["verdict"] == "unsupported"
            ]
        }

        return report

    def _simulate_nli(self, premise: str,
                       hypothesis: str) -> str:
        """模拟NLI判断（实际应使用真正的NLI模型）"""
        # 简单实现：基于关键词重叠
        hypothesis_words = set(hypothesis.lower().split())
        premise_words = set(premise.lower().split())
        overlap = len(hypothesis_words & premise_words)
        total = len(hypothesis_words)
        ratio = overlap / total if total > 0 else 0

        if ratio > 0.7:
            return "entailment"
        elif ratio < 0.2:
            return "contradiction"
        else:
            return "neutral"

    def _find_evidence(self, context: str,
                        claim: str) -> str:
        """在上下文中找到支撑或反对声明的证据文本"""
        # 将上下文按句子拆分，找到与声明最相关的句子
        sentences = context.replace("。", ".").split(".")
        # 使用embedding相似度找最相关的句子
        best_sentence = max(
            sentences,
            key=lambda s: self._similarity(s, claim),
            default=""
        )
        return best_sentence.strip()

    def _similarity(self, text_a: str, text_b: str) -> float:
        """文本相似度计算"""
        words_a = set(text_a.lower().split())
        words_b = set(text_b.lower().split())
        overlap = words_a & words_b
        union = words_a | words_b
        return len(overlap) / len(union) if union else 0
```

## RAG + Self-Reflection 组合策略

自我反思（Self-Reflection）是让Agent"检查自己的作业"——生成回答后，再次审视和修正。

```python
class RAGWithSelfReflection:
    """
    RAG + 自我反思 组合策略。

    流程：
    RAG生成 → 自我检查 → 发现问题 → 修正 → 最终输出
    """

    def __init__(self, retriever, generator):
        self.retriever = retriever
        self.generator = generator
        self.verifier = FactualConsistencyVerifier()

    def answer_with_reflection(self,
                                question: str,
                                max_reflection_rounds: int = 2) -> dict:
        """
        带自我反思的RAG回答流程。
        """
        # Step 1: RAG生成初始回答
        docs = self.retriever.retrieve(question, top_k=5)
        initial_answer = self.generator.generate(
            question=question,
            contexts=[d["content"] for d in docs]
        )

        current_answer = initial_answer["text"]

        for round_num in range(max_reflection_rounds):
            # Step 2: 自我检查
            reflection = self._self_reflect(
                question, current_answer,
                [d["content"] for d in docs]
            )

            # Step 3: 判断是否需要修正
            if not reflection["needs_correction"]:
                print(f"第{round_num+1}轮反思：无需修正，答案通过")
                break

            print(f"第{round_num+1}轮反思：发现{len(reflection['issues'])}个问题")

            # Step 4: 修正
            current_answer = self._correct_answer(
                question, current_answer,
                [d["content"] for d in docs],
                reflection["issues"]
            )

        # Step 5: 最终验证
        final_verification = self.verifier.verify(
            current_answer, [d["content"] for d in docs]
        )

        return {
            "answer": current_answer,
            "initial_answer": initial_answer["text"],
            "reflection_rounds": round_num + 1,
            "verification": final_verification,
            "sources": [d["source"] for d in docs]
        }

    def _self_reflect(self, question: str, answer: str,
                       contexts: list[str]) -> dict:
        """
        自我反思：让模型检查自己的回答。

        关键提示：
        - 对比回答与上下文的一致性
        - 识别没有依据的声明
        - 检查数字、日期、名称的准确性
        - 判断是否有过度解读
        """
        reflection_prompt = f"""
        你是一个事实检查员。请检查以下回答是否严格基于提供的文档。

        问题：{question}

        提供的上下文文档：
        {chr(10).join(f'[文档{i+1}] {c}' for i, c in enumerate(contexts))}

        待检查的回答：
        {answer}

        请逐条检查并回答：
        1. 回答中哪些陈述可以在文档中找到直接依据？
        2. 哪些陈述是文档中没有的（可能是编造的）？
        3. 是否有数字、日期、名称与文档不一致？
        4. 回答是否过度解读了文档内容？
        5. 总体评估：此回答是否需要修正？

        返回JSON格式：
        {{
            "needs_correction": true/false,
            "confidence_score": 0.0-1.0,
            "issues": [
                {{
                    "location": "回答中的具体位置",
                    "issue_type": "unsupported_claim/incorrect_number/over_interpretation",
                    "description": "问题描述",
                    "suggested_correction": "建议修正"
                }}
            ]
        }}
        """
        # response = self.generator.generate(reflection_prompt)
        # return json.loads(response)
        pass

    def _correct_answer(self, question: str,
                          original_answer: str,
                          contexts: list[str],
                          issues: list[dict]) -> str:
        """
        根据反思发现的问题修正回答。
        """
        correction_prompt = f"""
        请修正以下回答。原始回答中存在以下问题：

        {chr(10).join(f'{i+1}. {issue["description"]} (建议: {issue.get("suggested_correction", "删除无据陈述")})'
                      for i, issue in enumerate(issues))}

        原始问题：{question}

        上下文文档：
        {chr(10).join(f'[文档{i+1}] {c}' for i, c in enumerate(contexts))}

        原始回答：
        {original_answer}

        请提供修正后的回答。修正要求：
        - 只保留有文档依据的内容
        - 对不确定的内容标注"不确定"或"据现有资料无法确认"
        - 保持回答的流畅性和完整性
        """
        response = self.generator.generate(correction_prompt)
```

## 幻觉抑制效果评估

```python
class HallucinationSuppressionEvaluator:
    """
    幻觉抑制效果评估器。

    对比使用/不使用各种抑制技术前后的幻觉率变化。
    """

    def __init__(self):
        self.baseline_rate = None

    def set_baseline(self, hallucination_rate: float):
        """设置基线（不使用任何抑制技术）"""
        self.baseline_rate = hallucination_rate

    def evaluate_technique(self, technique_name: str,
                            hallucination_rate: float) -> dict:
        """评估单一抑制技术的效果"""
        reduction = ((self.baseline_rate - hallucination_rate) /
                     self.baseline_rate
                     if self.baseline_rate else 0)

        return {
            "technique": technique_name,
            "baseline_rate": self.baseline_rate,
            "suppressed_rate": hallucination_rate,
            "reduction_pct": reduction * 100,
            "effectiveness": "excellent" if reduction > 0.6
                        else "good" if reduction > 0.3
                        else "moderate" if reduction > 0.1
                        else "marginal"
        }

    def compare_techniques(self,
                            results: dict[str, float]) -> pd.DataFrame:
        """
        比较多组抑制技术效果。

        典型结果：
        | 技术 | 基线幻觉率 | 抑制后幻觉率 | 降低幅度 |
        |------|-----------|-------------|---------|
        | RAG  | 35%       | 12%         | 65.7%   |
        | RAG+约束解码 | 35% | 8%     | 77.1%   |
        | RAG+SelfRefl | 35% | 5%     | 85.7%   |
        | 全组合 | 35%  | 3%          | 91.4%   |
        """
        data = []
        for name, rate in results.items():
            data.append(self.evaluate_technique(name, rate))
        return data
```

## 小结

幻觉是AI Agent的内在属性，根植于LLM的概率生成本质。我们无法完全消除它，但可以通过系统性的技术手段将其抑制到可接受的水平。

**幻觉抑制技术层次**：

```
第0层：承认问题 → 告诉Agent"可以说不知道"
第1层：检索增强(RAG) → 用真实知识锚定生成（最重要）
第2层：约束解码 → 在生成过程中修正概率分布
第3层：事实验证 → 事后检查，发现并标记幻觉
第4层：自我反思 → 让Agent自己检查和修正（元认知）
第5层：人工审核 → 关键场景下人的最终判断
```

**实践建议**：
1. **从RAG开始**：RAG是性价比最高的幻觉抑制技术，优先实施
2. **分层防御**：不要依赖单一技术，而是要RAG + 约束解码 + 事后验证 + 反思的组合
3. **关键场景加人工**：医疗、金融、法律等高风险场景，必须有人工审核环节
4. **渐进式改进**：从最容易实施的（RAG）开始，逐步叠加更复杂的技术
5. **持续监控**：幻觉率要作为系统核心监控指标，持续跟踪和优化

记住：**一个诚实的"我不知道"，比一个自信的幻觉好一万倍**。设计Agent时，永远为"不确定性"留有余地。
