## 概述

在人工智能规划（AI Planning）领域，**状态空间搜索（State Space Search）** 和 **分层规划（Hierarchical Planning）** 是两种核心范式。状态空间搜索通过在状态图中探索从初始状态到目标状态的路径来求解问题；而分层规划则将复杂目标分解为高层任务，再逐步细化到可执行的原子动作，从而应对大规模问题的组合爆炸。

本文从原理、复杂度、适用场景三个维度深入讲解 BFS、DFS、A* 搜索算法，并结合 HTN（Hierarchical Task Network）框架展示分层规划的完整实现。

## 状态空间搜索

### 核心概念

在 AI 规划中，一个问题可以被定义为一个 **状态空间（State Space）** 元组：

```
(S, s₀, G, A, f)
```

| 符号 | 含义 | 示例 |
|------|------|------|
| `S`  | 所有可能状态的集合（State Set） | 地图上所有城市 |
| `s₀` | 初始状态（Initial State） | 出发城市（北京） |
| `G`  | 目标状态集合（Goal States） | 目标城市（上海） |
| `A`  | 可用动作集合（Actions） | 城市之间的交通方式 |
| `f`  | 状态转移函数（Transition Function） | `f(北京, 高铁) = 上海` |

搜索算法的任务是在状态空间中找出一条从 `s₀` 到某个 `g ∈ G` 的路径。

### BFS（Breadth-First Search，广度优先搜索）

#### 原理

BFS 按层级逐层扩展节点：首先访问初始状态的所有直接后继，然后访问后继的后继，依此类推。BFS 使用 **FIFO 队列（Queue）** 管理待扩展节点。

```
         s₀           ← 第 0 层
       /  |  \
      a   b   c       ← 第 1 层
     / \  |  / \
    d  e  f g   h     ← 第 2 层
```

**ASCII 流程示意：**

```
Queue: [s₀]
Step1: pop s₀ → expand → push [a, b, c]     (visited: s₀)
Step2: pop a  → expand → push [d, e]         (visited: s₀, a)
Step3: pop b  → expand → push [f]            (visited: s₀, a, b)
Step4: pop c  → expand → push [g, h]         (visited: s₀, a, b, c)
... 逐层进行直到找到目标
```

#### 性质

| 性质 | 值 | 说明 |
|------|-----|------|
| **完备性（Completeness）** | 完备 | 如果解存在，BFS 一定能找到 |
| **最优性（Optimality）** | 最优（单位代价） | 若每条边代价相同，BFS 找到的路径最短 |
| **时间复杂度** | `O(b^d)` | `b` = 分支因子，`d` = 解的深度 |
| **空间复杂度** | `O(b^d)` | 需要存储每一层所有节点 |
| **适用场景** | 状态空间小、每条边代价相等、需最短路径 |

#### 适用场景

- 迷宫最短路径
- 社交网络中查找最短关系链（如六度分隔）
- 无权图中的最短路径问题
- 状态空间较小且需保证找到最优解

#### 局限

当分支因子 `b` 较大时，空间需求急剧膨胀。例如 `b=10, d=10` 时约需存储 `10^10` 个节点，内存不可承受。

---

### DFS（Depth-First Search，深度优先搜索）

#### 原理

DFS 沿着一条路径尽可能深入，直到无法继续时回溯（Backtracking）。使用 **LIFO 栈（Stack）** 或递归实现。

```
        s₀
       /
      a           ← 优先深入这条路径
     /
    d            ← 继续深入
   /
  g (goal?)     ← 到达最深处，若未找到则回溯
```

**ASCII 流程示意：**

```
Stack: [s₀]
Step1: pop s₀ → expand → push [a, b, c]      (栈顶是 c，但会先 pop a)
Step2: pop a  → expand → push [d, e]          (深入路径 a)
Step3: pop d  → expand → push [...]           (继续深入)
...
若遇到死胡同 → 回溯到上一个有未探索分支的节点
```

#### 性质

| 性质 | 值 | 说明 |
|------|-----|------|
| **完备性（Completeness）** | 不完备（无限空间） | 在无限状态空间中可能永不返回 |
| **最优性（Optimality）** | 非最优 | 找到的路径不一定最短 |
| **时间复杂度** | `O(b^m)` | `m` = 最大深度（可能远大于 `d`） |
| **空间复杂度** | `O(b·m)` | 只需存储当前路径上的节点 |
| **适用场景** | 空间受限、解很深、不需要最优解 |

#### 适用场景

- 约束满足问题（CSP，Constraint Satisfaction Problems）
- 拓扑排序
- 需要探索所有可能性的场景
- 内存受限环境

---

### Dijkstra 算法

#### 原理

Dijkstra 算法处理 **带权图（Weighted Graph）** 上的最短路径问题。使用 **优先队列（Priority Queue）** 维护从起点到各节点的当前最短距离估计，每次取出距离最小的节点进行松弛（Relaxation）操作。

**核心思想：**

```
dist[v] = min(dist[v], dist[u] + weight(u, v))
```

**ASCII 流程示意：**

```
初始: dist[s₀]=0, 其他=∞
优先队列: [(0, s₀)]

Step1: pop s₀ (dist=0) → 更新邻居:
       dist[a]=5, dist[b]=2, dist[c]=6
       队列: [(2,b), (5,a), (6,c)]

Step2: pop b (dist=2) → 更新邻居:
       dist[d]=2+3=5
       队列: [(5,a), (5,d), (6,c)]

Step3: 继续弹出距离最小的节点...
       直到队列为空或找到目标
```

#### 性质

| 性质 | 值 | 说明 |
|------|-----|------|
| **完备性** | 完备 | 只要目标从起点可达 |
| **最优性** | 最优 | 找到的路径代价最小 |
| **时间复杂度** | `O((V+E)·log V)` | 使用二叉堆的优先队列 |
| **空间复杂度** | `O(V)` | 存储所有节点的距离和路径 |

---

### A* 算法（A-Star Search）

#### 原理

A* 是 Dijkstra 的启发式增强版本，使用评估函数 `f(n) = g(n) + h(n)` 来引导搜索方向：

- **`g(n)`**：从起点到节点 `n` 的实际代价（Actual Cost）
- **`h(n)`**：从节点 `n` 到目标的**启发式估计代价（Heuristic）**
- **`f(n)`**：经过节点 `n` 的估计总代价

**关键性质：**

```
若 h(n) 是可接受的（Admissible）——即从不高估实际代价——
则 A* 保证找到最优解。
```

**ASCII 搜索对比示意：**

```
Dijkstra (无启发):           A* (有启发):
      ╱│╲                        →→→→→
    ╱  │  ╲                    ↗     ↓
  ╱    │    ╲                ↗       ↓
s₀─────┼──────g             s₀────────g
  ╲    │    ╱                ↘       ↑
    ╲  │  ╱                    ↘     ↑
      ╲│╱                        →→→→→
  (向所有方向扩展)            (向目标方向优先扩展)
```

#### 性质

| 性质 | 值 | 说明 |
|------|-----|------|
| **完备性** | 完备 | 若状态空间有限且解存在 |
| **最优性** | 最优（Admissible Heuristic） | 启发函数必须不高估 |
| **时间复杂度** | `O(b^d)`（最坏） | 好的启发式可大幅减少 |
| **空间复杂度** | `O(b^d)` | 需存储所有已生成节点 |
| **适用场景** | 带权图最短路径、游戏 AI 寻路 | 需设计合理启发式函数 |

#### 常见启发式函数（Heuristic Functions）

| 领域 | 启发式 | 公式 | 可接受性 |
|------|--------|------|----------|
| 网格地图 | 曼哈顿距离（Manhattan） | `|x₁-x₂|+|y₁-y₂|` | 可接受（无对角线移动时） |
| 网格地图 | 欧几里得距离（Euclidean） | `√((x₁-x₂)²+(y₁-y₂)²)` | 可接受 |
| 网格地图 | 切比雪夫距离（Chebyshev） | `max(|x₁-x₂|,|y₁-y₂|)` | 可接受（允许对角线移动） |
| 15-Puzzle | 错位方块数 | 统计不在目标位置的方块数 | 可接受 |
| 15-Puzzle | 曼哈顿距离和 | 每个方块到目标位置的曼哈顿距离之和 | 可接受且更紧 |
| 路线规划 | 大圆距离（Great-circle） | 球面最短弧长 | 可接受 |

## 分层规划概念

### 为什么需要分层规划

在面对复杂问题时，直接在原子动作级别进行状态空间搜索会遭遇 **组合爆炸（Combinatorial Explosion）**：

- **旅行规划**：从北京到上海，可能的中间城市组合多达数百万种
- **机器人任务规划**：让机器人"做一杯咖啡"涉及数十个原子动作
- **自动驾驶**：从 A 到 B 的路径规划需要考虑道路网络的所有节点

**分层规划的核心思想：**

```
高层抽象目标
    │
    ├── 子目标 A （抽象）
    │     ├── 子子目标 A1 （具体）
    │     └── 子子目标 A2 （具体）
    │
    ├── 子目标 B （抽象）
    │     ├── 子子目标 B1 （具体）
    │     └── 子子目标 B2 （具体）
    │
    └── ...
```

通过将大问题分解为子问题，每个子问题再递归分解，直到每个子问题都变成可以直接执行的**原子动作（Primitive Action）**。

### 分层规划的层次结构

```
┌─────────────────────────────────────────────────────────┐
│  Level 3 (战略层):  "去欧洲旅行 7 天"                     │
│    → 解决：目的地选择、大日程安排                          │
├─────────────────────────────────────────────────────────┤
│  Level 2 (战术层):  "Day 1-3: 巴黎"                      │
│    → 解决：城市内交通、景点顺序                            │
├─────────────────────────────────────────────────────────┤
│  Level 1 (操作层):  "上午 9:00 前往埃菲尔铁塔"             │
│    → 解决：具体交通方式、购票、步行路线                     │
├─────────────────────────────────────────────────────────┤
│  Level 0 (原子动作):  "走出酒店"、"乘坐地铁 M6 线"         │
│    → 不可再分的基本动作                                   │
└─────────────────────────────────────────────────────────┘
```

### 分层规划的价值

| 优势 | 说明 |
|------|------|
| **降低复杂度** | 每层只需处理有限的选择，避免组合爆炸 |
| **可重用性** | 高层子任务（如"预订酒店"）可在不同场景中重用 |
| **可解释性** | 规划结果的层次结构符合人类思维方式 |
| **容错和重规划** | 某层失败时，只需重新规划该层，不影响其他层 |
| **并行化** | 不同分支的分解可以并行处理 |

## HTN 基础

### 什么是 HTN（Hierarchical Task Network）

HTN 是分层规划中最经典的形式化框架。在 HTN 中：

- **任务（Task）** 分为两类：
  - **复合任务（Compound Task）**：需要通过分解来完成的非原子任务
  - **原子任务（Primitive Task）**：可以直接执行的动作

- **方法（Method）**：定义如何将复合任务分解为子任务集合
- **算子（Operator）**：定义原子任务的前提条件（Preconditions）和效果（Effects）

### HTN 核心组件

```
HTN-Planner = (S, T, O, M)
```

| 组件 | 符号 | 含义 |
|------|------|------|
| **状态（State）** | `S` | 当前世界状态的描述（谓词集合） |
| **任务网络（Task Network）** | `T` | 初始复合任务 + 约束 |
| **算子集合（Operators）** | `O` | 原子动作的定义 |
| **方法集合（Methods）** | `M` | 复合任务的分解规则 |

### HTN 规划流程

```
              开始
               │
               ▼
     ┌─────────────────────┐
     │  当前任务队列 TQ     │
     │  初始：[RootTask]    │
     └─────────┬───────────┘
               │
               ▼
     ┌─────────────────────┐     是
     │  TQ 为空？          │─────────▶  规划成功 ✓
     └─────────┬───────────┘
               │ 否
               ▼
     ┌─────────────────────┐
     │  取出队首任务 t      │
     └─────────┬───────────┘
               │
               ▼
     ┌─────────────────────┐     原子任务
     │  t 是原子任务？      │─────────▶  执行 → 更新状态 → 回到循环
     └─────────┬───────────┘
               │ 复合任务
               ▼
     ┌─────────────────────┐
     │  查找可用的 Method   │
     │  (前提条件满足？)     │
     └─────────┬───────────┘
               │
          ┌────┴────┐
          │         │
       找到方法   无可用方法
          │         │
          ▼         ▼
    将子任务插入   回溯（Backtrack）
    TQ 队首         或 规划失败 ✗
          │
          └──▶ 回到循环
```

### HTN vs 经典规划

| 维度 | 经典规划（STRIPS-style） | HTN 规划 |
|------|--------------------------|----------|
| **搜索空间** | 状态空间 | 任务分解空间 |
| **知识表示** | 仅靠算子的前提条件/效果 | 算子的前提条件/效果 + 方法的分解知识 |
| **控制策略** | 搜索算法决定（如 A*） | 方法定义分解偏好（领域知识驱动） |
| **适用范围** | 通用 | 结构化领域（旅行、制造、军事） |
| **效率** | 易组合爆炸 | 领域知识削减搜索空间，高效 |
| **灵活性** | 高 | 受方法定义的约束，需专家知识 |

## 完整代码实现

### 1. 通用状态表示与搜索框架

```python
"""
状态空间搜索框架 + 分层规划器实现
=====================================
本模块包含：
  1. 通用图搜索算法的实现 (BFS, DFS, Dijkstra, A*)
  2. HTN 风格的分层任务网络规划器
  3. 旅行行程规划的实际案例

作者：AI Planning Tutorial
语言：Python 3.8+
依赖：heapq (标准库), collections (标准库), typing (标准库)
"""

from __future__ import annotations
from abc import ABC, abstractmethod
from collections import deque
from dataclasses import dataclass, field
from enum import Enum, auto
from typing import (
    Any, Callable, Dict, Generic, List, Optional, Set, Tuple, TypeVar,
)
import heapq
import math


# ============================================================
# 第一部分：通用类型定义
# ============================================================

# 泛型状态类型
StateT = TypeVar("StateT")

# 动作类型：接受一个状态，返回一个 (新状态, 代价) 的迭代器
Action = Callable[[StateT], Any]

# 启发式函数类型：接受状态，返回预估代价
Heuristic = Callable[[StateT], float]

# 目标检测函数类型
GoalTest = Callable[[StateT], bool]


# ============================================================
# 第二部分：通用状态空间搜索算法实现
# ============================================================

@dataclass(order=True)
class _SearchNode(Generic[StateT]):
    """
    搜索树中的节点。

    Attributes:
        cost: 从起点经过此节点的总代价 (用于优先队列排序)
        state: 当前状态
        parent: 父节点 (用于回溯路径)
        action_name: 到达此状态所执行的动作名称
        depth: 从起点到当前节点的深度
    """
    cost: float
    state: Any = field(compare=False)
    parent: Optional["_SearchNode"] = field(compare=False, default=None)
    action_name: str = field(compare=False, default="")
    depth: int = field(compare=False, default=0)


class SearchResult(Generic[StateT]):
    """
    搜索结果封装类。

    Attributes:
        found: 是否找到解
        path: 从初始状态到目标的状态序列
        actions: 到达目标的动作序列
        cost: 路径总代价
        nodes_expanded: 扩展的节点总数
    """
    def __init__(self) -> None:
        self.found: bool = False
        self.path: List[StateT] = []
        self.actions: List[str] = []
        self.cost: float = float("inf")
        self.nodes_expanded: int = 0

    def __repr__(self) -> str:
        if not self.found:
            return "SearchResult(found=False)"
        return (
            f"SearchResult(found=True, cost={self.cost:.2f}, "
            f"path_length={len(self.path)}, expanded={self.nodes_expanded})"
        )


def reconstruct_path(node: _SearchNode) -> Tuple[List[Any], List[str]]:
    """
    从目标节点回溯，重建完整路径和动作序列。

    Args:
        node: 目标节点

    Returns:
        (状态序列, 动作名称序列)
    """
    state_path: List[Any] = []
    action_path: List[str] = []
    current: Optional[_SearchNode] = node
    while current is not None:
        state_path.append(current.state)
        if current.action_name:
            action_path.append(current.action_name)
        current = current.parent
    state_path.reverse()
    action_path.reverse()
    return state_path, action_path


def bfs_search(
    start_state: StateT,
    is_goal: GoalTest,
    get_successors: Callable[[StateT], List[Tuple[StateT, float, str]]],
    max_depth: int = 1000,
) -> SearchResult:
    """
    广度优先搜索 (BFS)。

    使用 FIFO 队列逐层扩展节点，保证在无权图中找到最短路径（最少步数）。

    Args:
        start_state: 初始状态
        is_goal: 目标状态判定函数
        get_successors: 后继状态生成函数，返回 [(新状态, 代价, 动作名), ...]
        max_depth: 最大搜索深度

    Returns:
        SearchResult 对象
    """
    result = SearchResult()
    # FIFO 队列：存储待扩展节点
    frontier: deque[_SearchNode] = deque()
    # visited 集合：避免重复访问
    visited: Set[Any] = set()

    root = _SearchNode(cost=0, state=start_state, depth=0)
    frontier.append(root)
    visited.add(start_state)

    while frontier:
        node = frontier.popleft()  # FIFO: 从队首取出
        result.nodes_expanded += 1

        # 目标检测
        if is_goal(node.state):
            result.found = True
            result.path, result.actions = reconstruct_path(node)
            result.cost = node.depth  # BFS 中每步代价为 1
            return result

        # 深度限制
        if node.depth >= max_depth:
            continue

        # 扩展后继
        for next_state, _cost, action_name in get_successors(node.state):
            if next_state not in visited:
                visited.add(next_state)
                child = _SearchNode(
                    cost=node.depth + 1,
                    state=next_state,
                    parent=node,
                    action_name=action_name,
                    depth=node.depth + 1,
                )
                frontier.append(child)

    return result  # 未找到解


def dfs_search(
    start_state: StateT,
    is_goal: GoalTest,
    get_successors: Callable[[StateT], List[Tuple[StateT, float, str]]],
    max_depth: int = 1000,
) -> SearchResult:
    """
    深度优先搜索 (DFS)。

    使用 LIFO 栈沿一条路径深入到底，然后回溯。
    优势：空间复杂度低 O(b·m)。
    劣势：不保证最优解，在无限状态空间中可能不完备。

    Args:
        start_state: 初始状态
        is_goal: 目标判定函数
        get_successors: 后继生成函数
        max_depth: 最大搜索深度限制

    Returns:
        SearchResult 对象
    """
    result = SearchResult()
    # LIFO 栈
    frontier: List[_SearchNode] = []
    visited: Set[Any] = set()

    root = _SearchNode(cost=0, state=start_state, depth=0)
    frontier.append(root)

    while frontier:
        node = frontier.pop()  # LIFO: 从栈顶取出
        result.nodes_expanded += 1

        # 跳过已访问节点 (DFS 可能通过不同路径到达同一节点)
        if node.state in visited:
            continue
        visited.add(node.state)

        if is_goal(node.state):
            result.found = True
            result.path, result.actions = reconstruct_path(node)
            result.cost = node.depth
            return result

        if node.depth >= max_depth:
            continue

        # 将后继压入栈 (逆序压入以保证原序)
        successors = get_successors(node.state)
        for next_state, _cost, action_name in reversed(successors):
            if next_state not in visited:
                child = _SearchNode(
                    cost=node.depth + 1,
                    state=next_state,
                    parent=node,
                    action_name=action_name,
                    depth=node.depth + 1,
                )
                frontier.append(child)

    return result


def dijkstra_search(
    start_state: StateT,
    is_goal: GoalTest,
    get_successors: Callable[[StateT], List[Tuple[StateT, float, str]]],
) -> SearchResult:
    """
    Dijkstra 最短路径算法。

    适用于带权图，保证找到代价最小的路径。
    使用优先队列维护当前最短距离估计。

    Args:
        start_state: 初始状态
        is_goal: 目标判定函数
        get_successors: 后继生成函数 (状态, 代价, 动作名)

    Returns:
        SearchResult 对象
    """
    result = SearchResult()
    # 优先队列：按累计代价排序
    frontier: List[_SearchNode] = []
    # best_cost: 记录到达每个已知状态的最小代价
    best_cost: Dict[Any, float] = {}

    root = _SearchNode(cost=0, state=start_state, depth=0)
    heapq.heappush(frontier, root)
    best_cost[start_state] = 0

    while frontier:
        node = heapq.heappop(frontier)  # 弹出最小代价节点
        result.nodes_expanded += 1

        # 如果已有更优路径到达此状态，跳过
        if node.cost > best_cost.get(node.state, float("inf")):
            continue

        if is_goal(node.state):
            result.found = True
            result.path, result.actions = reconstruct_path(node)
            result.cost = node.cost
            return result

        for next_state, step_cost, action_name in get_successors(node.state):
            new_cost = node.cost + step_cost
            if new_cost < best_cost.get(next_state, float("inf")):
                best_cost[next_state] = new_cost
                child = _SearchNode(
                    cost=new_cost,
                    state=next_state,
                    parent=node,
                    action_name=action_name,
                    depth=node.depth + 1,
                )
                heapq.heappush(frontier, child)

    return result


def a_star_search(
    start_state: StateT,
    is_goal: GoalTest,
    get_successors: Callable[[StateT], List[Tuple[StateT, float, str]]],
    heuristic: Heuristic,
) -> SearchResult:
    """
    A* 搜索算法。

    使用 f(n) = g(n) + h(n) 引导搜索。若 heuristic 是可接受的（Admissible），
    则保证找到最优解。h(n) 越接近实际代价，搜索效率越高。

    Args:
        start_state: 初始状态
        is_goal: 目标判定函数
        get_successors: 后继生成函数
        heuristic: 启发式函数 h(state) → 从该状态到目标的预估代价

    Returns:
        SearchResult 对象
    """
    result = SearchResult()
    frontier: List[Tuple[float, int, _SearchNode]] = []
    best_cost: Dict[Any, float] = {}
    counter = 0  # 用于打破 f(n) 平局

    h_start = heuristic(start_state)
    root = _SearchNode(cost=0, state=start_state, depth=0)
    heapq.heappush(frontier, (h_start, counter, root))
    counter += 1
    best_cost[start_state] = 0

    while frontier:
        f_val, _, node = heapq.heappop(frontier)
        result.nodes_expanded += 1

        if node.cost > best_cost.get(node.state, float("inf")):
            continue

        if is_goal(node.state):
            result.found = True
            result.path, result.actions = reconstruct_path(node)
            result.cost = node.cost
            return result

        for next_state, step_cost, action_name in get_successors(node.state):
            new_g = node.cost + step_cost
            if new_g < best_cost.get(next_state, float("inf")):
                best_cost[next_state] = new_g
                new_f = new_g + heuristic(next_state)
                child = _SearchNode(
                    cost=new_g,
                    state=next_state,
                    parent=node,
                    action_name=action_name,
                    depth=node.depth + 1,
                )
                heapq.heappush(frontier, (new_f, counter, child))
                counter += 1

    return result

def demo_graph_search() -> None:
    """
    演示在一个简单加权图上对比 BFS、DFS、Dijkstra 和 A* 四种算法。

    图结构 (ASCII):
              A ──3── B ──1── D
              │       │       │
              1       2       4
              │       │       │
              C ──5── E ──2── G (目标)

    起始状态：A
    目标状态：G
    """
    print("=" * 72)
    print("  图搜索算法对比演示")
    print("=" * 72)

    # 定义图的邻接关系：{节点: [(邻居, 代价, 动作描述), ...]}
    graph: Dict[str, List[Tuple[str, float, str]]] = {
        "A": [("B", 3, "A→B"), ("C", 1, "A→C")],
        "B": [("A", 3, "B→A"), ("D", 1, "B→D"), ("E", 2, "B→E")],
        "C": [("A", 1, "C→A"), ("E", 5, "C→E")],
        "D": [("B", 1, "D→B"), ("G", 4, "D→G")],
        "E": [("B", 2, "E→B"), ("C", 5, "E→C"), ("G", 2, "E→G")],
        "G": [("D", 4, "G→D"), ("E", 2, "G→E")],
    }

    # 启发式函数 h(n): 各节点到 G 的直线距离估计 (必须 ≤ 实际最短距离)
    heuristic_values: Dict[str, float] = {
        "A": 3,  # 直线 ≈ 3 (实际最短: A→C→E→G = 1+5+2=8 或 A→B→E→G = 3+2+2=7)
        "B": 3,  # 直线 ≈ 3 (实际最短: B→E→G = 2+2=4)
        "C": 4,  # 直线 ≈ 4 (实际最短: C→E→G = 5+2=7)
        "D": 3,  # 直线 ≈ 3 (实际最短: D→G = 4)
        "E": 2,  # 直线 ≈ 2 (实际最短: E→G = 2)
        "G": 0,  # 已到达目标
    }

    def get_successors(state: str) -> List[Tuple[str, float, str]]:
        return graph.get(state, [])

    def is_goal(state: str) -> bool:
        return state == "G"

    def heuristic(state: str) -> float:
        return heuristic_values[state]

    # 运行四种算法
    algorithms: Dict[str, Callable] = {
        "BFS": lambda: bfs_search("A", is_goal, get_successors),
        "DFS": lambda: dfs_search("A", is_goal, get_successors),
        "Dijkstra": lambda: dijkstra_search("A", is_goal, get_successors),
        "A*": lambda: a_star_search("A", is_goal, get_successors, heuristic),
    }

    for name, algo in algorithms.items():
        result = algo()
        status = "✓ 找到解" if result.found else "✗ 未找到"
        print(f"\n  [{name}]")
        print(f"    状态: {status}")
        print(f"    路径: {' → '.join(result.path)}")
        print(f"    动作: {' → '.join(result.actions)}")
        print(f"    总代价: {result.cost:.1f}")
        print(f"    扩展节点数: {result.nodes_expanded}")

    print("\n" + "-" * 72)
    print("  分析:")
    print("  - BFS: 找到步数最少的路径，但不保证总代价最小")
    print("  - DFS: 路径可能很深，不保证最优")
    print("  - Dijkstra: 保证总代价最小 (A→B→E→G = 3+2+2=7)")
    print("  - A*: 同样最优，且扩展节点数通常少于 Dijkstra")
    print("=" * 72)

class TaskStatus(Enum):
    """任务执行状态"""
    SUCCESS = auto()
    FAILURE = auto()
    RUNNING = auto()


@dataclass
class WorldState:
    """
    世界状态表示。

    使用字典存储键值对来模拟谓词逻辑。
    例如: {"at": "home", "has_ticket": True, "budget": 5000}
    """
    facts: Dict[str, Any] = field(default_factory=dict)

    def get(self, key: str, default: Any = None) -> Any:
        return self.facts.get(key, default)

    def set(self, key: str, value: Any) -> None:
        self.facts[key] = value

    def matches(self, conditions: Dict[str, Any]) -> bool:
        """
        检查当前状态是否满足给定的条件集合。

        Args:
            conditions: 条件字典，如 {"at": "home", "has_ticket": True}

        Returns:
            所有条件是否都满足
        """
        for key, expected in conditions.items():
            if self.facts.get(key) != expected:
                return False
        return True

    def apply_effects(self, effects: Dict[str, Any]) -> "WorldState":
        """
        应用动作效果，返回新的 WorldState (不修改原对象)。

        Args:
            effects: 效果字典，如 {"at": "airport", "budget": -500}

        Returns:
            新的 WorldState 对象
        """
        new_state = WorldState(self.facts.copy())
        for key, value in effects.items():
            if isinstance(value, (int, float)) and key in new_state.facts:
                # 数值类型的效果视为增量
                new_state.facts[key] = new_state.facts[key] + value
            else:
                new_state.facts[key] = value
        return new_state

    def copy(self) -> "WorldState":
        return WorldState(self.facts.copy())

    def __repr__(self) -> str:
        return f"WorldState({self.facts})"


class HtnOperator:
    """
    HTN 算子：表示一个原子动作 (Primitive Action)。

    每个算子包含：
      - name: 动作名称
      - preconditions: 执行前必须满足的状态条件
      - effects: 执行后对状态的修改
      - cost: 执行代价
    """

    def __init__(
        self,
        name: str,
        preconditions: Dict[str, Any],
        effects: Dict[str, Any],
        cost: float = 1.0,
    ) -> None:
        self.name = name
        self.preconditions = preconditions
        self.effects = effects
        self.cost = cost

    def is_applicable(self, state: WorldState) -> bool:
        """检查当前状态下此算子是否可执行"""
        return state.matches(self.preconditions)

    def apply(self, state: WorldState) -> WorldState:
        """
        执行此算子，返回新状态。
        注意：调用前应先用 is_applicable 检查前提条件。
        """
        if not self.is_applicable(state):
            raise ValueError(
                f"算子 '{self.name}' 的前提条件不满足！"
                f"\n  需要: {self.preconditions}"
                f"\n  当前: {state.facts}"
            )
        return state.apply_effects(self.effects)

    def __repr__(self) -> str:
        return (
            f"Operator({self.name}, pre={self.preconditions}, "
            f"eff={self.effects}, cost={self.cost})"
        )


@dataclass
class HtnMethod:
    """
    HTN 方法：定义如何将一个复合任务分解为子任务。

    Attributes:
        name: 方法名称
        task: 此方法处理的复合任务名
        preconditions: 方法适用的前提条件
        subtasks: 分解后的子任务名称列表
    """
    name: str
    task: str
    preconditions: Dict[str, Any]
    subtasks: List[str]

    def is_applicable(self, state: WorldState, task_name: str) -> bool:
        """检查此方法是否可以应用于给定的任务和状态"""
        return task_name == self.task and state.matches(self.preconditions)

    def __repr__(self) -> str:
        return (
            f"Method({self.name}: {self.task} → "
            f"{' → '.join(self.subtasks) if self.subtasks else '[]'})"
        )


@dataclass
class HtnDomain:
    """
    HTN 规划域定义。

    包含：
      - operators: 所有原子算子
      - methods: 所有复合任务分解方法
      - compound_tasks: 复合任务名称集合
    """
    operators: Dict[str, HtnOperator] = field(default_factory=dict)
    methods: Dict[str, List[HtnMethod]] = field(default_factory=dict)
    compound_tasks: Set[str] = field(default_factory=set)

    def add_operator(self, op: HtnOperator) -> None:
        self.operators[op.name] = op

    def add_method(self, method: HtnMethod) -> None:
        if method.task not in self.methods:
            self.methods[method.task] = []
        self.methods[method.task].append(method)
        self.compound_tasks.add(method.task)

    def get_methods_for_task(self, task: str) -> List[HtnMethod]:
        """获取分解指定任务的所有方法"""
        return self.methods.get(task, [])

    def is_compound(self, task_name: str) -> bool:
        """判断任务名是否为复合任务"""
        return task_name in self.compound_tasks

    def is_primitive(self, task_name: str) -> bool:
        """判断任务名是否为原子任务"""
        return task_name in self.operators


class HtnPlanner:
    """
    HTN 规划器：实现递归任务分解。

    规划流程：
      1. 从根任务开始，若为原子任务则直接执行，更新状态
      2. 若为复合任务，查找可用的 Method 进行分解
      3. 将分解出的子任务递归处理
      4. 若某复合任务无可用方法，回溯尝试其他分解路径

    使用 DFS 策略在任务分解空间搜索（也可扩展为 BFS 或启发式搜索）。
    """

    def __init__(self, domain: HtnDomain) -> None:
        self.domain = domain
        self.debug: bool = False

    def plan(
        self,
        initial_state: WorldState,
        root_tasks: List[str],
        max_depth: int = 50,
    ) -> Optional[Tuple[List[str], WorldState]]:
        """
        执行 HTN 规划。

        Args:
            initial_state: 初始世界状态
            root_tasks: 需要完成的根任务列表
            max_depth: 最大分解深度

        Returns:
            若成功: (执行的动作序列, 最终状态)
            若失败: None
        """
        result = self._decompose(
            state=initial_state,
            task_queue=deque(root_tasks),
            executed_plan=[],
            depth=0,
            max_depth=max_depth,
        )
        if result:
            actions, final_state = result
            return (actions, final_state)
        return None

    def _decompose(
        self,
        state: WorldState,
        task_queue: deque,
        executed_plan: List[str],
        depth: int,
        max_depth: int,
    ) -> Optional[Tuple[List[str], WorldState]]:
        """
        递归分解任务队列。

        Args:
            state: 当前世界状态
            task_queue: 待处理的任务队列
            executed_plan: 已执行的动作序列
            depth: 当前分解深度
            max_depth: 最大深度限制

        Returns:
            成功返回 (动作序列, 最终状态)，失败返回 None
        """
        # 基础情况：所有任务完成
        if not task_queue:
            return (executed_plan, state)

        if depth > max_depth:
            if self.debug:
                print(f"  [DEBUG] 达到最大深度 {max_depth}，回溯")
            return None

        task_name = task_queue.popleft()

        if self.debug:
            indent = "  " * depth
            print(f"{indent}[分解] 任务: '{task_name}' | 状态: {state.facts}")

        # 情况 1：原子任务 — 直接执行算子
        if self.domain.is_primitive(task_name):
            op = self.domain.operators[task_name]
            if op.is_applicable(state):
                new_state = op.apply(state)
                new_plan = executed_plan + [task_name]
                if self.debug:
                    print(f"{indent}  → 执行原子动作 '{task_name}' 成功")
                # 递归处理剩余任务
                return self._decompose(
                    new_state, task_queue, new_plan, depth + 1, max_depth
                )
            else:
                if self.debug:
                    print(f"{indent}  → 原子动作 '{task_name}' 前提条件不满足，回溯")
                return None

        # 情况 2：复合任务 — 尝试所有可用的方法 (带回溯)
        methods = self.domain.get_methods_for_task(task_name)
        if not methods:
            if self.debug:
                print(f"{indent}  → 复合任务 '{task_name}' 无可用方法")
            return None

        for method in methods:
            if not method.is_applicable(state, task_name):
                if self.debug:
                    print(f"{indent}  → 方法 '{method.name}' 前提条件不满足")
                continue

            if self.debug:
                print(
                    f"{indent}  → 尝试方法 '{method.name}': "
                    f"{' → '.join(method.subtasks)}"
                )

            # 将分解出的子任务插入队首 (保持顺序)
            new_queue = deque(method.subtasks)  # type: deque[str]
            # 先处理子任务，再处理原来的剩余任务
            new_queue.extend(task_queue)

            result = self._decompose(
                state=state,
                task_queue=new_queue,
                executed_plan=list(executed_plan),
                depth=depth + 1,
                max_depth=max_depth,
            )
            if result is not None:
                return result  # 找到解

            if self.debug:
                print(f"{indent}  ← 方法 '{method.name}' 失败，尝试下一种方法")

        # 所有方法都失败
        return None

    def plan_all(
        self,
        initial_state: WorldState,
        root_tasks: List[str],
        max_depth: int = 50,
    ) -> List[Tuple[List[str], WorldState]]:
        """
        搜索所有可能的规划方案 (用于对比分析)。
        """
        all_plans: List[Tuple[List[str], WorldState]] = []
        self._decompose_all(
            state=initial_state,
            task_queue=deque(root_tasks),
            executed_plan=[],
            depth=0,
            max_depth=max_depth,
            results=all_plans,
        )
        return all_plans

    def _decompose_all(
        self,
        state: WorldState,
        task_queue: deque,
        executed_plan: List[str],
        depth: int,
        max_depth: int,
        results: List[Tuple[List[str], WorldState]],
    ) -> None:
        """搜索所有解 (不回溯停止)"""
        if not task_queue:
            results.append((executed_plan, state))
            return

        if depth > max_depth:
            return

        task_name = task_queue.popleft()

        if self.domain.is_primitive(task_name):
            op = self.domain.operators[task_name]
            if op.is_applicable(state):
                new_state = op.apply(state)
                new_plan = executed_plan + [task_name]
                self._decompose_all(
                    new_state, task_queue, new_plan, depth + 1, max_depth, results
                )
            return

        methods = self.domain.get_methods_for_task(task_name)
        for method in methods:
            if method.is_applicable(state, task_name):
                new_queue = deque(method.subtasks)
                new_queue.extend(task_queue)
                self._decompose_all(
                    state=state,
                    task_queue=new_queue,
                    executed_plan=list(executed_plan),
                    depth=depth + 1,
                    max_depth=max_depth,
                    results=results,
                )

def build_travel_domain() -> HtnDomain:
    """
    构建旅行规划领域的 HTN 知识库。

    领域概述 (ASCII):
    ┌──────────────────────────────────────────────────────┐
    │  顶层任务: PlanTrip (规划一次旅行)                     │
    │    ├── BookTransport (预订交通)                       │
    │    │     ├── ChooseFlight (选择航班)                  │
    │    │     └── PayTicket (支付票款)                     │
    │    ├── BookAccommodation (预订住宿)                   │
    │    │     └── ReserveHotel (预订酒店)                  │
    │    ├── PrepareDocuments (准备证件)                    │
    │    │     └── CheckPassport (检查护照)                 │
    │    └── PackLuggage (打包行李)                         │
    │          └── PackItems (整理物品)                     │
    └──────────────────────────────────────────────────────┘
    """
    domain = HtnDomain()

    # ----- 原子算子 -----

    # 选择航班
    domain.add_operator(HtnOperator(
        name="ChooseFlight",
        preconditions={"destination_selected": True},
        effects={"flight_chosen": True},
        cost=0.5,
    ))

    # 支付票款
    domain.add_operator(HtnOperator(
        name="PayTicket",
        preconditions={"flight_chosen": True, "budget": lambda b: b >= 1000},
        effects={"ticket_paid": True, "budget": -1200},
        cost=0.3,
    ))
    # 修复 PayTicket 前提条件 — 使用数值比较
    # (在 is_applicable 中通过 matches 方法处理，因此需要调整)
    domain.operators["PayTicket"].preconditions = {
        "flight_chosen": True,
        "_min_budget": 1000,
    }

    # 预订酒店
    domain.add_operator(HtnOperator(
        name="ReserveHotel",
        preconditions={"destination_selected": True},
        effects={"hotel_booked": True, "budget": -2000},
        cost=0.7,
    ))

    # 检查护照
    domain.add_operator(HtnOperator(
        name="CheckPassport",
        preconditions={},
        effects={"passport_ready": True},
        cost=0.2,
    ))

    # 整理物品
    domain.add_operator(HtnOperator(
        name="PackItems",
        preconditions={"ticket_paid": True, "hotel_booked": True},
        effects={"luggage_packed": True},
        cost=1.0,
    ))

    # 前往机场
    domain.add_operator(HtnOperator(
        name="GoToAirport",
        preconditions={"luggage_packed": True, "passport_ready": True},
        effects={"at_airport": True},
        cost=1.5,
    ))

    # 登机
    domain.add_operator(HtnOperator(
        name="BoardFlight",
        preconditions={"at_airport": True, "ticket_paid": True},
        effects={"on_plane": True, "at_destination": True},
        cost=0.5,
    ))

    # ----- 复合任务方法 -----

    # 预订交通 = 选择航班 + 支付票款
    domain.add_method(HtnMethod(
        name="TransportByAir",
        task="BookTransport",
        preconditions={"destination_selected": True},
        subtasks=["ChooseFlight", "PayTicket"],
    ))

    # 预订住宿 = 预订酒店
    domain.add_method(HtnMethod(
        name="AccommodationByHotel",
        task="BookAccommodation",
        preconditions={"destination_selected": True},
        subtasks=["ReserveHotel"],
    ))

    # 准备证件 = 检查护照
    domain.add_method(HtnMethod(
        name="DocCheckMethod",
        task="PrepareDocuments",
        preconditions={},
        subtasks=["CheckPassport"],
    ))

    # 打包行李 = 整理物品
    domain.add_method(HtnMethod(
        name="PackMethod",
        task="PackLuggage",
        preconditions={"ticket_paid": True, "hotel_booked": True},
        subtasks=["PackItems"],
    ))

    # 出发 = 前往机场 + 登机
    domain.add_method(HtnMethod(
        name="DepartMethod",
        task="DepartForTrip",
        preconditions={
            "luggage_packed": True,
            "passport_ready": True,
            "ticket_paid": True,
        },
        subtasks=["GoToAirport", "BoardFlight"],
    ))

    # ---- 顶层任务：PlanTrip ----
    # 两种规划策略:
    #   策略A: 先交通 → 住宿 → 证件 → 打包 → 出发
    #   策略B: 先证件 → 交通 → 住宿 → 打包 → 出发 (可并行)
    domain.add_method(HtnMethod(
        name="PlanSequential",
        task="PlanTrip",
        preconditions={"destination_selected": True},
        subtasks=[
            "BookTransport",
            "BookAccommodation",
            "PrepareDocuments",
            "PackLuggage",
            "DepartForTrip",
        ],
    ))

    domain.add_method(HtnMethod(
        name="PlanParallel",
        task="PlanTrip",
        preconditions={"destination_selected": True},
        subtasks=[
            "PrepareDocuments",   # 证件准备可与交通/住宿并行
            "BookTransport",
            "BookAccommodation",
            "PackLuggage",
            "DepartForTrip",
        ],
    ))

    return domain


def demo_travel_planning() -> None:
    """
    旅行行程规划完整案例。

    展示 HTN 规划器如何分解旅行规划任务，
    并对比不同策略的执行结果。
    """
    print("\n" + "=" * 72)
    print("  HTN 旅行行程规划案例")
    print("=" * 72)

    domain = build_travel_domain()

    # 初始状态：选择目的地为"巴黎"，预算充足
    initial_state = WorldState({
        "destination_selected": True,
        "budget": 10000,
        "passport_ready": False,
        "ticket_paid": False,
        "flight_chosen": False,
        "hotel_booked": False,
        "luggage_packed": False,
        "at_airport": False,
        "at_destination": False,
        "on_plane": False,
    })

    print(f"\n  初始状态:")
    for k, v in initial_state.facts.items():
        print(f"    {k}: {v}")

    print(f"\n  目标任务: ['PlanTrip']")
    print(f"  领域定义:")
    print(f"    原子算子: {list(domain.operators.keys())}")
    print(f"    复合任务: {list(domain.compound_tasks)}")

    # ---- 执行规划 ----
    planner = HtnPlanner(domain)
    planner.debug = True

    print(f"\n  {'─' * 68}")
    print(f"  开始 HTN 规划...")
    print(f"  {'─' * 68}\n")

    result = planner.plan(initial_state, ["PlanTrip"])

    if result:
        actions, final_state = result
        total_cost = sum(
            domain.operators[a].cost for a in actions
            if a in domain.operators

            cost_info = f"(cost={op.cost})" if op else ""
            print(f"    {i:2d}. {action} {cost_info}")

        print(f"\n  最终状态:")
        for k, v in final_state.facts.items():
            print(f"    {k}: {v}")
        print(f"\n  关键指标:")

        print(f"    已登机: {final_state.facts['on_plane']}")

    # ---- 搜索所有可能的规划方案 ----
    print(f"\n\n  {'─' * 68}")
    print(f"  搜索所有可能的规划方案...")
    print(f"  {'─' * 68}")

    all_plans = planner.plan_all(initial_state, ["PlanTrip"])
    print(f"\n  共找到 {len(all_plans)} 种规划方案:")

    for i, (plan_actions, plan_state) in enumerate(all_plans, 1):
        plan_cost = sum(
            domain.operators[a].cost for a in plan_actions
            if a in domain.operators
        )
        print(f"\n  方案 {i}:")
        print(f"    动作序列: {' → '.join(plan_actions)}")
        print(f"    总代价: {plan_cost:.1f}")
        print(f"    步数: {len(plan_actions)}")

    print("\n" + "=" * 72)

def demo_astar_with_htn() -> None:
    """
    结合 A* 搜索与 HTN 分层思想：在任务分解空间中用启发式引导选择。

    场景：简化的城市间旅行规划
    状态 = 当前城市 + 已完成的任务
    目标 = 到达指定城市

    图结构 (ASCII):
          北京 ──500km── 上海
           │              │
         300km          200km
           │              │
          南京 ──400km── 杭州 ──100km── 目标(深圳)
    """
    print("\n" + "=" * 72)
    print("  A* + HTN 结合：分层旅行路线规划")
    print("=" * 72)

    # ---- 定义城市间距离 (带权图) ----
    CityGraph = Dict[str, List[Tuple[str, float, str]]]

    city_graph: CityGraph = {
        "北京": [("上海", 500, "高铁G1"), ("南京", 300, "高铁G3")],
        "上海": [("北京", 500, "高铁G2"), ("杭州", 200, "高铁G5")],
        "南京": [("北京", 300, "高铁G4"), ("杭州", 400, "高铁G7")],
        "杭州": [("上海", 200, "高铁G6"), ("南京", 400, "高铁G8"),
                 ("深圳", 100, "高铁G9")],
        "深圳": [("杭州", 100, "高铁G10")],
    }

    # 启发式：各城市到深圳的直线距离 (km, 低估 = admissible)
    straight_line_to_goal: Dict[str, float] = {
        "北京": 600,   # 实际最短约 300+400+100 = 800
        "上海": 300,   # 实际最短约 200+100 = 300
        "南京": 450,   # 实际最短约 400+100 = 500
        "杭州": 100,   # 实际最短 = 100
        "深圳": 0,
    }

    # 城市经纬度 (用于可视化距离)
    city_coords: Dict[str, Tuple[float, float]] = {
        "北京": (116.4, 39.9),
        "上海": (121.5, 31.2),
        "南京": (118.8, 32.1),
        "杭州": (120.2, 30.3),
        "深圳": (114.1, 22.5),
    }

    def euclidean_heuristic(city: str) -> float:
        """基于经纬度的欧几里得距离启发式 (可接受)"""
        import math
        if city == "深圳":
            return 0.0
        cx, cy = city_coords[city]
        gx, gy = city_coords["深圳"]
        # 简化：1 度 ≈ 100 km
        return math.sqrt((cx - gx)**2 + (cy - gy)**2) * 100

    def get_neighbors(state: str) -> List[Tuple[str, float, str]]:
        return city_graph.get(state, [])

    def is_destination(state: str) -> bool:
        return state == "深圳"

    # ---- 运行对比 ----
    print("\n  [图结构]")
    print("    北京 ──500km── 上海")
    print("     │              │")
    print("   300km          200km")
    print("     │              │")
    print("    南京 ──400km── 杭州 ──100km── 深圳(目标)")
    print()

    algorithms_for_comparison = [
        ("Dijkstra (无启发)", lambda: dijkstra_search("北京", is_destination, get_neighbors)),
        ("A* (直线距离)", lambda: a_star_search("北京", is_destination, get_neighbors, straight_line_to_goal.__getitem__)),
        ("A* (欧几里得)", lambda: a_star_search("北京", is_destination, get_neighbors, euclidean_heuristic)),
    ]

    for name, algo in algorithms_for_comparison:
        result = algo()
        status = "✓" if result.found else "✗"
        print(f"  [{name}] {status}")
        print(f"    路径: {' → '.join(result.path)}")
        print(f"    动作: {' → '.join(result.actions)}")
        print(f"    总代价: {result.cost:.0f} km")
        print(f"    扩展节点: {result.nodes_expanded}")
        print()

    print("-" * 72)
    print("  分析:")
    print("  - Dijkstra 扩展了所有方向，节点数较多")
    print("  - A* 的启发式 h(n) 引导搜索朝向目标，扩展节点数更少")
    print("  - 好的启发式应尽量接近实际代价，但又不能高估 (Admissible)")
    print("=" * 72)
