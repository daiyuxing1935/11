# 记忆模块：短期记忆与长期记忆的设计与实现

## 一、智能体为什么需要记忆

人类能够进行连贯的对话和有效的决策，离不开记忆系统。想象一下，如果你的朋友在聊天时每句话之后都忘记了你之前说过什么，交流将变得多么困难。同样，智能体也需要记忆来维持对话的上下文连贯性、学习用户偏好、积累任务执行经验。

从技术角度看，大语言模型本身是无状态的——每次API调用都是独立的，模型不会自动记住之前的对话内容。因此，我们需要在模型之外构建记忆系统，在每次调用时把相关的历史信息"喂"给模型。

## 二、记忆的分类

借鉴认知心理学的理论，智能体的记忆系统通常分为以下层次：

### 感官记忆（Sensory Memory）

这是最短暂的记忆，保存原始输入。对于智能体来说，就是刚刚收到的用户消息。它的特点是非常短暂，仅在下一次推理时被使用，然后就被处理或丢弃。

### 短期记忆 / 工作记忆（Short-term / Working Memory）

短期记忆保存当前会话中的上下文信息。在一个对话中，用户说了什么、智能体回答了什么、调用了什么工具——这些信息构成了短期记忆。它通常以消息列表的形式存在，类似于聊天记录。

短期记忆的容量是有限的。大语言模型有上下文窗口限制（如128K tokens），而且随着消息列表增长，API调用的成本和延迟也会增加。

### 长期记忆（Long-term Memory）

长期记忆保存跨会话的关键信息。例如：
- 用户的偏好设置和个性化信息
- 之前完成的任务和结果
- 学到的知识和经验
- 重要的对话片段

长期记忆需要持久化存储，并在需要时被检索出来。

## 三、使用缓冲区实现对话记忆

缓冲区（Buffer）是实现短期记忆最直接的方式。

### 简单缓冲区

维护一个消息列表，每次与模型交互时将整个列表作为上下文发送：

```python
class ConversationBufferMemory:
    def __init__(self):
        self.messages = []
    
    def add_user_message(self, message):
        self.messages.append({"role": "user", "content": message})
    
    def add_ai_message(self, message):
        self.messages.append({"role": "assistant", "content": message})
    
    def get_context(self):
        return self.messages
```

这种方式的优点是简单直接，模型能看到完整的对话历史。缺点也很明显——对话越长，Token消耗越大，最终可能超出模型的上下文窗口。

### 滑动窗口缓冲区

只保留最近的N条消息：

```python
class ConversationBufferWindowMemory:
    def __init__(self, window_size=10):
        self.messages = []
        self.window_size = window_size
    
    def add_message(self, message):
        self.messages.append(message)
        # 保留最近的window_size条消息
        if len(self.messages) > self.window_size:
            self.messages = self.messages[-self.window_size:]
```

这种方式的优势是内存占用可控，但会丢失较早的对话信息。

### 摘要缓冲区

当对话变长时，用模型自动生成历史摘要，替代原始消息：

```python
class ConversationSummaryMemory:
    def __init__(self, llm):
        self.messages = []
        self.summary = ""
        self.llm = llm
        self.summary_threshold = 20  # 超过20条消息后触发摘要
    
    def add_message(self, message):
        self.messages.append(message)
        if len(self.messages) > self.summary_threshold:
            self._summarize()
    
    def _summarize(self):
        old_messages = self.messages[:-5]  # 保留最近5条
        prompt = f"请用一段话总结以下对话：\n{old_messages}"
        self.summary = self.llm.generate(prompt)
        self.messages = self.messages[-5:]  # 只保留最近的
    
    def get_context(self):
        if self.summary:
            return [{"role": "system", "content": f"对话历史摘要：{self.summary}"}] + self.messages
        return self.messages
```

这种方式在保留核心信息的同时大幅减少了Token消耗。

## 四、向量化的语义记忆

对于一些需要根据内容语义进行检索的记忆，简单的消息列表就不够用了。这时候我们需要用到**向量化语义记忆**。

### 基本原理

1. **文本转向量（Embedding）**：用嵌入模型将文本转换为高维向量
2. **存储向量**：将向量和原文一起存入向量数据库（如Chroma、Pinecone、Milvus）
3. **相似度检索**：当需要相关记忆时，将查询也转为向量，在数据库中搜索最相似的记录
4. **返回结果**：将检索到的记忆片段放入模型的上下文

### 代码示例

```python
from chromadb import Client
from chromadb.utils import embedding_functions

class SemanticMemory:
    def __init__(self, collection_name="agent_memory"):
        self.client = Client()
        self.ef = embedding_functions.DefaultEmbeddingFunction()
        self.collection = self.client.get_or_create_collection(
            name=collection_name,
            embedding_function=self.ef
        )
    
    def store(self, content, metadata=None):
        """存储一段记忆"""
        doc_id = str(uuid.uuid4())
        self.collection.add(
            ids=[doc_id],
            documents=[content],
            metadatas=[metadata or {}]
        )
        return doc_id
    
    def retrieve(self, query, top_k=5):
        """根据语义检索相关记忆"""
        results = self.collection.query(
            query_texts=[query],
            n_results=top_k
        )
        return results["documents"][0]
    
    def delete(self, doc_id):
        """删除某段记忆"""
        self.collection.delete(ids=[doc_id])
```

### 实际应用场景

- **用户偏好记忆**："用户喜欢简洁的回答风格"——存储后，每次对话前检索
- **知识积累**："数据库连接字符串是xxx"——智能体学到的重要信息
- **经验记录**："上次使用API A处理这个问题效果很好"

## 五、记忆检索策略

如何从海量的长期记忆中精准地找到当前任务需要的内容？以下是几种常用策略：

### 基于时间衰减

越近的记忆越重要。为每条记忆分配一个时间衰减权重：

```python
def calculate_relevance(memory, current_time, decay_rate=0.1):
    age = current_time - memory.timestamp
    return memory.base_score * math.exp(-decay_rate * age)
```

### 基于重要性评分

为每条记忆赋予重要性分数。被频繁访问的记忆、与关键决策相关的记忆获得更高的分数。

### 混合检索

结合语义相似度、时间衰减和重要性评分，计算综合检索分数：

```python
def hybrid_score(memory, query_embedding, current_time):
    semantic_score = cosine_similarity(query_embedding, memory.embedding)
    time_score = time_decay(memory.timestamp, current_time)
    importance_score = memory.importance
    return 0.5 * semantic_score + 0.2 * time_score + 0.3 * importance_score
```

## 六、记忆的巩固与遗忘

就像人脑一样，智能体的记忆系统也需要维护机制。

### 记忆巩固（Memory Consolidation）

将短期记忆中的重要内容转移到长期记忆中：

```python
class MemoryConsolidator:
    def consolidate(self, short_term_memory, long_term_memory):
        """从短期记忆中提取关键信息，存入长期记忆"""
        prompt = f"""
        从以下对话中提取需要长期记住的关键信息。
        只提取事实性、偏好性、经验性的内容，忽略问候等日常对话。
        
        对话内容：
        {short_term_memory.get_all_messages()}
        
        以JSON列表格式输出，每项包含type（fact/preference/experience）和content。
        """
        key_info = self.llm.generate(prompt)
        for item in json.loads(key_info):
            long_term_memory.store(
                content=item["content"],
                metadata={"type": item["type"]}
            )
```

### 记忆遗忘（Forgetting）

不是所有记忆都应该永久保留。遗忘机制包括：

- **时间过期**：超过一定时间的记忆自动删除
- **访问频率衰减**：长期未被访问的记忆降低权重
- **容量限制**：当记忆总量超过上限时，删除最不重要的记忆
- **信息更新**：当新信息与旧信息冲突时，用新信息替代

## 七、LangChain记忆类实战

LangChain提供了丰富的记忆组件，我们可以直接使用：

```python
from langchain.memory import (
    ConversationBufferMemory,
    ConversationBufferWindowMemory,
    ConversationSummaryMemory,
    ConversationSummaryBufferMemory,
    VectorStoreRetrieverMemory
)
from langchain_openai import ChatOpenAI
from langchain.chains import ConversationChain

# 使用摘要缓冲区记忆
llm = ChatOpenAI(model="gpt-4", temperature=0)

memory = ConversationSummaryBufferMemory(
    llm=llm,
    max_token_limit=2000,  # 缓冲区token上限
    return_messages=True
)

conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=True
)

# 模拟对话
conversation.predict(input="我叫张三，是一名Python开发者。")
conversation.predict(input="我喜欢用FastAPI开发后端。")
# 询问较早的信息，智能体仍能记住
conversation.predict(input="我叫什么名字？我是做什么的？")
```

ConversationSummaryBufferMemory是一个很好的折中方案：它在内存中保留最近的原始消息，同时将较早的消息压缩为摘要，既保留了细节又控制了Token消耗。

## 八、设计记忆系统的最佳实践

1. **分层设计**：同时使用短期缓冲区和长期向量存储，各司其职
2. **并非越多越好**：只给模型提供真正相关的记忆，避免信息噪音
3. **考虑隐私**：记忆可能包含敏感信息，需要加密存储和权限控制
4. **可观测性**：记录记忆的读写操作，方便调试"智能体为什么这样做"
5. **成本意识**：向量存储和检索也有成本，不是所有信息都值得长期保存

记忆是智能体从"工具"进化为"伙伴"的关键一步。有了有效的记忆系统，智能体才能真正做到"记住你是谁、记得你们聊过什么、从过去的经验中学习"，提供连贯、个性化的体验。
