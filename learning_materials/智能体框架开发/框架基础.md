# LangChain框架基础

## 什么是LangChain

LangChain是由Harrison Chase于2022年创建的开源框架，目前是最流行的**大语言模型（LLM）应用开发框架**。它提供了一套标准化的工具和抽象，让开发者能够快速构建基于LLM的复杂应用，而不用从零开始处理模型调用、提示管理、记忆系统、工具集成等底层细节。

截至2024年，LangChain在GitHub上获得了超过90000颗星，形成了一个庞大的生态系统，包括：
- **LangChain**：核心框架，支持Python和JavaScript/TypeScript
- **LangSmith**：LLM应用的调试、测试、评估和监控平台
- **LangServe**：将LangChain链部署为REST API的工具
- **LangGraph**：用于构建有状态、多参与者的Agent应用
- **LangChain Hub**：社区共享提示词模板的平台

## 为什么需要LLM开发框架

直接用LLM的API（如OpenAI API）开发应用看起来很简单，但当应用变得复杂时，会遇到一系列共性问题：

**问题1：提示词管理混乱**
```python
# 没有框架时，提示词散落在代码各处
response1 = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "翻译为英文：" + text1}]
)
# 另一处代码
response2 = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "把这段话翻译成英语，保持语气：" + text2}]
)
# 提示词格式不一致，修改起来很痛苦
```

**问题2：多步骤流程难以管理**
```
用户问题 → 搜索知识库 → 筛选相关文档 → 拼接上下文 → 生成回答 → 后处理格式
```
每一步都需要处理输入输出、错误处理、状态传递，代码很快变得复杂。

**问题3：记忆系统需要自己实现**
对话机器人需要记住历史消息、控制上下文长度、管理多轮对话状态——这些都是非平凡的系统设计问题。

**问题4：工具集成繁琐**
需要让LLM使用搜索引擎、数据库、计算器、API等工具，每种工具的集成方式和输出格式都不同。

**问题5：缺乏评估和调试手段**
很难回答"这个提示词的改动真的让效果变好了吗？"、"为什么这次模型的输出质量下降了？"。

LangChain的目标就是将这些问题抽象为简洁、可组合、可复用的组件，让开发者专注于业务逻辑而非基础设施。

## 核心概念深入解析

### 1. Model（模型）—— 统一的LLM接口

LangChain提供了两层模型抽象：
- **LLM（大语言模型）**：接收文本字符串，返回文本字符串。适合简单的文本补全场景。
- **ChatModel（聊天模型）**：接收消息列表（包含角色信息），返回消息。适合对话场景。

```python
# LLM 示例 —— 文本输入，文本输出
from langchain_openai import OpenAI

llm = OpenAI(model="gpt-3.5-turbo-instruct", temperature=0.7)
result = llm.invoke("用一句话解释什么是机器学习")
print(result)  # "机器学习是让计算机从数据中自动学习规律和模式..."

# ChatModel 示例 —— 消息列表输入，消息输出
from langchain_openai import ChatOpenAI
from langchain.schema import HumanMessage, SystemMessage, AIMessage

chat = ChatOpenAI(model="gpt-4", temperature=0.7)

messages = [
    SystemMessage(content="你是一位幽默的技术科普作家。"),
    HumanMessage(content="什么是机器学习？用一句话解释。")
]

response = chat.invoke(messages)
print(response.content)  # "机器学习就像是教电脑'举一反三'..."
```

LangChain统一了不同模型提供商的调用接口。无论底层是OpenAI、Anthropic（Claude）、Google（Gemini）、国内的大模型（文心、通义、智谱），还是本地部署的开源模型（通过Ollama、vLLM等），调用代码的结构都是一致的：

```python
# 不同提供商的模型，调用方式高度统一
from langchain_openai import ChatOpenAI           # OpenAI GPT系列
from langchain_anthropic import ChatAnthropic      # Anthropic Claude系列
from langchain_google_genai import ChatGoogleGenerativeAI  # Google Gemini
from langchain_community.chat_models import ChatOllama     # 本地Ollama模型

# 所有模型都有相同的 .invoke() 接口
def call_model(model, prompt):
    return model.invoke(prompt)

# 可以轻松切换模型进行对比实验
models = {
    "gpt-4": ChatOpenAI(model="gpt-4"),
    "claude-3": ChatAnthropic(model="claude-3-opus-20240229"),
    "gemini": ChatGoogleGenerativeAI(model="gemini-pro"),
}

for name, model in models.items():
    print(f"=== {name} ===")
    print(call_model(model, "什么是人工智能？"))
```

### 2. Prompt Template（提示模板）—— 提示词工程化

Prompt Template将提示词的结构和内容分离，实现了提示词的复用和参数化。

```python
from langchain.prompts import ChatPromptTemplate, PromptTemplate
from langchain.prompts import MessagesPlaceholder

# 基础模板：变量注入
template = PromptTemplate(
    input_variables=["language", "text"],
    template="将以下文本翻译为{language}：\n\n{text}"
)
prompt = template.format(language="日语", text="人工智能正在改变世界")
# 输出: "将以下文本翻译为日语：\n\n人工智能正在改变世界"

# 聊天模板：支持系统消息、用户消息、AI消息
chat_template = ChatPromptTemplate.from_messages([
    ("system", "你是一位精通{target_language}的专业翻译。"),
    ("human", "请将以下内容翻译为{target_language}，保持专业术语的准确性：\n\n{source_text}"),
    ("ai", "好的，我会确保翻译准确。请提供需要翻译的内容。"),
    ("human", "内容就是上面那段话，请开始翻译。"),
])

messages = chat_template.format_messages(
    target_language="英文",
    source_text="深度学习的核心在于通过多层神经网络的逐层抽象来提取数据中的高级特征。"
)
```

**Few-shot提示模板**：在模板中嵌入示例，让模型通过示例学习任务模式。

```python
from langchain.prompts import FewShotChatMessagePromptTemplate

examples = [
    {
        "input": "这个产品质量太差了，用了两天就坏了！",
        "output": "情感：负面 | 方面：产品质量 | 强度：强"
    },
    {
        "input": "快递速度很快，包装也很用心，好评！",
        "output": "情感：正面 | 方面：物流配送、包装 | 强度：强"
    },
    {
        "input": "价格还行，中规中矩吧。",
        "output": "情感：中性 | 方面：价格 | 强度：弱"
    },
]

example_prompt = ChatPromptTemplate.from_messages([
    ("human", "{input}"),
    ("ai", "{output}"),
])

few_shot_prompt = FewShotChatMessagePromptTemplate(
    example_prompt=example_prompt,
    examples=examples,
)

final_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个情感分析助手。请分析用户评论的情感、涉及方面和情感强度。"),
    few_shot_prompt,
    ("human", "{input}"),
])
```

**Pipeline Prompt（流水线提示模板）**：对于复杂的多变量模板，使用Pipeline组合多个模板。

```python
from langchain.prompts.pipeline import PipelinePromptTemplate

# 角色设定模板
role_template = PromptTemplate.from_template(
    "你是一位{role}，拥有{experience}年经验。"
)

# 任务描述模板
task_template = PromptTemplate.from_template(
    "请{task_description}。要求：{requirements}"
)

# 组合为完整模板
full_template = PromptTemplate.from_template(
    "{role_setting}\n\n{task_instruction}"
)

pipeline_prompt = PipelinePromptTemplate(
    final_prompt=full_template,
    pipeline_prompts=[
        ("role_setting", role_template),
        ("task_instruction", task_template),
    ]
)

result = pipeline_prompt.format(
    role="资深Python工程师",
    experience="10",
    task_description="写一个冒泡排序函数",
    requirements="代码要有详细注释，时间复杂度为O(n²)"
)
```

### 3. Chain（链）—— 构建处理流水线

Chain是LangChain最核心的抽象，它将多个处理步骤串联成一条可执行的流水线。最常见的LCEL（LangChain Expression Language）使用`|`（管道）操作符组合步骤：

```python
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema import StrOutputParser

# 使用LCEL构建一个翻译链
model = ChatOpenAI(model="gpt-4")

prompt = ChatPromptTemplate.from_template(
    "将以下{source_lang}文本翻译为{target_lang}：\n{text}"
)

# LCEL 管道语法：每个步骤的输出成为下一个步骤的输入
chain = prompt | model | StrOutputParser()

# 调用链
result = chain.invoke({
    "source_lang": "中文",
    "target_lang": "英文",
    "text": "机器学习和深度学习的区别是什么？"
})
```

**复杂Chain示例：检索增强生成（RAG）**

这是一个更贴近实际应用的Chain，实现了文档问答的完整流程：

```python
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain.prompts import ChatPromptTemplate
from langchain_community.document_loaders import TextLoader

# 步骤1：加载文档
loader = TextLoader("knowledge_base.txt")
documents = loader.load()

# 步骤2：文档分块
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,      # 每块1000个字符
    chunk_overlap=200,    # 相邻块重叠200字符，保持语义连贯
    separators=["\n\n", "\n", "。", "！", "？", "，", " ", ""]
)
chunks = text_splitter.split_documents(documents)

# 步骤3：创建向量存储
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

# 步骤4：创建检索器
retriever = vectorstore.as_retriever(
    search_type="similarity",  # 语义相似度检索
    search_kwargs={"k": 4}     # 返回最相关的4个文档块
)

# 步骤5：构建RAG提示模板
system_prompt = (
    "你是一个知识库问答助手。使用以下检索到的上下文来回答用户的问题。"
    "如果你不知道答案，请诚实地说不知道，不要编造信息。"
    "回答时请引用具体的上下文来源。"
    "\n\n检索到的上下文：\n{context}"
)

prompt = ChatPromptTemplate.from_messages([
    ("system", system_prompt),
    ("human", "{input}"),
])

# 步骤6：组合为完整RAG链
model = ChatOpenAI(model="gpt-4", temperature=0)
question_answer_chain = create_stuff_documents_chain(model, prompt)
rag_chain = create_retrieval_chain(retriever, question_answer_chain)

# 步骤7：使用
response = rag_chain.invoke({"input": "公司2024年的战略目标是什么？"})
print(response["answer"])
```

**多种Chain类型对比：**

| Chain类型 | 适用场景 | 数据流 | 复杂度 |
|-----------|---------|--------|--------|
| LLMChain | 单步LLM调用 | 线性 | 低 |
| SequentialChain | 多步骤顺序执行 | 线性 | 中 |
| RouterChain | 根据输入选择不同处理路径 | 分支 | 中 |
| RetrievalQA | 文档问答 | 检索→生成 | 中 |
| ConversationChain | 对话 | 循环+记忆 | 中 |
| Custom Chain | 任意自定义逻辑 | 任意 | 高 |

### 4. Memory（记忆）—— 让对话有上下文

记忆系统管理对话历史，让AI能够"记住"之前的交流。不同场景需要不同的记忆策略：

```python
from langchain.memory import (
    ConversationBufferMemory,
    ConversationBufferWindowMemory,
    ConversationSummaryMemory,
    ConversationSummaryBufferMemory,
    ConversationTokenBufferMemory,
)
from langchain_openai import ChatOpenAI
from langchain.chains import ConversationChain

llm = ChatOpenAI(model="gpt-4", temperature=0.7)

# 1. 完整记忆：保存所有对话历史
# 问题：对话越长，token消耗越多，最终可能超出上下文窗口
buffer_memory = ConversationBufferMemory(return_messages=True)

# 2. 滑动窗口记忆：只保留最近K轮对话
# 优点：控制token消耗；缺点："忘记"早期重要信息
window_memory = ConversationBufferWindowMemory(k=5, return_messages=True)

# 3. 摘要记忆：对历史对话进行摘要
# 优点：用较少token保留关键信息；缺点：细节可能丢失
summary_memory = ConversationSummaryMemory(
    llm=ChatOpenAI(model="gpt-4"),
    return_messages=True
)

# 4. 混合记忆：滑动窗口+摘要（推荐！）
# 保留最近窗口内的完整消息，超出窗口的自动摘要
hybrid_memory = ConversationSummaryBufferMemory(
    llm=ChatOpenAI(model="gpt-4"),
    max_token_limit=2000,  # 总token限制
    return_messages=True
)

# 5. Token限制记忆：基于token数量控制
token_memory = ConversationTokenBufferMemory(
    llm=llm,
    max_token_limit=500,
    return_messages=True
)

# 使用任意记忆创建对话链
conversation = ConversationChain(
    llm=llm,
    memory=hybrid_memory,
    verbose=True  # 打印详细的执行过程
)

# 模拟多轮对话
conversation.predict(input="你好，我是小明，我是一名大学生。")
conversation.predict(input="我喜欢编程，正在学习Python。")
conversation.predict(input="你还记得我的名字和爱好吗？")
# 模型能正确回答："你叫小明，是一名大学生，喜欢用Python编程。"
```

**记忆策略选择指南：**

| 策略 | 适用场景 | Token效率 | 信息保真度 |
|------|---------|-----------|-----------|
| Buffer（完整） | 短对话（<10轮） | 低 | 最高 |
| Window（窗口） | 只需近期上下文 | 中 | 中 |
| Summary（摘要） | 需要长期记忆 | 高 | 中（细节丢失） |
| Summary+Buffer | 长对话（推荐） | 高 | 高 |
| Token Buffer | 精确控制token | 可控 | 可控 |

### 5. Agent（智能体）—— 让LLM自主决策

Agent是LangChain最强大的功能——它让LLM不仅能够"说话"，还能够"做事"。Agent的核心循环是：

```
1. LLM分析当前状态，决定使用哪个工具
2. 执行工具调用
3. 接收工具返回的结果
4. 判断是否需要继续调用其他工具
5. 重复直到给出最终答案
```

**构建一个完整的Agent：**

```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain.tools import tool
from langchain.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_community.tools import DuckDuckGoSearchRun
import math

# 定义工具：使用@tool装饰器
@tool
def calculator(expression: str) -> str:
    """执行数学计算。输入应为有效的数学表达式，如 '2+3*4' 或 'sqrt(16)'"""
    try:
        # 安全评估数学表达式
        allowed_names = {
            "sqrt": math.sqrt, "sin": math.sin, "cos": math.cos,
            "abs": abs, "pow": pow, "pi": math.pi
        }
        result = eval(expression, {"__builtins__": {}}, allowed_names)
        return f"计算结果：{result}"
    except Exception as e:
        return f"计算出错：{str(e)}"

@tool
def get_current_weather(city: str) -> str:
    """获取指定城市的当前天气信息"""
    # 实际应用中应调用天气API
    weather_data = {
        "北京": "晴，25°C，湿度45%，风力3级",
        "上海": "多云转小雨，28°C，湿度70%，风力2级",
        "深圳": "雷阵雨，30°C，湿度85%，风力4级",
    }
    return weather_data.get(city, f"未找到{city}的天气数据")

@tool
def web_search(query: str) -> str:
    """搜索互联网获取最新信息"""
    search = DuckDuckGoSearchRun()
    return search.run(query)

# 创建Agent
llm = ChatOpenAI(model="gpt-4", temperature=0)
tools = [calculator, get_current_weather, web_search]

# 系统提示词
system_prompt = """你是一个有用的AI助手，可以使用工具来帮助用户解决问题。

使用工具时遵循以下规则：
1. 优先使用最合适的工具完成任务
2. 如果计算器可以解决，就不要搜索互联网
3. 如果需要最新信息，使用web_search
4. 每次只调用一个工具
5. 给出最终答案前确认信息完整"""

prompt = ChatPromptTemplate.from_messages([
    ("system", system_prompt),
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="agent_scratchpad"),
])

# 创建Agent和Executor
agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,         # 打印思考过程
    max_iterations=5,     # 最多5轮工具调用
    handle_parsing_errors=True,  # 处理输出解析错误
)

# 使用Agent
response = agent_executor.invoke({
    "input": "北京的天气怎么样？计算一下如果温度升高10°C会是多少华氏度？"
})
print(response["output"])
```


## LangChain生态对比：选择合适的工具

除了LangChain，市面上还有其他优秀的LLM应用开发工具。以下是主要选项的对比：

| 框架 | 定位 | 优势 | 劣势 | 适用场景 |
|------|------|------|------|---------|
| **LangChain** | 全能型LLM应用框架 | 生态完整，组件丰富，文档详尽 | 学习曲线陡峭，抽象层次多 | 从简单到复杂的各种LLM应用 |
| **LlamaIndex** | 数据增强LLM框架 | 数据处理和索引能力极强，RAG的首选 | Agent能力弱于LangChain | 文档问答、知识库、数据分析 |

### 示例1：智能翻译助手（5分钟）

```python
# 安装依赖
# pip install langchain langchain-openai

from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema import StrOutputParser

# 初始化模型
llm = ChatOpenAI(
    model="gpt-4",
    temperature=0.3,  # 翻译任务用较低温度
    openai_api_key="your-api-key"  # 或设置环境变量 OPENAI_API_KEY
)

# 创建翻译提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", """你是一位专业翻译专家。翻译规则：
1. 准确传达原文含义
2. 保持专业术语的一致性
3. 对文化特定表达进行适当本地化
4. 括号中标注不确定的地方"""),
    ("human", "将以下{source_lang}文本翻译为{target_lang}：\n\n{text}"),
])

# 构建链
translation_chain = prompt | llm | StrOutputParser()

# 使用
result = translation_chain.invoke({
    "source_lang": "英文",
    "target_lang": "中文",
    "text": "The quick brown fox jumps over the lazy dog. "
            "This pangram contains every letter of the English alphabet."
})

print(result)
# 输出：
# "敏捷的棕色狐狸跳过了懒狗。这个全字母句包含了英文字母表中的每一个字母。"
```

### 示例3：带记忆的客服机器人（30分钟）

```python
from langchain_openai import ChatOpenAI
from langchain.memory import ConversationSummaryBufferMemory
from langchain.chains import ConversationChain
from langchain.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_community.callbacks import get_openai_callback
import hashlib
import time

class CustomerServiceBot:
    def __init__(self, company_name: str, business_type: str):
        self.llm = ChatOpenAI(model="gpt-4", temperature=0.5)

        # 系统提示词
        self.system_prompt = f"""你是{company_name}的AI客服代表。请遵循以下服务准则：

1. **公司背景**：{company_name}是一家{business_type}
2. **服务态度**：热情、耐心、专业，使用"您"称呼客户
3. **问题处理**：
   - 先确认客户的问题和需求
   - 提供清晰、准确的解答
   - 如果问题超出范围，引导至人工客服
4. **隐私保护**：不询问不必要的个人信息
5. **结束对话**：确认客户问题已解决后再结束

当前时间：{time.strftime('%Y年%m月%d日 %H:%M')}
        """

        # 带摘要的记忆系统
        self.memory = ConversationSummaryBufferMemory(
            llm=self.llm,
            max_token_limit=1500,
            return_messages=True
        )

        # 创建对话链
        prompt = ChatPromptTemplate.from_messages([
            ("system", self.system_prompt),
            MessagesPlaceholder(variable_name="history"),
            ("human", "{input}"),
        ])

        self.conversation = ConversationChain(
            llm=self.llm,
            prompt=prompt,
            memory=self.memory,
            verbose=False,
        )

        # 会话管理
        self.active_sessions = {}

    def _get_session(self, user_id: str):
        """获取或创建用户会话"""
        if user_id not in self.active_sessions:
            self.active_sessions[user_id] = {
                "start_time": time.time(),
                "message_count": 0,
            }
        return self.active_sessions[user_id]

    def chat(self, user_id: str, message: str):
        """处理用户消息"""
        session = self._get_session(user_id)
        session["message_count"] += 1

        # 追踪token使用
        with get_openai_callback() as cb:
            response = self.conversation.predict(input=message)
            token_info = {
                "total_tokens": cb.total_tokens,
                "prompt_tokens": cb.prompt_tokens,
                "completion_tokens": cb.completion_tokens,
                "total_cost": cb.total_cost
            }

        return {
            "response": response,
            "message_count": session["message_count"],
            "tokens": token_info
        }

    def get_memory_summary(self):
        """获取对话记忆摘要"""
        return self.memory.load_memory_variables({})

# 使用示例
bot = CustomerServiceBot(
    company_name="智学科技",
    business_type="在线教育平台，提供编程、AI、数据科学等课程"
)

# 模拟对话
conversations = [
    ("user_001", "你好，我想了解一下你们的Python课程"),
    ("user_001", "我完全没有编程基础，适合吗？"),
    ("user_001", "课程费用是多少？可以分期吗？"),
    ("user_001", "好的，谢谢你的详细解答！"),
]

for user_id, message in conversations:
    result = bot.chat(user_id, message)
    print(f"\n用户{user_id}: {message}")
    print(f"客服: {result['response']}")
    print(f"本轮token消耗: {result['tokens']['total_tokens']}")
```

## 常见问题与最佳实践

**问题1：LangChain版本升级频繁，代码容易过时**

LangChain的API在0.1.x到0.2.x期间经历了较大变化。建议：
- 锁定版本号（如langchain==0.2.0），不要使用latest
- 关注官方迁移指南（Migration Guide）
- 优先使用LCEL（LangChain Expression Language）而非旧版Chain类
- 使用`langchain-openai`、`langchain-anthropic`等独立包而非`langchain.llms`

**问题2：verbose日志太多太乱**

```python
# 全局设置日志级别
import logging
logging.getLogger("langchain").setLevel(logging.WARNING)

# 或在特定Chain中控制
chain = SomeChain(verbose=False)  # 关闭单条链的日志
```

**问题3：回调（Callback）使用不当**

回调是LangChain中强大的扩展机制，但容易被忽略：

```python
from langchain.callbacks import StdOutCallbackHandler, FileCallbackHandler
from langchain.callbacks.tracers import LangChainTracer

# 同时输出到控制台和文件
callbacks = [
    StdOutCallbackHandler(),
    FileCallbackHandler("langchain_log.txt"),
]

chain.invoke({"input": "..."}, config={"callbacks": callbacks})
```

**最佳实践总结：**

1. **使用LCEL**：新的项目优先使用`|`管道操作符构建链，而不是旧的Chain子类。LCEL更简洁、更易于组合和测试。
2. **温度参数**：需要创造性的任务（写作、头脑风暴）用0.7-0.9；需要精确性的任务（翻译、分类、代码生成）用0-0.3；对话用0.5-0.7。
3. **错误处理**：始终使用try-except包裹LLM调用，处理API限流、超时、内容过滤等异常。
4. **成本控制**：使用`get_openai_callback()`或LangSmith追踪token消耗。对于频繁调用的应用，考虑缓存（`langchain.cache`）。
5. **从简单开始**：先用最简单的Chain实现功能，验证可行后再逐步添加记忆、Agent等复杂特性。
6. **测试提示词**：在LangSmith或独立的Playground中充分测试提示词，确认在各种输入下都能稳定输出，再集成到Chain中。
